# Story 80.3: Task Lifecycle Buttons — Complete, Skip, Snooze

**Epic:** 80 — Stream Deck Plugin (Elgato SDK)
**Status:** Not Started
**Priority:** P1
**Estimate:** S (3 actions, simple RPC calls)
**Repo:** `ArcavenAE/threedoors-streamdeck`

## Context

After selecting a door, users need buttons to act on the task: mark it complete, skip it, or snooze it. These are simple button actions that call the corresponding daemon API methods.

## Acceptance Criteria

- [ ] `com.arcavenae.threedoors.complete` action — calls `task.complete` on keyDown
- [ ] `com.arcavenae.threedoors.skip` action — calls `task.skip` on keyDown
- [ ] `com.arcavenae.threedoors.snooze` action — calls `task.snooze` on keyDown
- [ ] Snooze action supports configurable duration via property inspector (15m, 30m, 1h, 2h, tomorrow)
- [ ] All three show `showOk()` on success
- [ ] All three show `showAlert()` if no task is currently selected (daemon returns error)
- [ ] All three show `showAlert()` if daemon is disconnected
- [ ] Complete button shows brief celebration animation (green flash via `setState`)
- [ ] Button titles show "Complete", "Skip", "Snooze" (static — not dynamic)
- [ ] Icons designed for each action (checkmark, forward arrow, clock)
- [ ] Tests: each action handler, error states, snooze duration parameter

## Technical Notes

### Snooze Duration

The property inspector for snooze presents a dropdown:
- 15 minutes
- 30 minutes
- 1 hour (default)
- 2 hours
- Tomorrow (calculated as next day 9:00 AM local time)

The selected duration is sent as the `duration` param to `task.snooze`.

### Error Handling

If the daemon returns a "no task selected" error, the button flashes alert. The user learns they need to select a door first. This is better than disabling the buttons (Stream Deck SDK doesn't support dynamic enable/disable).
