## Story

As a user,
I want door borders to have a "raised surface" bevel effect with lighter top/left edges and darker bottom/right edges,
So that doors appear as 3D raised surfaces rather than flat outlines.

## Background

The classic "raised button" effect from GUI design works in TUI: top and left edges rendered in a lighter color (implied light source from top-left), bottom and right edges in a darker color (in shadow). Combined with the background fill from Story 56.1, this creates instant 3D "raised surface" perception — the missing depth cue that shadow alone cannot provide.

The hinge/handle asymmetry from Epic 48 already differentiates left from right borders structurally. Bevel lighting adds a color dimension to that asymmetry, reinforcing the physical door metaphor.

```
Border color assignment:
╓━━━━━━━━━━━━━━━━━━┑     ← top: Highlight color
║                  │     ← left (hinge): Highlight | right: ShadowEdge
║  Content         │
╟━━━━━━━━━━━━━━━━━━┤     ← divider: ShadowEdge color
║                  │
║                  ●
╘━━━━━━━━━━━━━━━━━━┙     ← bottom: ShadowEdge color
```

## Acceptance Criteria

**Given** a door is rendered in any of the 8 themes
**When** the top border row is displayed
**Then** the top border characters (horizontal line + corners) are rendered in the theme's `Highlight` color

**Given** a door is rendered in any of the 8 themes
**When** the left (hinge) border is displayed
**Then** the left border characters are rendered in the theme's `Highlight` color

**Given** a door is rendered in any of the 8 themes
**When** the right border and bottom border are displayed
**Then** these characters are rendered in the theme's `ShadowEdge` color

**Given** a door is rendered in any of the 8 themes
**When** the panel divider row is displayed
**Then** the divider is rendered in the theme's `ShadowEdge` color

**Given** a door is selected
**When** rendered with selection emphasis
**Then** all borders use the `Selected` color (overriding bevel), maintaining the existing selection visual

**Given** a door at minimum width (15 chars)
**When** rendered with bevel lighting
**Then** content area is unchanged (zero width cost)

**Given** existing golden file tests
**When** the bevel changes are applied
**Then** golden files are regenerated to match the new rendering

## Technical Notes

- Uses `Highlight` and `ShadowEdge` fields from `ThemeColors` (added in Story 56.1)
- Create two `lipgloss.Style` instances per theme render: one for Highlight (top/left), one for ShadowEdge (bottom/right)
- Top border row: apply Highlight style to all characters
- Left border (hinge chars ╓, ║, ╟, ╙): apply Highlight style
- Right border (│, ┐, ●, ┘): apply ShadowEdge style
- Bottom border row: apply ShadowEdge style
- Divider row (╟━━━┤): apply ShadowEdge style
- Selected override: when `selected == true`, both styles use `Selected` color
- ~15 LOC per theme (~120 LOC total), zero width cost
- **Can parallelize with Story 56.1** — both are independent changes to the render pipeline
- Must pass `go test -race ./internal/tui/...`
