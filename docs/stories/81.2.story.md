# Story 81.2: Persistent Agent Self-Sync Documentation

**Epic:** 81 — Git Coordination Fixes
**Status:** Done (PR #930)
**Priority:** P1
**Estimate:** S (< 1 hour)

## User Story

As a persistent agent operator,
I want agent definitions to clearly document that persistent agents CAN and SHOULD periodically fetch origin/main,
So that agents don't remain perpetually stale and operators don't waste time debugging staleness issues.

## Context

There is a widespread misconception in the agent system that ALL agents are blocked from running `git fetch`. In reality, the git safety hook only blocks fetch/pull in worker worktrees (paths containing `/.multiclaude/wts/`). Persistent agents sharing the main checkout CAN fetch freely.

However, no persistent agent definition currently instructs the agent to self-sync. The CLAUDE.md "Git Safety" section focuses on what's blocked but doesn't clarify what's allowed for persistent agents. This leads to:
- Agents accumulating stale local state over hours/days
- Operators manually running `git pull` in the shared checkout
- Confusion when agents report stale data

## Acceptance Criteria

- **AC1:** `CLAUDE.md` Git Safety section includes a note clarifying that persistent agents (non-worker worktree paths) are exempt from fetch restrictions
- **AC2:** `agents/pr-shepherd.md` includes a periodic `git fetch origin` in its startup or polling routine (pr-shepherd needs current refs for rebase operations)
- **AC3:** `agents/project-watchdog.md` includes a periodic `git fetch origin main` before planning doc checks (needs current main to detect drift)
- **AC4:** `agents/arch-watchdog.md` includes a note that it may fetch to check architecture docs against latest main
- **AC5:** Each agent's fetch instruction includes error handling guidance: log warning on failure, continue with stale data

## Technical Notes

- envoy and retrospector primarily use `gh` CLI (GitHub API) and don't need local refs as much — optional for them
- pr-shepherd already uses temporary worktrees for rebase operations but needs current refs to identify which PRs need rebasing
- The shared checkout working tree state matters less than the ref state — most agents use `gh` CLI

## Out of Scope

- Changing the git safety hook behavior
- Per-agent worktrees (that's a longer-term architectural change)
- Daemon-managed refresh of persistent agents (partially covered by Story 81.4)
