# Story 80.2: Door Action Buttons — Three Doors + Refresh

**Epic:** 80 — Stream Deck Plugin (Elgato SDK)
**Status:** Not Started
**Priority:** P1
**Estimate:** M (4 actions, dynamic titles, icon design)
**Repo:** `ArcavenAE/threedoors-streamdeck`

## Context

The core Stream Deck experience: three door buttons showing current task titles, plus a refresh button to re-roll. Each door button shows its task title and responds to press events by calling the daemon's `doors.select` method.

## Acceptance Criteria

- [ ] `com.arcavenae.threedoors.door` action registered with `controllers: ["Keypad"]`
- [ ] Action supports `settings.doorNumber` (1, 2, or 3) configured via property inspector
- [ ] Button title dynamically shows the task text (truncated to fit, ~15 chars)
- [ ] `keyDown` on a door button calls `doors.select` with the door number
- [ ] After selection, button shows visual feedback via `showOk()`
- [ ] If daemon is disconnected, `keyDown` shows `showAlert()` and button title shows "Offline"
- [ ] `com.arcavenae.threedoors.refresh` action calls `doors.refresh`
- [ ] Refresh button shows spinning animation during refresh (via rapid `setState` toggles)
- [ ] `willAppear` event triggers initial `doors.get` to populate button titles
- [ ] `doors.changed` server notification updates all door button titles automatically
- [ ] When no tasks are available, door buttons show "No tasks" with dimmed icon
- [ ] Placeholder door icons (Door 1 / Door 2 / Door 3) included in `.sdPlugin/imgs/`
- [ ] Tests: action keyDown handler, title truncation, offline state, event-driven updates

## Technical Notes

### Action Registration

Each door button is the same action type (`com.arcavenae.threedoors.door`) with a different `doorNumber` setting. This lets users place 1, 2, or 3 door buttons as they prefer.

### Title Truncation

Stream Deck button titles wrap at ~10 characters per line, with 2-3 lines visible. Use word-boundary truncation with ellipsis for long task names.

### Multi-Action Coordination

All door action instances share a single daemon client connection (singleton). When `doors.changed` fires, iterate all visible door action instances and update their titles.
