## Story

As a ThreeDoors user,
I want pressing quit to immediately exit the application,
So that the app respects my intent without adding friction.

## Background

Story 3.6 introduced an `ImprovementView` ("Session Reflection") that intercepts the quit action after productive sessions (1+ task completions or 5+ minutes). The user must press Enter or Esc a second time to actually exit. This violates three SOUL.md principles:

1. **Progress Over Perfection:** "Every design decision should reduce friction to starting, not add friction for 'correctness.'"
2. **Design Principle #1:** "Would this reduce friction? If yes, do it. If it adds a step for the user, don't."
3. **The Feeling We're Going For:** ThreeDoors should feel like a friend, not a survey.

The project owner's directive: "it's annoying and hostile to ask for input and require them to quit a second time."

## Acceptance Criteria

**AC-1: Immediate quit on q**
**Given** the user is on the doors view
**When** they press `q`
**Then** the application exits immediately with no intermediate screen

**AC-2: No improvement prompt**
**Given** a session with 1+ completed tasks or 5+ minutes elapsed
**When** the user presses `q`
**Then** the application exits immediately — no "Session Reflection" prompt is shown

**AC-3: Dead code removed**
**Given** the ImprovementView is no longer reachable
**When** the codebase is reviewed
**Then** `improvement_view.go`, `improvement_view_test.go`, and the improvement writer are deleted
**And** `ImprovementSubmittedMsg`, `ImprovementSkippedMsg` are removed from messages.go
**And** `ViewImprovement` is removed from the ViewMode enum

**AC-4: Existing tests pass**
**Given** the improvement view code is removed
**When** `make test` is run
**Then** all tests pass with zero failures
**And** `go test -race ./internal/tui/...` passes
