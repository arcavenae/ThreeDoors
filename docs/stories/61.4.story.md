# Story 61.4: Content Split — CLI, MCP, Configuration & Advanced

## Status: Done (PR #646)

## Epic

Epic 61: GitHub Pages User Guide

## References

- Research: PRs #481 and #500
- Planning: `_bmad-output/planning-artifacts/gh-pages-user-guide-plan.md`

## Story

As a power user or developer extending ThreeDoors,
I want complete CLI reference, MCP server docs, configuration reference, and advanced feature guides on the docs site,
So that I can find detailed technical information in well-organized pages.

## Background

This story completes the content split by porting the remaining sections from the monolithic user guide: CLI reference, MCP server, configuration, themes, task dependencies, troubleshooting, extending (from adapter-developer-guide.md), and changelog. After this story, the docs site has full coverage of all user-facing documentation.

## Acceptance Criteria

**Given** the docs site infrastructure from Story 61.1
**When** CLI pages are created
**Then** `docs-site/docs/cli/commands.md` contains full CLI command reference with all subcommands, flags, and examples
**And** `docs-site/docs/cli/mcp-server.md` covers MCP server setup, available tools list, usage with LLM agents, and example prompts

**Given** the docs site
**When** configuration pages are created
**Then** `docs-site/docs/configuration/config-file.md` has complete `config.yaml` schema reference with all fields, defaults, and examples
**And** `docs-site/docs/configuration/environment.md` lists environment variables that affect behavior
**And** `docs-site/docs/configuration/data-directory.md` describes data directory layout, log files, and session files

**Given** the docs site
**When** remaining guide and advanced pages are created
**Then** `docs-site/docs/guide/themes.md` documents available themes with descriptions (screenshots noted as future enhancement)
**And** `docs-site/docs/advanced/task-dependencies.md` covers dependency types, linking, and blocking behavior
**And** `docs-site/docs/advanced/extending.md` documents writing custom TaskProvider implementations (from adapter-developer-guide.md)
**And** `docs-site/docs/troubleshooting.md` covers common issues, diagnostics commands, and FAQ
**And** `docs-site/docs/changelog.md` contains formatted content from CHANGELOG.md

**Given** all navigation in mkdocs.yml
**When** finalized
**Then** every page in the site is reachable from the navigation

**Given** the complete docs site
**When** `mkdocs build --strict` is run
**Then** the build succeeds with zero warnings
**And** all internal cross-links work
