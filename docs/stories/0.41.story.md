# Story 0.41: Adapters & Enrichment Test Coverage

**Epic:** 0 — Infrastructure & Quality
**Status:** Done (PR #492)
**Priority:** Medium

## Description

Improve test coverage for `internal/adapters` (contract tests) and `internal/enrichment` packages, which were hovering near the 75% floor.

## Acceptance Criteria

- [ ] `internal/adapters` coverage increased from 76.1% to 83%+
- [ ] `internal/enrichment` coverage increased from 80.4% to 85%+
- [ ] No non-test files modified
- [ ] All tests pass (`make test`)
- [ ] Code passes `make fmt` and `make lint`

## Implementation Notes

### Adapters (76.1% → 83.8%)

Added `contract_readonly_test.go` with:
- **Read-only provider**: Exercises `skipIfReadOnly` branches (ErrReadOnly path) in all contract test functions
- **Watching provider**: Exercises the non-nil `Watch()` channel path in `testWatchReturnsChannelOrNil`
- **Healthy provider**: Exercises `HealthCheck()` returning non-empty Items
- **In-memory provider**: Thread-safe map-based provider exercising all success paths

Remaining gap to 85%: The uncovered lines are `t.Fatalf`/`t.Errorf` assertion branches inside contract test functions that only fire when a provider is buggy — structurally unreachable for correct provider implementations.

### Enrichment (80.4% → 87.6%)

Added `enrichment_coverage_test.go` with:
- Error path tests: closed DB operations, sql.ErrNoRows wrapping, unknown migration version
- Timestamp verification: CreatedAt/UpdatedAt populated on insert
- Tag round-trip: empty tags and multi-tag serialization
- Cross-reference: multiple pairs, bidirectional queries
- Feedback: multiple entries, session/task queries
- Learning patterns: upsert updates, multiple keys per type
- Delete verification: all delete operations verify removal
