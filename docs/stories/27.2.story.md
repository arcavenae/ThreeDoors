# Story 27.2: Review Incomplete Tasks Flow

## Story Info

- **Epic:** 27 — Daily Planning Mode
- **Priority:** P1
- **Status:** Done (PR #339)
- **Estimated Effort:** Medium
- **Dependencies:** 27.1 (planning data model)

## User Story

As a ThreeDoors user, I want to review yesterday's incomplete tasks at the start of my planning session so that I can decide which tasks to continue, defer, or drop before selecting today's focus.

## Acceptance Criteria

### AC 27.2.1: ReviewView Bubbletea Model
- [x] `ReviewView` implements `tea.Model` with `Init()`, `Update()`, `View()` methods
- [x] Constructor: `NewReviewView(incompleteTasks []*Task) *ReviewView`
- [x] Displays one task at a time with full task text, status, and tags
- [x] Shows progress counter: "Task 2/5"

### AC 27.2.2: Review Actions
- [x] `C` key — Continue: marks task for continued focus consideration (no status change)
- [x] `D` key — Defer: marks task to remain in pool without special treatment (no status change)
- [x] `X` key — Drop: transitions task to deferred status
- [x] `Enter` — advances to next task after action, or completes review step when all tasks processed
- [x] Each action records the decision for metrics (continued/deferred/dropped counts)

### AC 27.2.3: Empty State
- [x] When there are zero incomplete tasks, ReviewView displays "No incomplete tasks from yesterday" and auto-advances to the next step after a brief pause (1 second)

### AC 27.2.4: Visual Design
- [x] Task displayed in a Lipgloss-styled panel matching existing TUI aesthetic
- [x] Action keys clearly displayed at bottom: `[C]ontinue  [D]efer  [X]Drop`
- [x] Progress bar or step indicator at bottom: "Step 1/3 - Review"
- [x] Elapsed time shown subtly (e.g., top-right corner)

### AC 27.2.5: Navigation
- [x] `Esc` key — skips remaining reviews and advances to Step 2 (select)
- [x] `?` key — shows help overlay with action descriptions
- [x] Review decisions can be undone with `U` (undo) before advancing past a task

## Technical Notes

- Follow existing Bubbletea model patterns (see onboarding flow in `internal/tui/`)
- Use Lipgloss for all styling — no raw ANSI codes
- ReviewView is a sub-model composed within PlanningView
- All user-visible text through `View()` — no `fmt.Println`
