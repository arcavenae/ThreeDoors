# Story 77.1: Remove Justfile from CI Code Filter

## Status: Done (PR #877)

## References

- Trigger: PR #866 (Story 73.5) ran full Go Quality Gate despite only adding shell scripts + docs

## Story

As a contributor,
I want the CI code filter to not trigger on justfile-only changes,
So that scripts-only and docs-only PRs skip the expensive Go Quality Gate.

## Background

The `dorny/paths-filter` `code` filter in `ci.yml` includes `justfile`. PR #866 added `quota-status` targets to the justfile (no Go code changed), which triggered the full Quality Gate (~2-3 min of Go tests, linting, vet, govulncheck, coverage). Justfile changes to Go build/test recipes are almost always paired with `.go` file changes; the rare justfile-only change to Go targets would be caught locally via `just test`.

## Acceptance Criteria

**AC1: Justfile removed from code filter**
**Given** the `ci.yml` path filter configuration
**When** a PR modifies only `justfile`, `scripts/**`, and/or `docs/**`
**Then** the `code` output is `false`
**And** the Quality Gate job is skipped

**AC2: Go-related PRs still trigger Quality Gate**
**Given** a PR that modifies `.go` files, `go.mod`, `go.sum`, `.golangci.yml`, or `.github/workflows/**`
**When** the path filter runs
**Then** the `code` output is `true`
**And** the Quality Gate runs as before

**AC3: Push-to-main still runs full suite**
**Given** a push event to `main`
**When** CI runs
**Then** Quality Gate executes regardless of which files changed (existing behavior, unchanged)

## Tasks

### Task 1: Remove `justfile` from code filter in `ci.yml`
Remove the `- 'justfile'` line from the `code:` filter in the `changes` job.

### Task 2: Verify skip-pass job covers justfile-only PRs
