## Story

As a user,
I want to re-authenticate a data source when its token expires without having to disconnect and reconnect,
So that I can fix auth issues quickly and resume syncing.

## Background

When a connection enters AuthExpired state, the user presses `r` in the source detail view to trigger re-auth. For API token providers, this opens a masked input field for a new token. For OAuth providers, this opens the device code flow (Story 46.2-46.3). On success, credentials are updated in the keychain and the connection transitions back to Connecting → Connected.

**Note:** OAuth re-auth depends on Epic 46, which is not yet implemented. This story implements the full re-auth framework with API token re-entry as the primary flow. OAuth re-auth will show a "not yet supported" message until Epic 46 lands.

## Acceptance Criteria

**AC1: API Token Re-Auth Dialog**
**Given** a connection in AuthExpired state with API token auth (todoist, jira, linear)
**When** `r` is pressed in the source detail view
**Then** a masked input field appears for the new API token
**And** the provider's TokenHelp text is shown as guidance
**And** on submission, the token is updated in the credential store
**And** a connection test runs to verify the new token
**And** on success, the connection transitions AuthExpired → Connecting → Connected
**And** a flash message confirms "Re-authenticated successfully"

**AC2: Re-Auth Failure Handling**
**Given** re-authentication fails (empty token submitted, health check fails)
**When** the failure occurs
**Then** an error message is shown via flash
**And** the connection remains in AuthExpired state
**And** the user is returned to the source detail view

**AC3: Re-Auth Cancellation**
**Given** the re-auth dialog is open
**When** the user presses Escape
**Then** the dialog is dismissed
**And** the connection state is unchanged
**And** the user returns to the source detail view

**AC4: OAuth Re-Auth Placeholder**
**Given** a connection in AuthExpired state with OAuth auth (github)
**When** `r` is pressed
**Then** a flash message shows "OAuth re-authentication not yet supported — disconnect and reconnect"
**And** the connection state is unchanged

**AC5: Non-AuthExpired Guard**
**Given** a connection NOT in AuthExpired state
**When** `r` is pressed in the source detail view
**Then** a flash message shows "Re-authentication only available for expired connections"
**And** no dialog is opened

**AC6: Event Logging**
**Given** a successful re-authentication
**Then** a `token_refreshed` event is logged via SyncEventLog

## Technical Notes

### Architecture

- **ReauthDialog** (`internal/tui/reauth_dialog.go`): New Bubbletea component, follows DisconnectDialog pattern
  - Uses `huh.NewForm` with `huh.NewInput().EchoMode(huh.EchoModePassword)` for masked token entry
  - Emits `ReauthCompleteMsg{ConnectionID, NewToken}` on submit
  - Emits `ReauthCancelledMsg{}` on Esc/abort
  - Shows provider name and TokenHelp from `ProviderFormSpec`

- **ConnectionService.ReAuthenticate()** (`internal/core/connection/service.go`): New method
  - Updates credential via `CredentialStore.Set(ConnCredentialKey(conn), newToken)`
  - Transitions `AuthExpired → Connecting`
  - Runs `TestConnection()` to verify
  - On health check success: transitions `Connecting → Connected`, logs `EventTokenRefreshed`
  - On health check failure: transitions back to `AuthExpired` with error, returns error

- **MainModel changes** (`internal/tui/main_model.go`):
  - Add `connSvc *ConnectionService` field + `SetConnectionService()` setter
  - Add `reauthDialog *ReauthDialog` field
  - Add `ViewReauth` ViewMode constant
  - Handle `SourceActionMsg{Action: "reauth"}`:
    - Guard: connection must be in AuthExpired state
    - Look up auth type from providerSpecs
    - OAuth → flash "not yet supported"
    - API Token → spawn ReauthDialog
  - Handle `ReauthCompleteMsg`: call `connSvc.ReAuthenticate()`, flash result
  - Handle `ReauthCancelledMsg`: dismiss dialog, return to detail view

- **Message types** (`internal/tui/messages.go`):
  - `ReauthCompleteMsg{ConnectionID string, NewToken string}`
  - `ReauthCancelledMsg{}`

### Auth Type Detection

Provider auth type is stored in `ProviderFormSpec.AuthType`. To look up the auth type for a connection's provider, iterate `DefaultProviderSpecs()` matching on `conn.ProviderName == spec.Name`.

### Existing Infrastructure Used

- State machine already allows `AuthExpired → Connecting` and `Connecting → Connected`
- `ConnCredentialKey(conn)` builds credential store key
- `huh.NewInput().EchoMode(huh.EchoModePassword)` for masked token input
- `SyncEventLog.LogTokenRefreshed(connID)` for event logging
- `HealthChecker.CheckHealth()` for token verification
- DisconnectDialog pattern for dialog lifecycle
