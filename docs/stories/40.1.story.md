## Story

As a user,
I want the insights dashboard to have a structured, visually appealing layout with bordered panels and a hero number,
So that viewing my stats feels like a celebration rather than reading a plain text log.

## Background

The current `insights_view.go` renders all stats as plain text with manual spacing (`fmt.Fprintf` with `%-5d` alignment). No Lipgloss styling is applied to data sections. The view has four sections (completion trends, streaks, mood correlations, door preferences) rendered sequentially with no visual hierarchy.

This story restructures the entire `View()` method to use Lipgloss bordered panels arranged in a responsive 2-column layout, adds a hero number at the top ("X tasks completed since you started"), and implements 4 terminal width breakpoints (compact < 60, narrow 60-79, standard 80-119, wide 120+).

## Acceptance Criteria

**Given** the insights view is opened
**When** the view renders
**Then** a styled header "YOUR INSIGHTS DASHBOARD" appears at the top with a Lipgloss border
**And** a hero number displays the total tasks completed with gold-colored styling
**And** each stats section (completion trends, streaks, mood, door picks) renders inside a Lipgloss bordered panel with a section title

**Given** a terminal width of 80-119 columns
**When** the insights view renders
**Then** panels are arranged in a 2-column layout using `lipgloss.JoinHorizontal()`
**And** the total rendered width does not exceed the terminal width

**Given** a terminal width less than 60 columns
**When** the insights view renders
**Then** all panels stack vertically in a single column
**And** no content is truncated or wraps incorrectly

**Given** a terminal width of 120+ columns
**When** the insights view renders
**Then** the top row uses 3 columns (sparkline + streaks + door picks)
**And** the mood section uses full width below

**Given** the insights view has insufficient data (< minimum sessions)
**When** the view renders
**Then** the existing "Keep using ThreeDoors" message displays with styling
**And** no empty bordered panels are shown

## Technical Notes

- Primary changes in `internal/tui/insights_view.go` (major rewrite of `View()`)
- Add stats-specific styles to `internal/tui/styles.go` (panel borders, hero number color)
- Use `lipgloss.NewStyle().Border(lipgloss.RoundedBorder()).BorderForeground(lipgloss.Color("#555555"))`
- Hero number uses `lipgloss.Color("#FCD34D")` (gold)
- Use `lipgloss.AdaptiveColor` for light/dark terminal support
- Add `layoutMode()` method returning "compact", "narrow", "standard", or "wide"
- Add `height int` field to InsightsView for future viewport support
- Implement render caching: compute once, re-render only on resize
- Add golden file tests at 80 and 120 column widths
