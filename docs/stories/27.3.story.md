# Story 27.3: Focus Selection & Energy Matching Flow

## Story Info

- **Epic:** 27 — Daily Planning Mode
- **Priority:** P1
- **Status:** Done (PR #352)
- **Estimated Effort:** Medium
- **Dependencies:** 27.1 (planning data model)

## User Story

As a ThreeDoors user, I want to select my focus tasks for the day filtered by my current energy level so that I commit to tasks that match my capacity and see them prioritized in door selection.

## Acceptance Criteria

### AC 27.3.1: SelectView Bubbletea Model
- [x] `SelectView` implements `tea.Model` with `Init()`, `Update()`, `View()` methods
- [x] Constructor: `NewSelectView(pool []*Task, energy EnergyLevel) *SelectView`
- [x] Displays scrollable task list filtered and sorted by energy match
- [x] Shows selection count: "Focus: 2/3 selected" (target 3, max 5)

### AC 27.3.2: Energy Level Display and Override
- [x] Energy level auto-inferred from time of day on view initialization
- [x] Current energy level displayed prominently: "Energy: High (morning)"
- [x] `E` key — cycles through energy levels: High -> Medium -> Low -> All (show everything)
- [x] When energy level changes, task list re-filters and re-sorts
- [x] Override tracked in metrics (`energy_overridden` field)

### AC 27.3.3: Focus Task Selection
- [x] `Space` key — toggles task as focus (adds/removes `+focus` tag)
- [x] Visual indicator on selected tasks (checkmark or highlight)
- [x] When 3 tasks are selected, show subtle "Target reached!" message but allow up to 5
- [x] When 5 tasks are selected, prevent additional selection with "Maximum 5 focus tasks" message
- [x] `Enter` — confirms selection and advances to Step 3 (confirm)

### AC 27.3.4: Task List Presentation
- [x] Tasks sorted by energy match score (best match first)
- [x] Each task shows: text, effort tag, type tag, current status
- [x] Arrow keys (Up/Down) navigate the list
- [x] List is scrollable when tasks exceed terminal height

### AC 27.3.5: Edge Cases
- [x] When task pool has fewer than 3 tasks, allow selection of all available
- [x] When no tasks match current energy level, show "No matching tasks" with option to switch to "All" energy filter
- [x] When zero tasks are selected and Enter pressed, show warning: "No focus tasks selected. Continue anyway? [Y/N]"

### AC 27.3.6: Navigation
- [x] `Esc` key — returns to Step 1 (review) to re-do decisions
- [x] `?` key — shows help overlay with key bindings
- [x] Step indicator: "Step 2/3 - Select Focus"

## Technical Notes

- Use `MatchesEnergy()` from Story 27.1 for filtering
- Use `InferEnergyFromTime()` from Story 27.1 for default
- Apply `+focus` tags via `TaskPool.AddTag()` from Story 27.1
- Scrollable list can use existing list patterns from TUI codebase
- All styling via Lipgloss
