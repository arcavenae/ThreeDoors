# Story 0.53: Docker E2E Scenario Expansion

## Status: Done (PR #621)

**Epic:** 0 — Infrastructure & Process
**Priority:** P2
**Depends On:** None

## User Story

As a developer,
I want the Docker E2E test suite to cover all primary user workflows,
So that I can be confident the full application works end-to-end in a clean environment.

## Context

The state-of-testing audit (R-001, 2026-03-09) noted that Docker E2E tests exist (Epic 18, ADR-0019) but scenario coverage is unclear. The three-tier TUI testing infrastructure (headless teatest, golden files, Docker E2E) is fully implemented, but the Docker E2E tier should be audited to verify all primary user workflows are represented.

## Acceptance Criteria

- AC1: Audit of existing Docker E2E scenarios documented (what's covered, what's missing)
- AC2: Missing scenarios identified and added for: task completion flow, task blocking flow, daily planning mode, and source connection flow
- AC3: Docker E2E tests pass locally via `docker compose -f docker-compose.test.yml up`
- AC4: Each new scenario has a descriptive name and clear pass/fail criteria
- AC5: No flaky tests — scenarios use deterministic timing with teatest's `WaitFor` pattern
- AC6: CI integration unchanged (Docker E2E runs on push only per Story 55.1)

## Technical Notes

- Docker E2E framework: `Dockerfile.test`, `docker-compose.test.yml`
- Existing E2E tests: `internal/tui/e2e_test.go` (576 lines)
- Uses `teatest` package from charmbracelet
- Golden file snapshots in `internal/tui/testdata/`
- Reference ADR-0019 for the three-tier testing architecture
