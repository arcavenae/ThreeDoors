## User Story

As a ThreeDoors user, I want to see a list of subtasks when viewing a parent task in detail view, so that I can track decomposition progress at a glance.

## Context

After Stories 31.1 (ParentID model) and 31.2 (sequential expand), parent tasks can have subtasks. This story adds visual rendering of the subtask list in the detail view, including status icons, an indented tree format, and a completion ratio summary.

**Architecture reference:** `_bmad-output/planning-artifacts/architecture-expand-fork.md` (Section 2, "Detail View Rendering")
**Party mode artifact:** `_bmad-output/planning-artifacts/party-mode-expand-fork-2026-03-08.md` (Sally's rendering spec)
**Requirements:** FR122


The subtask list renders between the task text and the separator line in `DetailView.View()`. Uses `pool.GetSubtasks(task.ID)` to retrieve children.

### Design Constraints

- Completion ratio is display-only — no enforcement of "complete all subtasks first"
- No auto-completion of parent when all children finish
- Use Lipgloss for styling — never ANSI escape codes directly
- Tree connector characters: `├─` for non-last, `└─` for last item
