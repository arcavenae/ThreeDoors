# Story 65.2: Subcommand Test Coverage ‚Äî Config, Mood, Health, Stats, Plan

## Status: Done (PR #739)

**Epic:** 65 ‚Äî CLI Test Coverage Hardening
**Priority:** P0
**Depends On:** None

## User Story

As a developer,
I want all CLI subcommands to have table-driven test coverage,
So that command-line argument parsing, flag handling, and output formatting are verified for every user-facing command.

## Context

The state-of-testing audit (TEA report, 2026-03-09) identified 0% coverage on config subcommands (`runConfigShow`, `runConfigGet`, `runConfigSet`), mood subcommands (`runMoodSet`, `runMoodHistory`), and the health check command (`runHealth`). These are actively used commands with no safety net against regressions.

## Acceptance Criteria

- AC1: `runConfigShow()` tested ‚Äî verify output includes all config fields, JSON mode works
- AC2: `runConfigGet()` tested ‚Äî table-driven tests for valid keys, invalid keys, and missing config
- AC3: `runConfigSet()` tested ‚Äî table-driven tests for valid key-value pairs, invalid keys, validation errors
- AC4: `runMoodSet()` tested ‚Äî valid mood values, invalid values, persistence verification
- AC5: `runMoodHistory()` tested ‚Äî empty history, populated history, JSON output mode
- AC6: `runHealth()` tested ‚Äî healthy state, unhealthy state, JSON output, provider-specific checks
- AC7: Stats command tested ‚Äî verify metric calculation, output formatting, empty data handling
- AC8: Plan command tested ‚Äî verify daily plan generation, empty task pool handling
- AC9: Version command tested ‚Äî verify version string formatting
- AC10: All tests use table-driven pattern with `t.Run` subtests
- AC11: All tests pass with `go test -race ./internal/cli/...`

## Technical Notes

- Config tests need a temp directory for config file I/O ‚Äî use `t.TempDir()`
- Mood tests need mock session log ‚Äî use `testkit` factories or temp JSONL files
- Health command depends on provider health checks ‚Äî mock the provider interface
- Stats command reads session logs ‚Äî create minimal JSONL fixtures in `testdata/`
- Use `bytes.Buffer` to capture command output for assertion
- Follow existing patterns in `doctor_test.go` and `llm_status_test.go`
