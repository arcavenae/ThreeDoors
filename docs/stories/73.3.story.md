## Story

As a project owner,
I want mechanical hook enforcement preventing workers from running dangerous git commands,
So that INC-002-style worktree corruption is impossible regardless of prompt compliance.

## Background

INC-002 documented workers running `git fetch origin main && git rebase origin/main` which caused mid-rebase conflicts in multiclaude-managed worktrees. The current prevention is prompt-level: worker agent definitions say "NEVER run git fetch, git pull, git rebase, or git merge." This is a suggestion that agents can violate.

The chainlink ecosystem (R-010) demonstrates hook-enforced discipline via `.claude/settings.json` PreToolUse hooks — code-level enforcement that cannot be bypassed by the agent. Per Q-C-005, hooks should replace prompt-level guardrails entirely.

Claude Code hooks system supports:
- `PreToolUse` hooks that run before a tool executes and can block it
- Pattern matching on tool names (Bash, Write, Edit, etc.)
- Exit code 2 = block the tool call with the hook's stdout as the reason

## Acceptance Criteria

**AC1: PreToolUse hook blocks dangerous git commands in worker worktrees**
**Given** a `.claude/settings.json` (or `.claude/settings.local.json`) contains a PreToolUse hook for Bash
**When** a worker agent attempts to run `git fetch`, `git pull`, `git rebase`, or `git merge` (with remote refs)
**Then** the hook exits with code 2 and provides a clear error message explaining why the command is blocked
**And** the worker agent sees the block reason and can adjust its approach

**AC2: Hook allows safe git operations**
**Given** the PreToolUse hook is active
**When** a worker runs `git add`, `git commit`, `git push`, `git status`, `git log`, `git diff`, `git branch`, or `git checkout -b`
**Then** the hook exits with code 0 (allow)

**AC3: Hook prevents unsigned commits**
**Given** the PreToolUse hook is active
**When** a worker attempts `git commit` with `--no-gpg-sign` or `-c commit.gpgsign=false`
**Then** the hook exits with code 2 with a message requiring signed commits

**AC4: Hook prevents direct push to main**
**Given** the PreToolUse hook is active
**When** a worker attempts `git push origin main` or `git push origin HEAD:main`
**Then** the hook exits with code 2 with a message requiring feature branches

**AC5: Hook prevents Co-Authored-By trailers**
**Given** the PreToolUse hook is active
**When** a worker attempts a `git commit` command containing "Co-Authored-By"
**Then** the hook exits with code 2 with a message explaining the policy

**AC6: Hook script is portable across worktrees**
**Given** multiclaude creates fresh worktrees for each worker
**When** the hook configuration references a hook script
**Then** the script path is relative to the repo root (not absolute) so it works in any worktree

**AC7: Hook configuration documented**
**Given** the hook is implemented
**When** CLAUDE.md is reviewed
**Then** the hook enforcement is documented, replacing the prompt-level INC-002 guardrail text with a reference to the mechanical enforcement

## Tasks

### Task 1: Create the git-safety hook script
Write a shell script (e.g., `scripts/hooks/git-safety.sh`) that:
- Reads the command from `$TOOL_INPUT` (JSON with "command" field)
- Parses git subcommands from the command string
- Blocks: `git fetch`, `git pull`, `git rebase`, `git merge` (with remote refs)
- Blocks: `--no-gpg-sign`, `-c commit.gpgsign=false`
- Blocks: `git push` to main/master
- Blocks: commit messages containing "Co-Authored-By"
- Allows: all other git commands and non-git commands
- Exits 0 (allow) or 2 (block with reason on stdout)

### Task 2: Configure .claude/settings.json PreToolUse hook
Add the hook configuration:
```json
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Bash",
        "command": "bash scripts/hooks/git-safety.sh"
      }
    ]
  }
}
```

### Task 3: Write tests for the hook script
Create test cases covering:
- Blocked commands (fetch, pull, rebase, merge, unsigned commit, push to main, Co-Authored-By)
- Allowed commands (add, commit -S, push origin work/*, status, log, diff)
- Edge cases (git commands embedded in pipes, quoted arguments)

### Task 4: Update CLAUDE.md
Replace prompt-level INC-002 guardrail with reference to hook enforcement. Keep the explanation of WHY these commands are dangerous but note that enforcement is now mechanical.

### Task 5: Update worker agent definition
Remove or reduce the INC-002 prompt-level warnings in the worker agent definition, noting that hooks now enforce these rules.

## Dev Notes

- The hook script must handle `$TOOL_INPUT` which is JSON: `{"tool_name": "Bash", "tool_input": {"command": "git fetch ..."}}`
- Claude Code passes tool input via environment variable or stdin — verify the exact mechanism
- The script should be fast (< 100ms) to avoid slowing down every Bash call
- Consider using simple grep/pattern matching rather than a full parser
- Must work in both worker worktrees and the main checkout
- Marvel portability: This hook pattern is the foundation for all DFCP enforcement — design it to be extractable
