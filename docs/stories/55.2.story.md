## Story

As a contributor,
I want benchmarks to only run on PRs that touch performance-relevant code,
So that non-performance PRs complete faster (3m33s → 2m08s).

## Background

The `benchmarks` job is the longest PR job at 3m33s and determines the PR critical path. It runs on every code-change PR, even when the changed code cannot affect performance (e.g., TUI rendering changes, new adapter code, documentation). The benchmarks test `internal/core/` and `internal/adapters/textfile/` — only changes to those packages (or `go.mod` dependency changes) can affect benchmark results.

**Party mode consensus:** Add path filtering. Push-to-main always runs benchmarks as a safety net.

**Rejected alternative:** Running benchmarks only on push-to-main — delays performance regression detection too much for the Brownian ratchet philosophy.

## Acceptance Criteria

**AC-1: Benchmarks skip on non-perf PRs**
**Given** a PR that only modifies files outside `internal/core/`, `internal/adapters/textfile/`, and `go.mod`
**When** CI runs
**Then** the `benchmarks` job does NOT execute
**And** the PR critical path is determined by Quality Gate (~2m08s)

**AC-2: Benchmarks run on perf PRs**
**Given** a PR that modifies any file in `internal/core/` or `internal/adapters/textfile/` or `go.mod`
**When** CI runs
**Then** the `benchmarks` job executes normally

**AC-3: Benchmarks always run on push-to-main**
**Given** a commit pushed to `main`
**When** CI runs
**Then** the `benchmarks` job always executes (regardless of which files changed)

**AC-4: Path filter correctly defined**
**Given** the changes detection job
**When** reviewed
**Then** a `perf` output is defined with paths:
- `internal/core/**`
- `internal/adapters/textfile/**`
- `go.mod`

## Technical Notes

**File: `.github/workflows/ci.yml`**

1. Add `perf` output to the `changes` job:
```yaml
changes:
  name: Detect Changes
  runs-on: ubuntu-latest
  outputs:
    code: ${{ steps.filter.outputs.code }}
    docker: ${{ steps.filter.outputs.docker }}
    perf: ${{ steps.filter.outputs.perf }}
  steps:
    - uses: actions/checkout@v6
    - uses: dorny/paths-filter@v3
      id: filter
      with:
        filters: |
          code:
            - '**.go'
            - 'go.mod'
            - 'go.sum'
            - 'Makefile'
            - '.golangci.yml'
            - '.github/workflows/**'
          docker:
            - 'Dockerfile*'
            - 'docker-compose*'
          perf:
            - 'internal/core/**'
            - 'internal/adapters/textfile/**'
            - 'go.mod'
```

2. Update the `benchmarks` job condition:
```yaml
benchmarks:
  name: Performance Benchmarks
  needs: changes
  if: github.event_name == 'push' || (needs.changes.outputs.code == 'true' && needs.changes.outputs.perf == 'true')
```

**Expected impact:** PR wall clock drops from 3m33s → 2m08s for ~70% of PRs (those not touching core/textfile).
