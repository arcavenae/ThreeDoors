## Story

As a CLI user,
I want to run `threedoors extract --file notes.txt` or pipe text via stdin to extract tasks,
So that I can batch-import tasks from scripts, shell workflows, and non-interactive contexts.

## Background

The CLI extraction command complements the TUI `:extract` command. It supports file input, clipboard input, and stdin piping. Output is human-readable by default with a `--json` flag for scripting.

## Acceptance Criteria

**AC-1: File input**
**Given** a text file
**When** `threedoors extract --file notes.txt` is run
**Then** tasks are extracted from the file contents
**And** displayed for confirmation before import

**AC-2: Clipboard input**
**Given** text on the system clipboard
**When** `threedoors extract --clipboard` is run
**Then** tasks are extracted from clipboard contents

**AC-3: Stdin input**
**Given** text piped via stdin
**When** `cat notes.txt | threedoors extract` is run
**Then** tasks are extracted from stdin
**And** stdin is auto-detected when no --file or --clipboard flag is given

**AC-4: Human-readable output**
**Given** extraction completes
**When** no format flags are set
**Then** tasks are displayed in a numbered list with text and effort
**And** user is prompted to confirm import: "Import all? [y/N/select]"

**AC-5: JSON output**
**Given** the `--json` flag
**When** extraction completes
**Then** output is a JSON array of ExtractedTask objects on stdout
**And** no interactive prompts are shown (for scripting)

**AC-6: Auto-import flag**
**Given** the `--yes` flag
**When** extraction completes
**Then** all extracted tasks are imported without confirmation prompt

**AC-7: Error messages**
**Given** the input file doesn't exist
**When** `threedoors extract --file missing.txt` is run
**Then** a clear error message is displayed
**Given** no LLM backend is available
**When** extract is run
**Then** a message explains how to install/configure an LLM backend
