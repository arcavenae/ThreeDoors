# Story 82.2: Add Explicit Scope Boundary Callout to Envoy Definition

**Epic:** 82 — Agent Definition Accuracy
**Status:** Not Started
**Priority:** P2
**Estimate:** S (< 1 hour)

## User Story

As the envoy agent,
I want a prominent SCOPE BOUNDARY section at the top of my definition,
So that my operational limits are immediately visible and I never attempt out-of-scope actions.

## Context

While `agents/envoy.md` already has a "What You Do NOT Do" section (line 279) and a "CANNOT (Forbidden)" authority table (line 304), these are buried deep in the document. Agent definitions are loaded into the system prompt — content near the top has stronger influence on agent behavior than content near the bottom.

A prominent SCOPE BOUNDARY callout near the top of the file would reinforce envoy's limits before the agent encounters any task instructions. This follows the pattern used by other agents (e.g., worker.md has "Incident-Hardened Guardrails" near the top).

## Acceptance Criteria

- **AC1:** `agents/envoy.md` has a `## SCOPE BOUNDARY` section added immediately after the "Your Mission" section (before "Your rhythm")
- **AC2:** The scope boundary clearly states: "You are the issues board. You screen, label, and relay. You do NOT create stories, run BMAD pipelines, write code, merge PRs, or authorize work of any kind."
- **AC3:** The scope boundary references the detailed authority table later in the document for the full breakdown
- **AC4:** Existing "What You Do NOT Do" section remains unchanged (the new section supplements, not replaces)

## Technical Notes

- `agents/envoy.md` is a CODEOWNERS-protected file — PR will need @skippy review
- Keep the callout concise (5-10 lines max) — it's a reinforcement, not a duplication of the full authority table
- Follow the pattern of worker.md's guardrails section for formatting consistency

## Out of Scope

- Changing envoy's actual capabilities
- Restructuring the entire envoy.md document
- Adding scope boundaries to other agent definitions (could be a follow-up)

## Implementation Notes

> [decision] 2026-04-02 — Place the scope boundary after "Your Mission" and before "Your rhythm" list. This position ensures it's read early in the context window before any operational instructions.
