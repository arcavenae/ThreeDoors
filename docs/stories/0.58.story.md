# Story 0.58: Fix govulncheck Vulnerabilities

**Epic:** 0 — Infrastructure & Maintenance
**Status:** Done (PR #761)
**Priority:** High
**Issue:** #592

## Description

Resolve all vulnerabilities reported by `govulncheck ./...`. The original issue (#592) reported 9 vulnerabilities: 7 in Go stdlib crypto/x509 (go1.25.4, fixed in go1.25.5) and 2 in jose2go@v1.5.0 (DoS vulns, fixed in v1.7.0).

By the time this story was picked up, the go.mod had already been bumped to `go 1.25.8` and jose2go to v1.7.0, resolving those original vulns. However, since the local Go toolchain is go1.26.0, govulncheck reported 4 new stdlib vulnerabilities (GO-2026-4599 through GO-2026-4602) fixed in go1.26.1.

The fix is to bump the `go` directive in go.mod to 1.26.1 so the Go toolchain downloads and uses go1.26.1, resolving all current vulnerabilities.
