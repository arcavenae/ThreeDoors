# Story 34.2: Create Custom Claude Code Slash Commands

**Status:** Done (PR #224)
**Epic:** 34 — SOUL.md + Custom Development Skills
**Depends On:** 34.1 (SOUL.md)
**Blocks:** Stories 34.3, 34.4

## User Story

As a developer (human or AI agent),
I want custom Claude Code slash commands for common ThreeDoors workflows,
So that pre-PR validation, pattern compliance, adapter checks, and story creation are automated and consistent.

## Acceptance Criteria

**Given** the need for repeatable, automated development workflows
**When** custom slash commands are created in `.claude/commands/`
**Then:**

- AC1: `/pre-pr` command exists that runs formatting, linting, testing, vet, branch freshness, scope review, and commit cleanliness checks — reports results in a summary
- AC2: `/validate-adapter` command exists that checks TaskProvider implementations for interface completeness, error wrapping, factory registration, test coverage, and atomic write usage
- AC3: `/check-patterns` command exists that scans for common anti-patterns: direct status mutation, direct file writes, fmt.Println in TUI code, panics in user-facing code, provider instantiation outside factory, missing error wrapping
- AC4: `/new-story` command exists that generates story files from a standard template with story ID, user story format, acceptance criteria, scope, architecture, testing, and task sections
- AC5: All commands are placed in `.claude/commands/` directory and are immediately invocable
- AC6: Commands reference CLAUDE.md and SOUL.md rules without duplicating them

## Quality Gate

- QG1: All four command files are valid markdown
- QG2: Commands are consistent with CLAUDE.md coding standards and SOUL.md philosophy
- QG3: No Go code changes — documentation/tooling only
- All standard checks pass (see CLAUDE.md)
- QG5: Scope-checked against this story — no extras
