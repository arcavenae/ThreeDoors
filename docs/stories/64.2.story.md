## Refined by Spike (ADR-0034)

Acceptance criteria refined based on Story 64.1 findings. Device identity uses UUID v5 seeded from machine ID + install path, with registration via the shared Git sync repository.

## User Story

As a ThreeDoors user with multiple computers,
I want each device to have a stable identity,
So that the sync system can track which device made which changes.

## Acceptance Criteria

- AC1: Device ID generated as UUID v5 (namespace UUID + machine-id + `~/.threedoors/` path hash) for deterministic stability across reinstalls
- AC2: Fallback to UUID v4 (random) when machine-id is unavailable (e.g., some Linux containers); still persisted for stability
- AC3: Device ID persisted in `~/.threedoors/device.yaml` with metadata: ID, human-readable name (defaults to `os.Hostname()`), first-seen timestamp (UTC), last-sync timestamp (UTC)
- AC4: Device registration writes entry to `devices/<device-id>.yaml` in the shared Git sync repo — other devices discover peers by reading this directory
- AC5: `threedoors devices` CLI command lists known devices (from sync repo `devices/` directory) with last-sync time and online/offline status
- AC6: `threedoors devices rename <id> <name>` updates the human-readable name in both local `device.yaml` and sync repo
- AC7: First device registration is automatic on first run — no manual setup required
- AC8: Second device registration triggers sync setup wizard (prompts for Git remote URL if not configured)
- AC9: `DeviceID` type is a named string type with validation (must be valid UUID v5 or v4)
- AC10: Unit tests for: ID generation determinism (same inputs → same ID), persistence round-trip, collision avoidance (different machines → different IDs), fallback to UUID v4

## Technical Notes

- Use `/etc/machine-id` (Linux) or `IOPlatformUUID` via `ioreg` (macOS) for machine-specific input
- UUID v5 namespace: generate a ThreeDoors-specific namespace UUID (hardcoded constant)
- Device metadata is local-first — synced to other devices via the Git sync repo `devices/` directory
- This story establishes the identity layer; transport (64.3) and conflict resolution (64.4) build on it
- `DeviceID` will be used in `FieldVersions` (ADR-0012 extension) for conflict attribution
