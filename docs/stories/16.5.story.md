# Story 16.5: Session Metrics & iCloud Sync

**Status:** Draft
**Epic:** 16 — iPhone Mobile App (SwiftUI)
**Depends on:** Stories 16.1, 16.2

## User Story

As a developer analyzing usage patterns,
I want mobile session metrics compatible with the desktop JSONL format,
So that mobile and desktop sessions can be analyzed together.

## Acceptance Criteria

**Given** the need for cross-device session analytics
**When** the session tracker is implemented
**Then:**

- AC1: `SessionTracker` Swift class mirrors Go's `SessionTracker` — tracks `session_id` (UUID), `start_time`, `end_time`, and behavioral counters
- AC2: Metrics recorded: `doors_viewed`, `tasks_completed`, `refreshes`, `status_changes`, `card_swipes`
- AC3: Session data appended to `sessions.jsonl` in the app's iCloud Drive container
- AC4: JSONL format matches Go's `MetricsWriter` output schema exactly — same field names, same JSON structure per line
- AC5: iCloud Drive sync configured for `~/.threedoors/` equivalent directory (app group container or iCloud Drive documents)
- AC6: Config file (`config.yaml`) readable from shared iCloud Drive location for note title and other settings
- AC7: Metrics written on app background/termination (`scenePhase` changes in SwiftUI lifecycle)
- AC8: Offline metrics cached locally in app sandbox, synced to iCloud Drive when connectivity available

## Quality Gate

## Technical Notes

- Use `@Environment(\.scenePhase)` in SwiftUI to detect app backgrounding/termination for metric flush
- iCloud Drive container: use `FileManager.default.url(forUbiquityContainerIdentifier:)` for the iCloud container
- The JSONL format is append-only — both TUI and mobile app append to the same file, iCloud Drive handles merge
- Session ID: use `UUID().uuidString` — format compatible with Go's UUID generation
- `SessionTracker` should be an `ObservableObject` so views can increment counters via published methods
- Config reading: parse YAML using a lightweight Swift YAML parser (Yams) or use a simple key-value reader for the subset needed
- Consider `NSUbiquitousKeyValueStore` as a simpler alternative for small config values
- Timestamps in ISO 8601 format, UTC — matching Go's `time.Now().UTC().Format(time.RFC3339)`
