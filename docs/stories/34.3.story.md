# Story 34.3: Story Template Update and Integration Notes

**Status:** Done (PR #228)
**Epic:** 34 — SOUL.md + Custom Development Skills
**Depends On:** 34.2 (Custom Slash Commands)
**Blocks:** None

## User Story

As a developer (human or AI agent),
I want the `/new-story` template verified as DRY-compliant and an integration notes document explaining how all AI tooling pieces work together,
So that new stories are lean by default and agents understand the layered information architecture.

## Acceptance Criteria

**Given** that CLAUDE.md now contains project-level coding standards, pre-PR checklist, and design patterns
**When** the `/new-story` slash command template is reviewed and integration notes are created
**Then:**

- AC1: The `/new-story` template (`new-story.md`) is verified to NOT embed pre-PR checklists, coding standards, or design pattern reminders already in CLAUDE.md — if any duplication exists, it is removed
- AC2: The template includes a "Quality Gate" section that references CLAUDE.md standards rather than duplicating them
- AC3: An integration notes document exists at `docs/architecture/ai-tooling-integration.md` explaining how SOUL.md, CLAUDE.md, and custom slash commands work together as a layered information model
- AC4: The integration notes document describes the DRY story approach — what belongs in stories vs. what's inherited from CLAUDE.md — with concrete guidance for story authors
- AC5: The integration notes reference all four custom slash commands created in Story 34.2 (`/pre-pr`, `/validate-adapter`, `/check-patterns`, `/new-story`) with their purpose and when to use each

## NOT In Scope

- Retroactive cleanup of existing story files (that's Story 34.4)
- Changes to CLAUDE.md or SOUL.md content
- New slash commands or modifications to existing ones beyond `/new-story`
- Go code changes
- Adding cross-references from CLAUDE.md to the integration notes doc

## Quality Gate

- All standard checks pass (see CLAUDE.md)
- Documentation is valid markdown with no broken links
- No duplication between the template and CLAUDE.md
- Scope-checked against this story — no extras

## Architecture & Design

This is a documentation-only story. Key design decisions:

1. **Layered information model:**
   - **SOUL.md** = philosophy and values (the *why*)
   - **CLAUDE.md** = technical standards and rules (the *how*)
   - **Slash commands** = workflow automation (the *do*)
   - **Story files** = task-specific context (the *what*)
2. **Template philosophy:** Story files should contain ONLY story-specific information. Everything generic lives in CLAUDE.md.
3. **Integration notes structure:**
   - Document purpose and audience
   - Layered information architecture
   - DRY story approach with what goes where
   - Slash command reference with usage guidance
   - Quick guide for story authors
