## Story

As a developer,
I want to validate whether harmonica spring-physics animations are feasible for door transitions,
So that we can decide whether to commit to a full animation system or reject it.

## Background

SOUL.md says "the UI should feel like physical objects — doors that open, selections that click into place." Current transitions are instantaneous state swaps. The harmonica library provides spring-physics animation that could deliver on this promise. However, frame-based animations require careful testing (time-dependent behavior) and performance validation. This spike implements a proof-of-concept on ONE transition to validate the approach before committing to a full system.

The primary deliverable is the **documented testing pattern** for frame-based animations in teatest, not the animation itself.

## Technical Notes

- `go get github.com/charmbracelet/harmonica`
- Implement spring animation on door selection → detail view transition only
- Use `harmonica.NewSpring()` with reasonable defaults (damping, stiffness)
- Animation driven by `tea.Tick` commands (frame-by-frame updates)
- Update() handles animation state; View() renders current frame
- Golden file testing strategy: snapshot initial state, final state, and optionally mid-animation frame
- Consider `harmonica.FPS(60)` for smooth animation at 60fps
- Performance: measure CPU during animation, ensure no visible lag
- If spike fails (testing too complex, performance issues), document why and recommend rejection
