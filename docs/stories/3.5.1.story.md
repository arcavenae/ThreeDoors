# Story 3.5.1: Core Domain Extraction

## Status: Done (PR #90)

**As a** developer,
**I want** `internal/tasks` split into `internal/core` (domain logic) and separate adapter packages,
**so that** the architecture follows the five-layer design specified in Architecture v2.0 and enables the Plugin SDK (Epic 7).

## Acceptance Criteria

1. **Core Domain Package**: `internal/core/` contains domain logic: Task, TaskPool, TaskStatus, DoorSelector, DoorSelection, StatusManager (task status transitions), SessionTracker, CompletionCounter, PatternAnalyzer, InsightsFormatter, MoodSelector, GreetingInsights, ValuesConfig, Onboarding, TagParser, TaskCategorization (TaskType, TaskEffort, TaskLocation), TaskImporter, ImprovementWriter, MetricsWriter, HealthChecker
2. **Provider Interface**: `internal/core/` exports the `TaskProvider` interface (and related types: Registry, AdapterFactory, ProviderConfig, ProviderEntry, FallbackProvider)
3. **TextFile Adapter**: `internal/adapters/textfile/` contains the YAML file adapter (extracted from FileManager + TextFileProvider)
4. **Apple Notes Adapter**: `internal/adapters/applenotes/` contains the Apple Notes adapter
5. **Obsidian Adapter**: `internal/adapters/obsidian/` contains the Obsidian adapter (including daily notes, watcher)
6. **Sync Infrastructure**: Sync-related code (SyncEngine, SyncState, SyncStatus, SyncManager, WriteQueue, WALProvider) stays in `internal/tasks/` for now OR moves to `internal/sync/` per architecture — this story focuses on core + adapters split
7. **Dependency Inversion**: `internal/tui/` depends only on `internal/core/`, not on adapter implementations
8. **All existing tests pass** without modification (behavior-preserving refactor)
9. **No user-facing behavior changes**

## NOT In Scope

* TaskProvider interface hardening (Watch, HealthCheck, ChangeEvent) — that's Story 3.5.2
* Config.yaml schema changes — that's Story 3.5.3
* Apple Notes adapter error handling improvements — that's Story 3.5.4
* New test suites — that's Story 3.5.5
* Moving sync infrastructure to `internal/sync/` — can be deferred

## Scope Reality Check

The original AC says "~2,100 LOC across 12 files" but `internal/tasks/` actually has:
- 25 source files (~14K LOC)
- 51 test files (~4K+ LOC)
- 76 total files (~18K LOC)
- 42 files across the codebase import `internal/tasks/`

This is a significant refactor. The approach must be incremental and behavior-preserving.

### Dev Notes

**Strategy: Package Aliasing Approach**

Rather than moving all files at once (which would break 42 importers), use a phased approach:

1. Create `internal/core/` package with the domain types
2. Create `internal/adapters/{textfile,applenotes,obsidian}/` packages
3. Move domain types to `internal/core/`, adapter types to adapter packages
4. Update all imports across the codebase (42 files)
5. Keep `internal/tasks/` as a thin re-export shim if needed during transition, or remove entirely
6. Register adapters in `cmd/threedoors/main.go` using new package paths

**Key constraint**: The `internal/adapters/` directory already exists with contract test files (`contract.go`, `contract_test.go`, `obsidian_contract_test.go`). These must be preserved/integrated.

**Five-Layer Target** (from Architecture v2.0):
1. TUI Layer (`internal/tui`) — Bubbletea views
2. Core Domain (`internal/core`) — Task management, door selection, progress tracking
3. Adapter Layer (`internal/adapters`) — Pluggable TaskProvider implementations
4. Sync Engine (`internal/sync`) — Offline-first queue, conflict resolution
5. Intelligence Layer (`internal/intelligence`) — LLM, calendar, learning
