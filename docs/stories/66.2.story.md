# Story 66.2: ClickUp Connect Wiring

**Epic:** 66 — CLI/TUI Adapter Wiring Parity
**Status:** Done (PR #TBD)
**Priority:** P1
**Estimate:** S (Small)

## User Story

**As a** ClickUp user,
**I want** to connect my ClickUp workspace via `threedoors connect clickup` or the TUI `:connect` wizard,
**so that** I can use ThreeDoors with my ClickUp tasks.

## Context

The ClickUp adapter (`internal/adapters/clickup/`) is fully implemented with tests and registered in `registerBuiltinAdapters()`, but it is not wired into the user-facing connect flows. Epic 63 (ClickUp Integration) is marked COMPLETE but the integration is inaccessible to users through either CLI or TUI connect paths.

**Sprint change proposal:** `_bmad-output/planning-artifacts/sprint-change-proposal-2026-03-13-unwired-features-gaps.md`
**Audit artifact:** `_bmad-output/planning-artifacts/unwired-features-audit.md` (Gap 3)

## Acceptance Criteria

1. `clickup` is added to CLI connect command's `ValidArgs` list (if not already present)
2. `clickup` is added to `knownProviderSpecs` in `internal/cli/connect.go` with correct flag spec (token key, required flags like workspace/list ID)
3. `clickup` is added to TUI connect wizard's `DefaultProviderSpecs()`
4. `threedoors connect clickup --label "My ClickUp" --token <token> --list-id <id>` works end-to-end
5. The TUI `:connect` wizard shows ClickUp as a selectable provider
6. `make test`, `make lint`, `make fmt` all pass

## Technical Notes

- Check `internal/adapters/clickup/config.go` for required settings keys (likely `api_token`, `list_id`, `workspace_id`)
- The flag spec needs to map CLI flags to the adapter's expected settings keys
- TUI `DefaultProviderSpecs()` lives in `internal/tui/connect_wizard.go`

## Dependencies

- Story 66.1 (CLI Adapter Registration Fix) should be merged first so CLI connect path has access to the registry

## Files Likely Modified

- `internal/cli/connect.go` — add `knownProviderSpecs["clickup"]`, update `ValidArgs` if needed
- `internal/tui/connect_wizard.go` — add ClickUp to `DefaultProviderSpecs()`
