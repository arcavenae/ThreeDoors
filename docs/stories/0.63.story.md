## Description

Only 9 user journeys exist in `docs/prd/user-journeys.md`, covering Phase 1-2 features. With 67+ completed epics, major user workflows have no journey representation. Party mode consensus: add 5 new journeys (10-14) covering first-run onboarding, task source connection, daily planning, snooze/defer, and CLI usage.

| # | Journey | Key Epics | Primary FRs |

## Acceptance Criteria

### AC1: Journey 10 — First-Run Onboarding
- [ ] Documents the complete first-run experience: launch → empty state → guided task addition → first door selection → theme picker → ready to use
- [ ] References FR31-FR34 (onboarding) and FR55-FR62 (themes)
- [ ] Follows existing journey format: User State, Flow (numbered steps), Supported By (FR refs), Success Criteria

### AC2: Journey 11 — Task Source Connection & Multi-Source Usage
- [ ] Documents: decide to connect external source → run setup wizard → configure credentials → verify sync → doors show mixed tasks from multiple sources → complete task → completion syncs back
- [ ] Explicitly mentions that doors show tasks from multiple sources after connection (architectural proof point)
- [ ] References FR63-FR80 (connection manager, sync lifecycle)

### AC3: Journey 12 — Daily Planning Mode
- [ ] Documents: morning ritual → review yesterday → energy-aware task selection → confirm today's focus → start working with curated doors
- [ ] References FR81-FR88 (daily planning mode)

### AC4: Journey 13 — Snooze & Defer Workflow
- [ ] Documents: encounter task → press Z to snooze → select duration → task disappears from pool → auto-returns on date → user completes
- [ ] References FR40-FR46 (snooze/defer)

### AC5: Journey 14 — CLI Task Management
- [ ] Documents power-user CLI workflow: `threedoors task list` → `threedoors task add` → `threedoors door` → `threedoors task complete` → `threedoors stats`
- [ ] References FR47-FR51 (CLI interface)

### AC6: Format consistency
- [ ] All 5 new journeys follow the exact format of existing journeys (User State, Flow, Supported By, Success Criteria)
- [ ] Each journey has measurable success criteria
- [ ] Journeys numbered sequentially 10-14 after existing Journey 9
- [ ] Section separators (`---`) between journeys
