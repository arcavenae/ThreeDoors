## Status: done

**As a** developer,
**I want** functional end-to-end tests covering full user workflows,
**so that** the complete user experience is validated automatically.

## Acceptance Criteria

1. [x] E2E tests exercise the complete workflow: launch → view doors → select door → manage task → exit
2. [x] Tests verify session metrics are correctly generated
3. [x] Tests cover: search, command palette, mood tracking workflows
4. [x] Uses Bubbletea's `teatest` package for TUI testing
5. [x] Tests run in CI without requiring a real terminal
6. [x] Tests use stdlib `testing` package only (no testify); table-driven for >2 cases; `t.Helper()` in helpers; `t.Cleanup()` for resources
