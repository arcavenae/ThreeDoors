## Story

As a contributor,
I want the Docker E2E job to run only on push-to-main (not on PRs),
So that PR CI cost is reduced without losing defense-in-depth on the main branch.

## Background

The Docker E2E job (`test-docker-e2e`) runs `go test ./...` inside a Docker container on every PR. This duplicates the Quality Gate job, which already runs the full test suite with race detection and coverage. The Docker E2E adds cross-platform validation (CGO_ENABLED=0 in a Linux container), but since Quality Gate also runs on `ubuntu-latest`, the incremental value is negligible for a TUI app.

Additionally, `Dockerfile.test` pins `golangci-lint` at `v2.1.6` while CI uses `v2.10.1` — a version skew that could cause inconsistent lint results between Docker and native CI runs.

**Party mode consensus:** Option A — move Docker E2E to push-only. Fixes lint version skew as a correctness issue.

## Acceptance Criteria

**AC-1: Docker E2E skipped on PRs**
**Given** a pull request is opened or updated
**When** CI runs
**Then** the `test-docker-e2e` job does NOT execute

**AC-2: Docker E2E runs on push-to-main**
**Given** a commit is pushed to the `main` branch
**When** CI runs
**Then** the `test-docker-e2e` job executes normally (same behavior as today)

**AC-3: golangci-lint version aligned**
**Given** the `Dockerfile.test` file
**When** reviewed
**Then** `GOLANGCI_LINT_VERSION` is set to `v2.10.1` (matching `.github/workflows/ci.yml`)

**AC-4: Required checks unaffected**
**Given** a PR with code changes
**When** CI completes
**Then** the `Quality Gate` and `Benchmarks` required checks still function correctly
**And** no new required check is added or removed

## Technical Notes

**File: `.github/workflows/ci.yml`**

Change the `test-docker-e2e` job's `if` condition from:
```yaml
if: needs.changes.outputs.code == 'true' || needs.changes.outputs.docker == 'true' || github.event_name == 'push'
```
to:
```yaml
if: github.event_name == 'push'
```

This makes Docker E2E run only on push-to-main. It no longer needs the `changes` job dependency since it always runs on push.

**File: `Dockerfile.test`**

Change line 9:
```dockerfile
ARG GOLANGCI_LINT_VERSION=v2.1.6
```
to:
```dockerfile
ARG GOLANGCI_LINT_VERSION=v2.10.1
```

**Impact:** Reduces CI runner minutes per PR by ~2.5 minutes. No PR wall-clock change (jobs run in parallel and Docker E2E is not the bottleneck). Eliminates a source of flaky failures that can block PRs.
