## Story

As a multiclaude user,
I want to manually trigger a supervisor handover and be notified when automatic handovers occur,
So that I can intervene when I notice degradation the shift clock missed, and stay informed about system state changes.

## Background

The shift clock (Story 58.1) handles automatic detection, but users may notice degradation before metrics trigger — the "false negative" scenario (Failure A1). A manual trigger provides an escape valve. Additionally, handovers are significant system events that users should be aware of.

**Open questions from synthesis (resolved here):**
- Manual trigger: Yes — `multiclaude supervisor handover` command
- User notification: Yes — brief notification via multiclaude message and optional tmux status bar indicator

## Acceptance Criteria

**AC-1: Manual handover command**
**Given** a supervisor is running
**When** the user runs `multiclaude supervisor handover`
**Then** the daemon writes a handover signal file (same mechanism as Story 58.1)
**And** the handover proceeds via the orchestrator (Story 58.3)

**AC-2: Manual handover respects anti-oscillation**
**Given** a handover completed less than 30 minutes ago
**When** the user runs `multiclaude supervisor handover`
**Then** the command warns the user but proceeds if they confirm (manual override of anti-oscillation)

**AC-3: Force option**
**Given** the user wants to skip the anti-oscillation check
**When** they run `multiclaude supervisor handover --force`
**Then** the handover proceeds immediately regardless of last handover time

**AC-4: Automatic handover notification**
**Given** an automatic handover is triggered by the shift clock
**When** the handover begins
**Then** the daemon sends a notification: `multiclaude message send <user-channel> "SUPERVISOR_HANDOVER: Automatic shift change initiated. Metrics: [JSONL size, compression count]. New supervisor starting."`

**AC-5: Handover completion notification**
**Given** a handover completes (automatic or manual)
**When** the incoming supervisor confirms ready
**Then** a completion notification is sent: "SUPERVISOR_HANDOVER_COMPLETE: New supervisor [name] is online. Handover took [N] seconds. [Normal/Emergency] mode."

**AC-6: Status command integration**
**Given** the user runs `multiclaude status`
**When** the shift clock has data
**Then** the status output includes current supervisor session metrics: zone (Green/Yellow/Red), JSONL size, compression count, session duration

## Technical Notes

- `multiclaude supervisor handover` is a new subcommand — it simply writes the handover signal file
- The `--force` flag bypasses anti-oscillation by writing a special flag in the signal file
- User notifications via `multiclaude message send` to a user-visible channel or log file
- Status integration extends `multiclaude status` to include shift clock metrics
- For tmux status bar: optional, could be deferred. A simple `#[fg=yellow]HANDOVER` indicator during transitions would be useful but not MVP-critical.
