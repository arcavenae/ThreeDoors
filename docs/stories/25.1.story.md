# Story 25.1: Todoist HTTP Client & Auth Configuration

## Status: Done (PR #308)

**Epic:** 25 — Todoist Integration
**Blocks:** Stories 25.2, 25.3, 25.4

## User Story

As a developer,
I want a thin HTTP client for the Todoist REST API v1,
So that the TodoistProvider can read and complete tasks without a third-party SDK dependency.

## Acceptance Criteria

**Given** the need for Todoist API access
**When** the HTTP client is implemented
**Then:**

- AC1: `Client` struct in `internal/adapters/todoist/todoist_client.go` with `NewClient(config AuthConfig) *Client`
- AC2: Bearer token authentication via personal API token
- AC3: `GetTasks(ctx, projectID, filter) ([]TodoistTask, error)` method using `GET /rest/v1/tasks`
- AC4: `CloseTask(ctx, taskID) error` method using `POST /rest/v1/tasks/{id}/close`
- AC5: `GetProjects(ctx) ([]TodoistProject, error)` method for health check and config validation
- AC6: HTTP 429 handling: parse `Retry-After` header, return `*RateLimitError` (reuse pattern from Jira adapter)
- AC7: `AuthConfig` struct with `APIToken string` field, sourced from config.yaml settings or `TODOIST_API_TOKEN` env var
- AC8: `TodoistConfig` struct with `APIToken`, `ProjectIDs`, `Filter`, `PollInterval` fields, with validation that `ProjectIDs` and `Filter` are mutually exclusive
- AC9: Unit tests using `httptest.NewServer` with canned Todoist API responses
- AC10: No third-party dependencies beyond stdlib

## Technical Notes

- Use `net/http` + `encoding/json` — no third-party Todoist SDK (go-todoist targets deprecated v2 API)
- Auth: `Authorization: Bearer <api_token>` on all requests
- Rate limits: 450 requests per 15 minutes — respect `Retry-After` header on 429 responses
- Todoist REST API v1 base URL: `https://api.todoist.com/rest/v1`
- Response filtering: `project_id` query parameter for project-scoped queries, or `filter` query parameter for Todoist filter expressions
- Accept interfaces, return concrete types: `NewClient(config AuthConfig) *Client`
- Context as first parameter on all methods
- Reuse `RateLimitError` pattern from `internal/adapters/jira/`
