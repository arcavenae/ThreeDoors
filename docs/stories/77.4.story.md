## Story

As a project maintainer,
I want changes to `scripts/hooks/**` to trigger the full Quality Gate,
So that modifications to safety-critical git hooks (which affect CI behavior and agent safety guardrails) are validated by the full Go test suite.

## Background

The `scripts/hooks/git-safety.sh` hook is mechanically enforced via `.claude/settings.json` and blocks dangerous git commands (fetch, rebase, merge, unsigned commits, push to main). Changes to this hook affect the integrity of the entire multiclaude agent safety system. Currently, a PR that only modifies hooks would skip Quality Gate — this is a gap.

The hooks directory currently contains `git-safety.sh` and `git-safety_test.sh`. Future hooks may be added here as well.

## Acceptance Criteria

**AC1: Hooks path added to code filter**
**Given** the `ci.yml` path filter configuration
**When** `scripts/hooks/**` is added to the `code` filter
**Then** any PR modifying files in `scripts/hooks/` triggers the Quality Gate

**AC2: Existing code filter paths unchanged**
**Given** the updated code filter
**When** compared to the previous configuration
**Then** all existing paths (`**.go`, `go.mod`, `go.sum`, `.golangci.yml`, `.github/workflows/**`) remain in the filter
**And** only `scripts/hooks/**` is added

**AC3: Hooks-only PR triggers Quality Gate**
**Given** a PR that modifies only `scripts/hooks/git-safety.sh`
**When** the path filter runs
**Then** the `code` output is `true`
**And** the Quality Gate executes

## Tasks

### Task 1: Add `scripts/hooks/**` to the code filter in `ci.yml`
Add `- 'scripts/hooks/**'` to the `code:` filter section.
