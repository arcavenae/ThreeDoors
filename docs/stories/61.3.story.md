## Story

As a user who wants to connect ThreeDoors to my existing task sources,
I want dedicated per-integration setup guides on the docs site,
So that I can configure each provider with clear, consistent instructions.

## Background

ThreeDoors supports 7 task source integrations plus local YAML files. The monolithic user guide covers all of them in one long file. This story creates dedicated pages per integration with a consistent structure (Overview → Prerequisites → Setup → Configuration → Usage → Troubleshooting), making it easy for users to find exactly the guide they need.

## Acceptance Criteria

**Given** the docs site infrastructure from Story 61.1
**When** provider pages are created
**Then** `docs-site/docs/providers/overview.md` explains multi-source architecture, connection manager, how providers work, and mixing sources
**And** `docs-site/docs/providers/local-files.md` covers YAML task file format, file location, and examples
**And** `docs-site/docs/providers/apple-notes.md` covers prerequisites, setup, sync behavior, limitations, and troubleshooting
**And** `docs-site/docs/providers/apple-reminders.md` covers prerequisites, setup, sync behavior, and limitations
**And** `docs-site/docs/providers/jira.md` covers OAuth setup, field mapping, JQL filtering, and config examples
**And** `docs-site/docs/providers/github-issues.md` covers token setup, repo filtering, and label mapping
**And** `docs-site/docs/providers/todoist.md` covers API key setup, project mapping, and sync behavior
**And** `docs-site/docs/providers/obsidian.md` covers vault path configuration, task format, and frontmatter mapping

**Given** each provider page
**When** examined for structure
**Then** it follows a consistent format: Overview → Prerequisites → Setup → Configuration → Usage → Troubleshooting

**Given** platform-specific integrations (Apple Notes, Apple Reminders)
**When** the page is viewed
**Then** admonitions clearly note macOS-only requirements

**Given** the mkdocs.yml navigation
**When** updated
**Then** all 8 provider pages appear under the "Task Sources" section

**Given** the complete site
**When** `mkdocs build --strict` is run
**Then** there are no broken internal links and no warnings
