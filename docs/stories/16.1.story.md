# Story 16.1: SwiftUI Project Setup & CI

**Status:** Draft
**Epic:** 16 — iPhone Mobile App (SwiftUI)
**Blocks:** Stories 16.2, 16.3, 16.4, 16.5, 16.6, 16.7

## User Story

As a developer,
I want a working SwiftUI project with CI pipeline,
So that I have a foundation for building the Three Doors mobile app.

## Acceptance Criteria

**Given** the need for an iOS project foundation
**When** the project is set up
**Then:**

- AC1: Xcode project created at `mobile/ThreeDoors/` with SwiftUI lifecycle (`@main` App struct)
- AC2: Target: iOS 17+, iPhone only (iPad layout deferred to Phase 3)
- AC3: Basic app shell renders "ThreeDoors" header with app icon placeholder
- AC4: GitHub Actions CI workflow (`.github/workflows/ios.yml`) for building and testing the Swift project using `xcodebuild`
- AC5: `.gitignore` configured for Xcode project artifacts (`*.xcuserstate`, `DerivedData/`, `build/`, etc.)
- AC6: SwiftUI previews configured for development with sample data
- AC7: App compiles and runs in iOS Simulator without errors

## Quality Gate

- QG1: `xcodebuild build` succeeds with zero warnings (treat warnings as errors)
- QG2: `xcodebuild test` passes (even if only default tests exist)
- QG3: SwiftUI previews render without errors
- QG4: CI workflow runs successfully on push
- QG5: Scope-checked against this story — no extras

## Technical Notes

- Use Swift 5.9+ with strict concurrency checking enabled
- SwiftUI lifecycle (not UIKit AppDelegate) — use `@main` struct conforming to `App`
- Xcode 16+ project format
- Organize project with Swift Package-style folder structure: `Sources/`, `Tests/`
- Bundle identifier suggestion: `com.threedoors.mobile`
- CI can use `macos-latest` runner with Xcode pre-installed
- No third-party dependencies in this story — pure SwiftUI and Apple frameworks

## Dependencies

- None — this is the foundation story for Epic 16
- Research context: `docs/research/mobile-app-research.md`

## Tasks

1. Create `mobile/ThreeDoors/` Xcode project with SwiftUI lifecycle
2. Configure iOS 17+ deployment target, iPhone-only device family
3. Build basic app shell with "ThreeDoors" header view
4. Set up SwiftUI previews with sample data
5. Add `.gitignore` for Xcode artifacts
6. Create GitHub Actions CI workflow for iOS build and test
7. Verify app compiles and runs in iOS Simulator
