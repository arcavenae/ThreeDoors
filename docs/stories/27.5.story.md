# Story 27.5: CLI `plan` Subcommand & Integration Polish

## Story Info

- **Epic:** 27 — Daily Planning Mode
- **Priority:** P1
- **Status:** Done (PR #360)
- **Estimated Effort:** Small
- **Dependencies:** 27.2 (review flow), 27.3 (select flow), 27.4 (confirm view + orchestration)

## User Story

As a ThreeDoors user, I want to start a planning session from the CLI with `threedoors plan` or from within the TUI with `:plan` so that I can access daily planning from wherever I am.

## Acceptance Criteria

### AC 27.5.1: CLI Subcommand
- [x] `threedoors plan` launches the TUI directly into PlanningView (bypasses doors view)
- [x] Follows existing Cobra subcommand patterns from Epic 23
- [x] Exits cleanly after planning session completes (returns to shell, not doors view)
- [x] `threedoors plan --help` shows usage description

### AC 27.5.2: TUI Command
- [x] `:plan` command in the TUI command palette triggers PlanningView
- [x] After planning session completes, returns to doors view (not exit)
- [x] PlanningView accessible from any TUI view state (doors, detail, search)

### AC 27.5.3: Integration with Doors View
- [x] After planning session, doors view refreshes and shows focus-boosted doors
- [x] Focus tasks appear with a visual indicator in doors view (e.g., small "focus" badge or color)
- [x] If no planning session has been run today, doors view works normally (no focus boost active)

### AC 27.5.4: Planning State Persistence
- [x] Planning session timestamp persisted (for focus expiry calculation)
- [x] Focus tags survive TUI process restarts (tags are in task YAML)
- [x] On next planning session, previous `+focus` tags are cleared before new selection

### AC 27.5.5: First-Time Planning Guidance
- [x] On first-ever planning session, show a brief one-time explanation: "Welcome to Daily Planning! This 3-step flow helps you focus your day." (dismissible with any key)
- [x] Subsequent sessions skip the explanation

## Technical Notes

- CLI subcommand follows Cobra patterns established in Epic 23 (`internal/cli/`)
- TUI command registration follows existing `:command` pattern in command palette
- Focus badge in doors view: small Lipgloss-styled indicator, minimal visual weight
- Planning timestamp can be stored in session state file or derived from latest `planning_session` event in JSONL
