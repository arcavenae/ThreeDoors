# Story 16.6: Swipe Gestures & Pull-to-Refresh

**Status:** Draft
**Epic:** 16 — iPhone Mobile App (SwiftUI)
**Depends on:** Stories 16.3, 16.4

## User Story

As a mobile user,
I want intuitive gestures for common actions,
So that the app feels native and fast to use.

## Acceptance Criteria

**Given** the Three Doors carousel and detail views are functional
**When** gesture interactions are added
**Then:**

- AC1: **Pull-to-refresh**: Pull down on carousel triggers Apple Notes re-read and generates a new set of three doors
- AC2: **Swipe right on card**: Quick-complete gesture — marks task as complete with confirmation haptic (`.success`)
- AC3: **Swipe left on card**: Defer/skip gesture — marks task as "not now" and removes from current set
- AC4: **Long press on card**: Opens context menu with all status options (Complete, Blocked, In Progress, In Review)
- AC5: Gesture animations smooth and responsive (60 FPS, no dropped frames)
- AC6: Gesture hints shown on first use — onboarding overlay explaining swipe left/right/pull-down (dismissible, shown once via `@AppStorage`)
- AC7: Undo option shown briefly after swipe-to-complete — 5-second window with "Undo" button, auto-dismisses
- AC8: Pull-to-refresh triggers Apple Notes re-read via `TaskProvider.loadTasks()`

## Technical Notes

- Use SwiftUI's `.refreshable` modifier for pull-to-refresh — native iOS feel with minimal code
- Swipe gestures on cards: use `.swipeActions` modifier (iOS 15+) or custom `DragGesture` with threshold detection
- Long press context menu: use `.contextMenu` modifier for native iOS context menu behavior
- Undo implementation: keep previous state in memory, present `SnackBar`-style overlay at bottom with "Undo" button, auto-dismiss with `Task.sleep` or `Timer`
- First-use onboarding overlay: store "has seen gesture hints" flag in `@AppStorage("hasSeenGestureHints")`
- Haptic feedback: `.success` notification for complete, `.medium` impact for defer, `.light` impact for card selection
- Be careful with gesture conflicts — swipe-to-complete vs carousel swipe need distinct gesture areas or thresholds
- Consider vertical swipe threshold (>100pt) for complete/defer to avoid accidental triggers during horizontal carousel swiping
