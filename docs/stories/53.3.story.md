## Story

As a remote Claude Code session,
I want MCP tools that expose multiclaude system status and messages,
So that I can query the supervisor system programmatically without running raw SSH commands.

## Background

MCP (Model Context Protocol) is the standard extensibility mechanism for Claude Code. Anthropic officially supports remote MCP servers via SSH stdio transport. By building a small MCP server that wraps multiclaude CLI commands as tools, remote Claude Code sessions can interact with the multiclaude system natively — tools appear in the agent's tool list with structured request/response semantics.

This story implements the **read-only** tools first, establishing the MCP server framework. Write tools (message sending, task dispatch) follow in Story 53.4.

The MCP-over-SSH pattern is a native Claude Code capability: `"command": "ssh", "args": ["user@host", "/path/to/mcp-server"]`. No custom transport code needed.

## Acceptance Criteria

**Given** the MCP bridge server binary `multiclaude-mcp-bridge`
**When** launched via stdio (stdin/stdout JSON-RPC)
**Then** it responds to MCP `initialize` and `tools/list` requests correctly

**Given** the MCP tool `multiclaude_status`
**When** invoked by a remote Claude Code session
**Then** it returns the current multiclaude system status (agents, their states, worker count) as structured JSON

**Given** the MCP tool `multiclaude_worker_list`
**When** invoked by a remote Claude Code session
**Then** it returns the list of active workers with their names, tasks, and creation times

**Given** the MCP tool `multiclaude_message_list`
**When** invoked by a remote Claude Code session
**Then** it returns pending messages for the requesting user (or all messages if no filter specified)

**Given** the MCP tool `multiclaude_message_read`
**When** invoked with a message ID parameter
**Then** it returns the full message content for that ID

**Given** the MCP tool `multiclaude_repo_history`
**When** invoked with optional `--count` and `--status` parameters
**Then** it returns recent task history entries

**Given** the MCP bridge server
**When** configured in a remote Claude Code session's `.claude/settings.json` as an SSH MCP server
**Then** all tools are discoverable and callable from the remote session

**Given** any MCP tool invocation
**When** the underlying `multiclaude` command fails
**Then** the tool returns a structured MCP error with the command's stderr output

## Technical Notes

- Language: Go. Binary name: `multiclaude-mcp-bridge`. Lives at `cmd/multiclaude-mcp-bridge/`.
- Uses stdio transport only (JSON-RPC over stdin/stdout). HTTP transport is Story 53.5.
- Each MCP tool shells out to the corresponding `multiclaude` CLI command and parses the output.
- MCP SDK: Use `github.com/mark3labs/mcp-go` or implement the minimal JSON-RPC protocol directly (the protocol is simple for stdio).
- Tools in this story are **read-only** — they execute `multiclaude status`, `multiclaude worker list`, `multiclaude message list`, `multiclaude message read`, `multiclaude repo history`.
- The binary must be installable on the multiclaude host machine (e.g., `go install` or `make build-mcp-bridge`).
- No authentication needed for stdio transport (SSH provides auth).
- All tool outputs should be structured JSON, not raw CLI text output. Parse multiclaude CLI output or use `--json` flags where available.
