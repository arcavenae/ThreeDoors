# Story 17.8: Redesign Shoji Theme Visual Layout

## Status: Done (PR #186)

## User Story

As a user,
I want the Shoji theme to evoke the elegance of a Japanese screen with large paper panes and thin wooden frames,
So that my task text is prominent and the decoration enhances rather than competes.

## Acceptance Criteria

**Given** the Shoji theme is selected
**When** three doors are rendered
**Then:**

- AC1: The grid uses fewer, larger panes вЂ” no more than 3-4 columns of lattice (down from the current ~8+ small cells)
- AC2: Task text occupies the majority of the door's visual area (content-to-decoration ratio visibly favors content)
- AC3: Lattice lines are thin (`в”Ђв”‚в”ј`) вЂ” the frame feel is "wooden strips between paper", not "graph paper"
- AC4: Selected state uses heavy characters (`в”Ѓв”ѓв•‹`) and brighter color, providing clear visual distinction
- AC5: Minimum 15 characters of usable text width at `MinWidth`
- AC6: Word wrapping continues to function correctly for long task text
- AC7: Golden file tests updated with new layout
- AC8: Visual width consistency (from Story 17.7) holds for new layout

Key differences from current implementation:
- Side frame is a single `в”‚` (not `в”‚  в”‚` with cell subdivisions)
- Horizontal bars at 3-4 fixed positions (top, content-top, mid-cross, content-bottom, bottom)

## Technical Notes

- Current grid in `shoji.go` uses `cellW = 3` with `numCols = (width - 1) / cellW`, creating many small cells
- New approach: single-column content area with thin `в”‚` side frame and 3-4 horizontal `в”Ђ` lattice bars at fixed row positions
- The content pane should be a single large area, not subdivided into cells
- Preserve the `shojiChars` struct and selected/unselected character sets
- The lattice feel comes from horizontal bars and a single mid-cross junction `в”ј`, not from many vertical cell dividers
- Content-to-decoration ratio: at 28-char width, content area should be 24 chars wide (width minus 2 frame chars minus 2 padding chars)
