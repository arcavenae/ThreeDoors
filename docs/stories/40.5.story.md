## Story

As a user,
I want to see an 8-week activity heatmap showing my daily completion patterns in a grid,
So that I can spot weekly rhythms and consistency at a glance without reading tables.

## Background

GitHub's contribution graph is one of the most recognizable data visualizations in software. This story adapts that pattern for ThreeDoors: a 7-row (days of week) by 8-column (weeks) grid where each cell's intensity represents the number of tasks completed that day. The grid uses Unicode shade characters (space, ░, ▒, ▓, █) combined with color intensity for dual-encoding (accessible to color-blind users).

The heatmap lives on the Detail tab (accessed via Tab from the Overview) to keep the main view clean and within 24 rows.

## Acceptance Criteria

**Given** at least 2 weeks of session data exist
**When** the Detail tab renders
**Then** a 7x8 grid displays with rows labeled Mo-Su and columns labeled W1-W8
**And** cell intensity (character + color) reflects daily completion count
**And** a legend shows the mapping: space=0, ░=1-2, ▒=3-4, ▓=5-6, █=7+

**Given** a day with zero completions
**When** the heatmap renders
**Then** that cell renders as a space character with no color

**Given** a day with the maximum completions in the range
**When** the heatmap renders
**Then** that cell renders as █ in the brightest gradient color

**Given** fewer than 8 weeks of data exist
**When** the heatmap renders
**Then** only available weeks are shown (right-aligned, most recent on right)
**And** missing weeks are not rendered

**Given** the terminal is narrower than 60 columns
**When** the heatmap renders
**Then** the heatmap is hidden with a message: "Widen terminal to see activity heatmap"

**Given** the custom rendering approach has significant issues during implementation
**Then** evaluate ntcharts heatmap component as alternative (per D-090 spike subtask)

## Technical Notes

- New render function: `renderHeatmap()` in `insights_view.go`
- Data: `iv.analyzer.GetDailyCompletions(56)` returns 56 days of data
- Aggregate into 7x8 grid: `[dayOfWeek][weekIndex]int`
- Normalize to 5 levels: 0, 1-2, 3-4, 5-6, 7+
- Unicode characters: `' '`, `'░'`, `'▒'`, `'▓'`, `'█'`
- Color gradient per cell: `Blend1D(dimColor, brightColor, level/4.0)`
- Grid alignment: each cell is 5 characters wide (`fmt.Fprintf(s, "%-5s", cell)`)
- Include a spike subtask: prototype custom approach first, evaluate ntcharts only if issues arise
