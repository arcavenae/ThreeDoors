## Story

As a user,
I want to see a brief celebration banner when I cross a milestone threshold,
So that ThreeDoors notices what I've accomplished and marks the moment.

## Background

SOUL.md says the app should feel like "a friend saying hey, here are three things you could do." A friend would also notice when you've done something noteworthy. Milestone celebrations are that noticing moment.

CRITICAL SOUL.md BOUNDARY: Milestones are observations, not achievements. No "unlocked" language, no badges, no levels, no "next milestone" messaging. The banner appears once, celebrates what happened, and never returns. Four milestones maximum:
1. First session: "Welcome! Your journey starts here."
2. 50 tasks completed: "50 tasks done — half a century of getting things done!"
3. 100 tasks completed: "Triple digits! 100 tasks completed."
4. 10-day streak: "10 days in a row — double digits!"

## Acceptance Criteria

**Given** the user has just completed their first session
**When** they open the insights view for the first time
**Then** a styled banner displays: "Welcome! Your journey starts here."
**And** the banner auto-dismisses after 5 seconds or on any keypress

**Given** the user has completed 50 tasks total
**When** they open the insights view after crossing the threshold
**Then** a celebration banner appears once
**And** the milestone is recorded in `~/.threedoors/milestones.json`
**And** the banner never appears again for this milestone

**Given** a milestone has already been shown (recorded in milestones.json)
**When** the insights view renders
**Then** no banner appears for that milestone

**Given** milestones.json does not exist (first run)
**When** the first milestone is triggered
**Then** the file is created with atomic write (write .tmp, sync, rename)

**Given** the user presses any key while the banner is displayed
**When** the key is processed
**Then** the banner dismisses immediately
**And** the key event is NOT passed to the underlying view

## Technical Notes

- Milestone persistence: `~/.threedoors/milestones.json`
  ```json
  {"shown": ["first-session", "50-tasks"]}
  ```
- Atomic file writes per CLAUDE.md: write to `.tmp`, fsync, rename
- New type: `MilestoneChecker` in `internal/core/milestones.go`
- `CheckMilestones(totalTasks int, currentStreak int, sessionCount int) *Milestone`
- `MarkShown(id string) error` — persists to milestones.json
- Banner rendering: Lipgloss styled box at top of insights view with gold background
- Auto-dismiss: `tea.Tick(5*time.Second)` sends `MilestoneDismissMsg`
- Render only one milestone per view entry (highest unshown priority)
- Do NOT show "next milestone" or progress toward next threshold
