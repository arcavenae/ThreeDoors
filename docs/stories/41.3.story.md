## Story

As a user,
I want the help view to use standard scrolling with mouse wheel support,
So that navigating help content feels consistent and intuitive.

## Background

The help view (`help_view.go`) has a custom page-based scrolling implementation. This is one of three custom scroll implementations in the codebase (along with synclog_view and keybinding_overlay). The `bubbles/viewport` component provides battle-tested scrolling with mouse wheel support, keyboard navigation, and proper ANSI handling. Help view is chosen as the first migration target because it has the most content and benefits most from continuous scrolling.

**Sequencing note:** This story should be sequenced after Epic 39 stories that modify the help view or keybinding overlay to avoid merge conflicts.

## Acceptance Criteria

**Given** the help view is opened
**When** content exceeds the visible area
**Then** a bubbles/viewport is used for scrollable content display

**Given** the help view is displayed
**When** the user scrolls with the mouse wheel
**Then** the content scrolls smoothly up or down

**Given** the help view is displayed
**When** the user presses j/k or up/down arrow keys
**Then** the content scrolls line by line (keyboard navigation preserved)

**Given** the help view is displayed
**When** the user presses Page Up or Page Down
**Then** the viewport scrolls by a full page

**Given** golden file tests for help view
**When** tests run after the migration
**Then** all golden file tests pass (updated for viewport rendering differences)

**Given** the help view content renders
**When** compared to the previous implementation
**Then** there is no regression in help content rendering (same information displayed)

## Technical Notes

- Add `github.com/charmbracelet/bubbles/viewport` dependency
- Replace custom scrolling state (offset, pageSize) with `viewport.Model`
- Help view's `Update()` delegates scroll-related keys to `viewport.Update()`
- Help view's `View()` renders content into `viewport.View()`
- Preserve existing help content generation — only change the scrolling mechanism
- Remove custom pagination logic (viewport subsumes it)
- Mouse wheel support comes free from viewport
- Test with `teatest` — viewport provides its own test helpers
