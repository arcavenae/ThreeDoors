## Story

As a ThreeDoors user,
I want to run `threedoors history` from the command line,
So that I can see my completed tasks without entering the TUI, and pipe the output to other tools.

## Background

The CLI is ThreeDoors' non-interactive interface for scripting, CI, and quick lookups. A `history` subcommand makes completion data accessible outside the TUI, with `--json` output for programmatic consumption.

## Acceptance Criteria

- [ ] `threedoors history` shows today's completions by default (human-readable)
- [ ] `--today` flag: show today's completions (explicit, same as default)
- [ ] `--week` flag: show this week's completions
- [ ] `--month` flag: show this month's completions
- [ ] `--all` flag: show all completions
- [ ] `--json` flag: output as JSON array of CompletionRecord objects
- [ ] Human-readable output format:
  ```
  Today — March 15
    Fix login bug                    14:32
    Update README                    11:15

  Yesterday — March 14
    Review PR #42                    16:45
  ```
- [ ] JSON output format:
  ```json
  [
    {"title": "Fix login bug", "completed_at": "2026-03-15T14:32:00Z", "source": "", "task_id": "abc123"},
    ...
  ]
  ```
- [ ] Empty result prints friendly message (not an error): "No completed tasks found."
- [ ] Exit code 0 on success, 1 on error
- [ ] `threedoors history --help` shows usage
- [ ] Cobra command registered under root command
- [ ] Tests covering: default (today), each flag, --json output, empty result, combined flags (--week --json)
- [ ] `make lint` passes with zero warnings
