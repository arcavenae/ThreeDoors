## Story

As a ThreeDoors developer,
I want the remaining auxiliary views and command dispatch logic extracted from main_model.go,
So that the main model file is reduced to ~800-1000 lines containing only the struct definition, Init(), and top-level Update/View dispatch.

## Background

After Stories 69.1-69.3 extract navigation, source, and task views, the remaining views are auxiliary: Help, BugReport, ThemePicker, Health, Insights, Mood, Feedback, ValuesGoals, Orphaned, Conflict, Proposals, DevQueue. Additionally, the command palette dispatch (`:help`, `:bug`, `:sources`, `:history`, etc.) can be extracted into its own file.

This is the final decomposition story. After this, `main_model.go` should contain only:
- The `MainModel` struct definition
- `NewMainModel()` constructor
- `Init()` method
- Top-level `Update()` that delegates to controller methods
- Top-level `View()` that delegates to controller methods
