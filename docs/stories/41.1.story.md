## Story

As a user,
I want to see a spinner when provider sync operations are in flight,
So that I know the app is working and not frozen.

## Background

When Todoist syncs, Apple Notes fetches, or other provider operations run, users see no visual feedback. This creates uncertainty — "is it frozen?" vs "it's thinking." The bubbles/spinner component is trivial to integrate and provides deterministic frame rendering for tests.

## Acceptance Criteria

**Given** a provider sync operation begins (Todoist, Apple Notes, etc.)
**When** the operation takes longer than 100ms
**Then** a spinner is visible in the sync status area

**Given** a provider sync operation completes successfully
**When** the result is received
**Then** the spinner stops and shows a success indicator

**Given** a provider sync operation fails
**When** the error is received
**Then** the spinner stops and shows an error indicator

**Given** a provider operation completes in under 100ms
**When** the UI renders
**Then** no spinner is shown (avoid flash for instant operations)

**Given** golden file tests run
**When** the spinner is active
**Then** a deterministic spinner frame is captured using spinner.Tick control

## Technical Notes

- Add `github.com/charmbracelet/bubbles/spinner` dependency
- Embed `spinner.Model` in views that trigger async provider operations
- Primary integration point: sync status rendering in `internal/tui/`
- Use `spinner.Dot` or `spinner.MiniDot` style for minimal visual weight
- Deterministic testing: freeze spinner frame for golden file snapshots
- No spinner for instant operations (<100ms) — use a `tea.Tick` delay before showing
