## Story

As a user,
I want to press space or enter again to close an open door (return from DetailView to DoorsView),
So that opening and closing a door uses the same natural gesture, like a real door.

## Background

Currently, space/enter opens a selected door (navigates to DetailView), but to go back the user must press Escape — a different key. This breaks the physical door metaphor: real doors open and close with the same gesture. Story 36.2 already established the toggle pattern for door selection keys (a/w/d), where pressing the same key again deselects. This story extends that proven toggle pattern to the open/close action.

The `isTextInputActive()` guard from Story 36.3 ensures that space/enter in text input sub-views (feedback form, snooze date picker, search) are not intercepted.

## Acceptance Criteria

**AC-1: Toggle closes door in passive DetailView**
**Given** the user is in DetailView (viewing a task's details)
**And** no text input is active (feedback form, snooze picker, etc.)
**When** the user presses space or enter
**Then** the view switches back to DoorsView
**And** the previously selected door remains highlighted (selectedDoorIndex preserved)

**AC-2: Text input not intercepted**
**Given** the user is in DetailView with a text input active (feedback form via 'N', snooze via 'Z')
**When** the user presses space or enter
**Then** the keypress is handled by the text input (not intercepted as toggle)
**And** the view does NOT switch back to DoorsView

**AC-3: Escape still works**
**Given** the user is in DetailView
**When** the user presses Escape
**Then** the view switches back to DoorsView (existing behavior preserved)

**AC-4: Door selection preserved on toggle-back**
**Given** door 1 (w) was selected when the user opened it
**When** the user presses space/enter to close (toggle back)
**Then** door 1 is still highlighted in DoorsView
**And** the visual state matches what the user saw before opening

**AC-5: Rapid toggle handled gracefully**
**Given** the user is in DoorsView with a door selected
**When** the user double-taps space quickly (open then immediately close)
**Then** the app returns to DoorsView without errors or visual artifacts

**AC-6: Keybinding registry updated**
**Given** the keybinding registry (Epic 39)
**When** the user is in DetailView
**Then** space/enter are registered with description "Close door" or "Go back"
**And** the help text reflects the toggle behavior

**AC-7: Table-driven tests**
**Given** table-driven tests in the appropriate test file
**When** tests exercise the toggle behavior
**Then** tests cover: open→close via space, open→close via enter, text input guard (space in feedback form), text input guard (enter in snooze picker), rapid toggle, selection preservation
