## Story

As a multiclaude operator,
I want usage monitoring data captured alongside existing session metrics,
So that quota trends are visible in the stats system and can inform retrospective analysis.

## Background

The operational data pipeline (Epic 67) captures session metrics for retrospector analysis. Quota usage data should flow into the same pipeline so that retrospectives can analyze token consumption trends, identify wasteful patterns, and recommend optimizations. This also establishes the data format for future OTEL export (Phase 3 Marvel).

## Acceptance Criteria

**AC1: Usage snapshots recorded**
**Given** a quota check runs (via cron or manual)
**When** it completes
**Then** a usage snapshot is appended to the operational data store (JSONL file in `docs/operations/` or similar)

**AC2: Snapshot data fields**
**Given** a recorded snapshot
**When** examined
**Then** contains: ISO 8601 timestamp, window usage %, per-agent token breakdown, active threshold tier, peak/off-peak flag, window start time, estimated reset time

**AC3: Retrospector compatibility**
**Given** recorded snapshots
**When** retrospector analyzes operational data
**Then** the format is compatible with existing analysis patterns (JSONL, parseable with jq)

**AC4: Trend analysis support**
**Given** historical snapshots over multiple days
**When** analyzed
**Then** enables: daily/weekly consumption patterns, per-agent usage trends over time, peak vs off-peak usage comparison

**AC5: OTEL integration point documented**
**Given** the snapshot data model
**When** documented
**Then** includes a mapping to R-016 OTEL metrics (`dark_factory.phase.token_usage`, etc.) for Phase 3 portability

**AC6: No data duplication**
**Given** raw JSONL session files already exist
**When** snapshots are recorded
**Then** they contain only derived/aggregated metrics — not raw token-per-interaction data that's already in the source files

## Dev Notes

- Keep it simple: JSONL file, one line per snapshot, parseable with `jq`
- File location: `docs/operations/quota-usage.jsonl` or `~/.multiclaude/quota/history.jsonl`
- The operational data pipeline (Epic 67) uses `docs/operations/` — follow that convention
- Snapshot frequency: driven by the cron in 76.6, typically every 5-10 minutes
- Data retention: no automatic pruning in Phase 1 — files grow slowly (one line per check)
