# Story 66.1: CLI Adapter Registration Fix

**Epic:** 66 — CLI/TUI Adapter Wiring Parity
**Status:** Not Started
**Priority:** P0
**Estimate:** S (Small)

## User Story

**As a** user with a non-textfile provider configured,
**I want** CLI commands (`threedoors task list`, `threedoors doors`, etc.) to work correctly,
**so that** I can manage my tasks from the command line regardless of which provider I use.

## Context

The unwired features audit (2026-03-13) found that `registerBuiltinAdapters()` in `cmd/threedoors/main.go` only runs in the TUI code path. CLI commands exit via `os.Exit(cli.Execute())` at line 57, before adapter registration at line 61. This means every CLI command that uses `core.DefaultRegistry()` operates against an empty registry and silently fails for non-textfile providers.

**Sprint change proposal:** `_bmad-output/planning-artifacts/sprint-change-proposal-2026-03-13-unwired-features-gaps.md`
**Audit artifact:** `_bmad-output/planning-artifacts/unwired-features-audit.md` (Gap 2)

## Acceptance Criteria

1. `registerBuiltinAdapters(core.DefaultRegistry())` is called before the CLI/TUI routing branch in `main()`
2. CLI commands (`task list`, `doors`, etc.) work correctly when a non-textfile provider is configured
3. Regression test exists that verifies the adapter registry is populated before CLI command execution
4. The TUI path continues to work correctly (no double-registration or other side effects)
5. `make test`, `make lint`, `make fmt` all pass
6. `go test -race ./cmd/threedoors/... ./internal/cli/...` passes

## Technical Notes

- The fix is moving line 61 (`registerBuiltinAdapters(core.DefaultRegistry())`) to before line 56 (the CLI routing `if` block)
- The connect wizard factory wiring (lines 45-52) should remain before registration since it's wiring a callback, not using the registry
- Test approach: add a test in `cmd/threedoors/` or `internal/cli/` that calls a CLI command with a mock provider config and verifies it doesn't fail with "provider not found"

## Files Likely Modified

- `cmd/threedoors/main.go` — move `registerBuiltinAdapters()` call
- `cmd/threedoors/main_test.go` or `internal/cli/bootstrap_test.go` — regression test
