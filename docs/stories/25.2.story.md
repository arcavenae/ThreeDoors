# Story 25.2: Read-Only Todoist Adapter with Field Mapping

## Status: Done (PR #321)

**Epic:** 25 — Todoist Integration
**Depends On:** Story 25.1
**Blocks:** Stories 25.3, 25.4

## User Story

As a ThreeDoors user who uses Todoist,
I want my Todoist tasks to appear as doors in the Three Doors TUI,
So that I can apply the Three Doors decision-friction reduction to my existing Todoist task list.

## Acceptance Criteria

**Given** a configured Todoist provider in `~/.threedoors/config.yaml`
**When** the adapter loads tasks
**Then:**

- AC1: `TodoistProvider` in `internal/adapters/todoist/todoist_provider.go` implementing `TaskProvider` interface
- AC2: `Name()` returns `"todoist"`
- AC3: `LoadTasks()` calls `Client.GetTasks()`, filters out deleted tasks (`is_deleted == true`), maps to `[]*core.Task`
- AC4: Field mapping: `content` -> `Text`, `description` -> `Context`, `is_completed` -> `Status` (true=complete, false=todo)
- AC5: Priority-to-Effort mapping with scale inversion: 0->quick-win, 1->quick-win, 2->medium, 3->deep-work, 4->deep-work
- AC6: `labels` mapped to task tags (string array)
- AC7: `SourceProvider` set to `"todoist:<project_name>"` using project name from API
- AC8: Write methods (`SaveTask`, `SaveTasks`, `DeleteTask`, `MarkComplete`) return `core.ErrReadOnly`
- AC9: `Watch()` returns `nil` (no webhook support for personal API tokens)
- AC10: `HealthCheck()` calls `Client.GetProjects()` to verify API connectivity, returns structured `HealthCheckResult`
- AC11: `Factory` function for adapter registry registration
- AC12: Tasks appear in TUI with `[TDT]` source badge
- AC13: Local cache updated on every successful `LoadTasks()` call

## Technical Notes

- Follow the same provider pattern as `internal/adapters/jira/jira_provider.go`
- Priority inversion: Todoist uses 1 (normal) to 4 (critical); ThreeDoors Effort uses opposite scale. Simple `switch` statement, not configurable mapping.
- Todoist priority 0 means "no priority set" — map to `quick-win` (lowest effort)
- Filter deleted tasks: Todoist API may return tasks with `is_deleted: true` — exclude these
- Source badge: Register `"todoist"` -> `"TDT"` in the source badge mapping
- Cache location: `~/.threedoors/todoist-cache.yaml` following the pattern from other API adapters
- Accept interfaces, return concrete types
