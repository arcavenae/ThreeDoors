## Story

As a ThreeDoors developer,
I want to use Justfile instead of Makefile for project commands,
So that I get better error messages, simpler syntax, and cross-platform support.

## Background

BOARD recommendation P-001 identified Justfile as a better fit for ThreeDoors' command runner needs. Key advantages over Make:
- No tab-sensitivity (a common contributor frustration)
- Better error messages
- Built-in `.env` loading
- Recipe parameters with defaults
- No implicit Make rules or silent failures
- `just --list` for self-documenting commands

This story requires owner sign-off on P-001 before implementation begins.

## Acceptance Criteria

**Given** the project uses a Makefile
**When** this story is implemented
**Then:**

- A `justfile` exists at project root with equivalent recipes for all Makefile targets
- All CLAUDE.md references to `make X` are updated to `just X` (or both documented)
- CI workflows updated to use `just` instead of `make`
- `just --list` shows all available commands with descriptions
- The old Makefile is removed (no dual maintenance)
- Agent definition files updated if they reference `make` commands
- README.md build instructions updated
