# Story 39.7: Global `:` Command Mode

## Status: Done (PR #365)

**Epic:** 39 — Keybinding Display System
**Priority:** P1
**Estimate:** S (1-2 hours)

---

## User Story

As a user, I want to open the `:` command palette from any screen (help, detail, insights, etc.), so that I don't have to navigate back to the doors view just to type a command.

## Context

Currently, the `:` key only opens command mode from the doors view (`updateDoors()` in main_model.go, line 1033-1040). Users on the help screen, detail view, stats view, or any other screen must press Escape/back to return to doors before typing a command like `:dashboard` or `:mood`. This is unnecessary friction.

The fix follows the exact same pattern as D-059 (universal `q` quit) and D-087 (global `?` help): intercept the key at the MainModel level, guarded by `isTextInputActive()` to prevent conflicts with text input views.

Research artifact: [Global Command Mode Analysis](../../_bmad-output/planning-artifacts/global-command-mode-analysis.md)
