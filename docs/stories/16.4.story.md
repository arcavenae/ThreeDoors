# Story 16.4: Door Detail & Status Actions

## Status: Draft

**Epic:** 16 — iPhone Mobile App (SwiftUI)
**Depends on:** Stories 16.2, 16.3
**Blocks:** Story 16.6

## User Story

As a mobile user,
I want to tap a card to see task details and change its status,
So that I can take action on tasks from my phone.

## Acceptance Criteria

**Given** the Three Doors carousel is displayed
**When** a user taps a card and interacts with the detail view
**Then:**

- AC1: Tapping a card opens a detail view (sheet or navigation push) with full task text, notes, status, and timestamps (created/updated)
- AC2: Detail view includes action buttons: Complete, Blocked, In Progress, In Review — styled as prominent, tappable buttons
- AC3: Status change writes back to Apple Notes using the same checkbox format (`- [ ]` / `- [x]`)
- AC4: Success haptic feedback (`.success` impact) on status change
- AC5: "Progress over perfection" toast/banner shown after completing a task — matches TUI behavior
- AC6: Detail view dismissible via swipe-down gesture or explicit close button
- AC7: After status change, returns to carousel with the updated card reflecting the new status
- AC8: Optimistic UI update — card updates immediately; rolls back with error alert on write failure

## Technical Notes

- Use SwiftUI `.sheet` modifier for the detail view — familiar iOS pattern, supports swipe-to-dismiss
- Haptic feedback via `UIImpactFeedbackGenerator` (`.medium` for status change, `.success` notification for completion)
- Optimistic update pattern: update local state immediately, fire async write, revert on failure with `.alert`
- The "Progress over perfection" toast can use a temporary overlay view with auto-dismiss after ~3 seconds
- Apple Notes write-back is the critical path — must handle permission denied, note not found, and concurrent edit scenarios
- Status button layout: consider a bottom action sheet or inline buttons depending on screen real estate
- The detail view should show the full task text without truncation (scrollable if needed)
