## Story

As a developer setting up remote collaboration for the first time,
I want a configuration guide and setup script for connecting a remote Claude Code session to the MCP bridge,
So that I can get remote collaboration working quickly without debugging MCP transport configuration.

## Background

Stories 53.3 and 53.4 build the MCP bridge server. But the server alone isn't useful — a remote Claude Code session needs to be configured to connect to it. This involves:

1. Installing the MCP bridge binary on the multiclaude host
2. Configuring the remote Claude Code session's MCP server settings
3. Verifying the connection works end-to-end

Anthropic's official Claude Code supports remote MCP servers via SSH stdio transport. The configuration goes in `.claude/settings.json` (or project-level settings). This story creates the documentation and a setup verification script.

## Acceptance Criteria

**Given** the integration guide at `docs/remote-collaboration-mcp.md`
**When** a developer follows the "SSH Transport Setup" section
**Then** they can configure their remote Claude Code session to connect to the MCP bridge via SSH and see multiclaude tools in their tool list

**Given** the MCP server settings example in the guide
**When** copied into a remote Claude Code session's `.claude/settings.json`
**Then** the MCP bridge tools (`multiclaude_status`, `multiclaude_message_send`, etc.) appear when Claude Code starts

**Given** the setup verification script `scripts/verify-mcp-bridge.sh`
**When** run on the multiclaude host
**Then** it verifies: the MCP bridge binary exists and is executable, the multiclaude daemon is running, a test JSON-RPC `initialize` request succeeds, and all expected tools are listed

**Given** the integration guide
**When** a developer reads the "Cloudflare Tunnel" section
**Then** they understand the steps to expose the MCP bridge via Cloudflare Tunnel for NAT-traversal scenarios (as an alternative to direct SSH)

**Given** the integration guide
**When** a developer reads the "Tailscale" section
**Then** they understand how to use Tailscale's mesh VPN as the transport layer between machines

**Given** the integration guide
**When** a developer reads the "Troubleshooting" section
**Then** they can diagnose common issues: SSH key problems, multiclaude not in PATH over SSH, MCP handshake failures, and firewall blocking

## Technical Notes

- Guide lives at `docs/remote-collaboration-mcp.md`. Complements `docs/remote-collaboration.md` (Story 53.1).
- Verification script: `scripts/verify-mcp-bridge.sh`. Performs end-to-end check of the MCP bridge.
- SSH MCP config example:
  ```json
  {
    "mcpServers": {
      "multiclaude-remote": {
        "command": "ssh",
        "args": ["user@host", "/usr/local/bin/multiclaude-mcp-bridge"]
      }
    }
  }
  ```
- Cloudflare Tunnel section is informational — no implementation, just setup steps using `cloudflared`.
- Tailscale section is informational — just explains that MCP bridge is accessible via Tailscale IP.
- The verification script should be non-destructive (read-only operations only).
- Include a "Quick Start" section at the top for developers who just want the minimum steps.
