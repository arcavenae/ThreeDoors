## User Story

As a release engineer, I want ThreeDoors releases to automatically verify that the shared homebrew tap CI passes for the ThreeDoors formula, so that broken releases are detected and flagged without manual monitoring.

## Context

ThreeDoors publishes to the shared `arcavenae/homebrew-tap` repo via GoReleaser. This tap contains formulas for multiple arcaven projects (currently ThreeDoors and switchboard). After a release, GoReleaser pushes a formula update commit to the tap. The tap's CI must pass for the ThreeDoors formula before the release is considered complete.

See: [Cross-Repo CI Strategy](../research/cross-repo-ci-strategy.md)

## Acceptance Criteria

### AC1: Tap CI Uses Matrix Strategy for Per-Formula Isolation
- [ ] Tap CI (`arcavenae/homebrew-tap/.github/workflows/ci.yml`) uses matrix strategy with one job per formula — **Blocked: requires separate PR to arcavenae/homebrew-tap**
- [ ] Each formula gets independent pass/fail status — **Blocked: requires separate PR to arcavenae/homebrew-tap**
- [ ] `fail-fast: false` ensures all formulas are tested — **Blocked: requires separate PR to arcavenae/homebrew-tap**
- [ ] CI runs `brew audit --strict`, `brew style`, `brew install`, and `brew test` per formula — **Blocked: requires separate PR to arcavenae/homebrew-tap**

### AC2: Release Verification Workflow Exists in ThreeDoors
- [x] `.github/workflows/release-verify.yml` triggers after GoReleaser release workflow completes
- [x] Workflow identifies GoReleaser's commit to tap repo by commit message pattern (`chore(formula): update threedoors`)
- [x] Workflow polls tap CI for that commit's result
- [x] 30-minute timeout with clear error message

### AC3: Failure Reporting
- [x] On tap CI failure: GitHub issue opened in ThreeDoors repo with failure details
- [x] Issue includes link to failed tap CI run
- [x] Issue labeled `bug` and `release`

### AC4: GoReleaser Config Targets Shared Tap
- [x] `.goreleaser.yml` `brews` section targets `arcavenae/homebrew-tap` (not a dedicated tap)
- [x] Formula path: `Formula/threedoors.rb`
- [x] Commit message pattern: `chore(formula): update threedoors to {{ .Tag }}`

### AC5: Spawnable Release-Manager Agent Prompt

- [x] Documents release lifecycle, failure scenarios, and recovery procedures
