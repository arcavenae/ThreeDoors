# Story 0.22: Cross-Repo CI Monitoring for Shared Homebrew Tap

**Epic:** 0 — Infrastructure & Process
**Priority:** P2
**Status:** Not Started
**Depends On:** GoReleaser workflow (not yet implemented)

---

## User Story

As a release engineer, I want ThreeDoors releases to automatically verify that the shared homebrew tap CI passes for the ThreeDoors formula, so that broken releases are detected and flagged without manual monitoring.

## Context

ThreeDoors publishes to the shared `arcaven/homebrew-tap` repo via GoReleaser. This tap contains formulas for multiple arcaven projects (currently ThreeDoors and switchboard). After a release, GoReleaser pushes a formula update commit to the tap. The tap's CI must pass for the ThreeDoors formula before the release is considered complete.

See: [Cross-Repo CI Strategy](../research/cross-repo-ci-strategy.md)

## Acceptance Criteria

### AC1: Tap CI Uses Matrix Strategy for Per-Formula Isolation
- [ ] Tap CI (`arcaven/homebrew-tap/.github/workflows/ci.yml`) uses matrix strategy with one job per formula
- [ ] Each formula gets independent pass/fail status
- [ ] `fail-fast: false` ensures all formulas are tested
- [ ] CI runs `brew audit --strict`, `brew style`, `brew install`, and `brew test` per formula

### AC2: Release Verification Workflow Exists in ThreeDoors
- [ ] `.github/workflows/release-verify.yml` triggers after GoReleaser release workflow completes
- [ ] Workflow identifies GoReleaser's commit to tap repo by commit message pattern (`chore(formula): update threedoors`)
- [ ] Workflow polls tap CI for that commit's result
- [ ] 30-minute timeout with clear error message

### AC3: Failure Reporting
- [ ] On tap CI failure: GitHub issue opened in ThreeDoors repo with failure details
- [ ] Issue includes link to failed tap CI run
- [ ] Issue labeled `bug` and `release`

### AC4: GoReleaser Config Targets Shared Tap
- [ ] `.goreleaser.yml` `brews` section targets `arcaven/homebrew-tap` (not a dedicated tap)
- [ ] Formula path: `Formula/threedoors.rb`
- [ ] Commit message pattern: `chore(formula): update threedoors to {{ .Tag }}`

### AC5: Spawnable Release-Manager Agent Prompt
- [ ] `agents/release-manager.md` exists with release domain knowledge
- [ ] Documents release lifecycle, failure scenarios, and recovery procedures
- [ ] Can be used by supervisor to spawn workers for release failure investigation

## Tasks

## Out of Scope

- Implementing the GoReleaser release workflow itself (separate story)
- homebrew-core submission
- Cosign signing or SLSA provenance
- Making pr-shepherd monitor cross-repo releases

## Technical Notes

- `HOMEBREW_TAP_TOKEN` secret already configured in ThreeDoors CI
- Tap CI currently uses `macos-latest` which is arm64
- GoReleaser commit message pattern is configurable via `commit_msg_template`
- Multiple arcaven projects may release concurrently — commit identification must be specific
