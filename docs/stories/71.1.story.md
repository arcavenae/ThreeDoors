# Story 71.1: Remove darwin/amd64 from CI Build and Alpha Release Pipeline

## Status: In Progress

## Epic

Epic 71: Drop Apple Intel (darwin/amd64) Builds

## References

- Issue: https://github.com/arcaven/ThreeDoors/issues/803
- CI workflow: `.github/workflows/ci.yml`
- GoReleaser config: `.goreleaser.yml`
- Justfile: `justfile` (build-all recipe)

## Story

As a project maintainer,
I want to remove darwin/amd64 build targets from the CI pipeline and alpha release flow,
So that CI runner minutes are saved and the project focuses exclusively on Apple Silicon (darwin/arm64) for macOS.

## Background

The project currently cross-compiles for darwin/amd64 (Intel Mac), darwin/arm64 (Apple Silicon), and linux/amd64. Apple ended Intel Mac sales in 2023 and the user base is Apple Silicon. Issue #803 requests dropping darwin/amd64 to save CI runner minutes — especially the expensive macOS runner time used for signing and notarizing Intel binaries.

## Acceptance Criteria

**AC1: GoReleaser config excludes darwin/amd64**
**Given** `.goreleaser.yml` defines build targets
**When** the ignore list is updated
**Then** `darwin/amd64` is excluded (added to `ignore:` alongside `linux/arm64`)
**And** `darwin/arm64` and `linux/amd64` remain as build targets

**AC2: Justfile build-all drops darwin/amd64**
**Given** the `build-all` recipe in `justfile` cross-compiles for all targets
**When** the darwin/amd64 build line is removed
**Then** only `darwin/arm64` and `linux/amd64` are built
**And** `just build-all` succeeds

**AC3: CI workflow removes darwin/amd64 binary builds**
**Given** `.github/workflows/ci.yml` builds `threedoors-darwin-amd64` and `threedoors-a-darwin-amd64`
**When** the amd64 build lines are removed from `build-binaries` job
**Then** only arm64 darwin binaries and linux/amd64 binaries are produced

**AC4: CI signing/verification drops amd64 binaries**
**Given** the `sign-and-notarize` job signs, verifies, bundles, and notarizes darwin-amd64 binaries
**When** all amd64-specific signing, verification, app bundle, DMG, and pkg lines are removed
**Then** only arm64 artifacts are signed, notarized, and uploaded
**And** the artifact upload step no longer references amd64 files

**AC5: Alpha release notes and Homebrew formula drop Intel references**
**Given** the `release` job creates GitHub releases and updates the alpha Homebrew formula
**When** Intel/amd64 references are removed from release notes body and formula generation
**Then** release notes list only arm64 and linux binaries
**And** the alpha Homebrew formula (`threedoors-a.rb`) removes the `elsif OS.mac?` Intel branch

**AC6: CI passes**
**Given** all changes are made
**When** the PR is submitted
**Then** CI quality-gate passes with zero warnings

## Tasks

### Task 1: Update `.goreleaser.yml`
Add `darwin/amd64` to the `ignore` list (same pattern as `linux/arm64`).

### Task 2: Update `justfile` build-all recipe
Remove the `GOOS=darwin GOARCH=amd64` build line from `build-all`.

### Task 3: Update `ci.yml` build-binaries job
Remove `threedoors-darwin-amd64` and `threedoors-a-darwin-amd64` build commands.

### Task 4: Update `ci.yml` sign-and-notarize job
Remove all amd64 references: codesign, verify, app bundle, DMG, pkg, notarize, and artifact upload lines.

### Task 5: Update `ci.yml` release job
Remove Intel binary references from release notes body. Remove amd64 SHA computation. Update alpha Homebrew formula to remove `elsif OS.mac?` Intel branch.

### Task 6: Verify CI passes
Run `just fmt`, `just lint`, `just test` locally. Confirm GoReleaser config is valid.

## Quality Gate

AC-Q1 (formatting), AC-Q5 (scope)

## Decisions

- D-71-1: Remove via `ignore` in GoReleaser (not by removing `amd64` from `goarch` list) — preserves linux/amd64 which is still needed
