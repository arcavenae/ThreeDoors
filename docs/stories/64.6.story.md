## User Story

As a developer,
I want comprehensive end-to-end tests for cross-computer sync,
So that I can verify the full sync lifecycle works correctly across simulated devices.

## Acceptance Criteria

- AC1: E2E test simulating two devices syncing tasks bidirectionally
- AC2: E2E test for offline → online reconciliation (device goes offline, makes changes, reconnects, syncs)
- AC3: E2E test for concurrent edits on both devices with conflict resolution
- AC4: E2E test for new device joining an existing sync group
- AC5: E2E test for device removal from sync group
- AC6: E2E test for large sync payload (100+ tasks, multiple changesets)
- AC7: E2E test for transport failure mid-sync (partial transfer recovery)
- AC8: All E2E tests pass with race detector: `go test -race`
- AC9: E2E tests run in Docker (reuse Epic 18 infrastructure) for CI isolation
- AC10: Test execution time under 60 seconds for the full suite

## Technical Notes

- Simulated "devices" can be separate goroutines with isolated data directories
- Use `t.TempDir()` for each simulated device's storage
- Transport can be loopback (in-process) for speed, with optional real-transport mode for integration
- Docker E2E reuses the three-tier testing infrastructure from ADR-0019 (Epic 18)
- Consider property-based testing for sync invariants (no data loss, eventual consistency)
