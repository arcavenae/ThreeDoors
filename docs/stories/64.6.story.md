## Refined by Spike (ADR-0034)

Test infrastructure refined: simulated devices use separate `t.TempDir()` directories with a shared bare Git repo. Git-based transport allows real end-to-end testing without network mocking.

## User Story

As a developer,
I want comprehensive end-to-end tests for cross-computer sync,
So that I can verify the full sync lifecycle works correctly across simulated devices.

## Acceptance Criteria

- AC1: E2E test infrastructure: `t.TempDir()` bare Git repo as "remote", two `t.TempDir()` directories as "devices" each with their own `GitSyncTransport` instance and device identity
- AC2: E2E test: two devices syncing tasks bidirectionally — device A creates task, syncs, device B pulls, modifies, syncs back, device A pulls and sees the modification
- AC3: E2E test: offline → online reconciliation — device A goes offline (mock `git push` failure via circuit breaker), makes changes, reconnects, syncs, verify all changes arrive at device B
- AC4: E2E test: concurrent edits — both devices modify different fields of the same task offline, both sync, verify property-level merge preserves both edits
- AC5: E2E test: concurrent conflicting edits — both devices modify the same field of the same task, both sync, verify LWW resolution + conflict log entry + rejected version preserved
- AC6: E2E test: new device joining — device C clones the sync repo, registers in `devices/`, pulls full task state, verify it matches device A and B
- AC7: E2E test: device removal — remove device entry from `devices/`, verify remaining devices continue syncing normally
- AC8: E2E test: large payload — 100+ tasks with 10+ modifications each, verify sync completes and all data is consistent across devices
- AC9: E2E test: transport failure mid-sync — kill push after partial transfer, verify next sync cycle completes cleanly (Git handles this natively)
- AC10: All E2E tests pass with race detector: `go test -race ./internal/core/sync/...`
- AC11: E2E tests run in Docker (reuse Epic 18 three-tier infrastructure from ADR-0019) for CI isolation
- AC12: Full E2E test suite completes in under 60 seconds
- AC13: Property-based test for sync invariant: after N random operations on M devices with arbitrary sync ordering, all devices converge to the same task state (eventual consistency)

## Technical Notes

- Each simulated "device" is a struct with: `t.TempDir()` data dir, `GitSyncTransport`, `DeviceID`, `TaskPool`
- Bare Git repo created via `git init --bare` in `t.TempDir()`
- No network involved — all Git operations are local filesystem paths
- Circuit breaker can be mocked by replacing the `git push` command with a failing script
- Property-based testing via `testing/quick` or a dedicated framework
- Docker E2E reuses the three-tier testing infrastructure from ADR-0019 (Epic 18)
