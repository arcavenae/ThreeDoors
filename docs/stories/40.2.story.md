## Story

As a user,
I want the completion trends sparkline to use a color gradient from cool (low) to warm (high),
So that I can instantly see my completion patterns through visual warmth rather than reading numbers.

## Background

The existing `sparkline()` function in `insights_view.go` returns plain Unicode block characters (▁▂▃▄▅▆▇█) as a monochrome string. Each character maps to a proportional value but all render in the same foreground color.

This story modifies the sparkline to apply per-character color using `lipgloss.Blend1D()` to create a smooth gradient from blue (low values) through teal to yellow (high values). This palette is color-blind safe (avoids the red-green axis entirely).

## Acceptance Criteria

**Given** the sparkline renders for the last 7 days
**When** a day has zero completions
**Then** the sparkline character renders in the gradient's starting color (blue #3B82F6)

**Given** the sparkline renders for the last 7 days
**When** a day has the maximum completions
**Then** the sparkline character renders in the gradient's ending color (yellow #EAB308)

**Given** the sparkline renders for the last 7 days
**When** days have varying completion counts
**Then** each character is individually styled with a color proportional to its value using `Blend1D()`
**And** the gradient transitions smoothly from blue through teal to yellow

**Given** a light terminal background
**When** the sparkline renders
**Then** colors are adapted via `lipgloss.AdaptiveColor` to remain visible

**Given** the sparkline data contains all zeros
**When** the sparkline renders
**Then** all characters render at the minimum height (▁) in the starting color
**And** no division-by-zero or rendering errors occur

## Technical Notes

- Modify `sparkline()` to return `[]string` (individually styled characters) instead of `string`
- Or: keep `sparkline()` as-is, add `styledSparkline()` that wraps each rune in Lipgloss style
- Gradient: `lipgloss.Blend1D(lipgloss.Color("#3B82F6"), lipgloss.Color("#EAB308"), float64(value)/float64(max))`
- Update `renderCompletionTrends()` to use styled sparkline output
- The sparkline function itself is 22 lines — this is a focused change
