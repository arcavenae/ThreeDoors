# Story 18.3: Input Sequence Replay Tests for User Workflows

## Status: done

## Story

As a developer,
I want automated tests that replay user input sequences against the TUI,
So that complete user workflows (launch → select → manage → exit) are validated end-to-end.

## Acceptance Criteria

- [x] Tests exercise workflow: Launch → view 3 doors → select door (A key) → Enter → verify detail view
- [x] Tests exercise workflow: Launch → re-roll doors (S key) → verify new doors displayed
- [x] Tests exercise workflow: Launch → select door → Enter → complete task (C key) → verify task removed from pool
- [x] Tests exercise workflow: Launch → select door → Enter → mark blocked (B key) → enter blocker text → verify
- [x] Tests exercise workflow: Launch → search (/) → :add command → verify task in pool
- [x] Tests exercise workflow: Launch → door feedback (N key) → select option → verify feedback recorded
- [x] Each workflow asserts on `FinalModel` state (not just output text)
- [x] Workflows use `teatest.WaitFor` for intermediate state assertions where needed
- [x] Test task files created via `t.TempDir()` with known task sets
- [x] Tests use stdlib `testing` package only; table-driven for workflow variants

## Technical Notes

- Input replays test the full Bubbletea Update() → View() cycle
- Model assertions are more stable than output assertions for workflow correctness
- `WaitFor` with timeout prevents tests from hanging on unexpected state
- Each workflow should complete in <2s (set `teatest.WithFinalTimeout(2*time.Second)`)
- N key opens door feedback (not quick-add); quick-add is via `:add` command in search mode
- No dedicated help overlay with `?` key — help available via `:help` command

## Dev Notes

- Build on existing test harness from Story 18.1 (`NewTestApp`, `testProvider`, etc.)
- Use `testProvider` for save/complete tracking in assertions
- Door selection keys: a (left), w (middle), d (right), Enter (open detail)
- Detail view keys: c (complete), b (blocked), i (in-progress), esc (back)
- Search entry: / key, then type `:add <text>`, Enter to submit
- Feedback entry: n key from doors view, 1-4 to select option
