## Story

As a multiclaude operator,
I want a `/quota-status` command that shows current usage at a glance,
So that I can quickly check quota health without digging through JSONL files.

## Background

R-004 Phase 1 calls for a `multiclaude quota status` command. We implement this as a Claude Code slash command (`/quota-status`) that wraps the underlying Go or shell tool. The command surfaces data from stories 76.1-76.3 in a human-readable format.

## Acceptance Criteria

**AC1: Window usage summary**
**Given** the command is invoked
**When** JSONL data is available
**Then** displays: current window token usage (absolute + percentage), estimated plan type, time since window start, estimated time until reset

**AC2: Per-agent breakdown table**
**Given** per-agent attribution data (from 76.2)
**When** displayed
**Then** shows a sorted table: agent name, tokens used, % of total, priority tier

**AC3: Threshold status indicator**
**Given** current usage level
**When** displayed
**Then** shows color-coded status: green (<70%), yellow (70-80%), orange (80-90%), red (>90%)

**AC4: Peak/off-peak indicator**
**Given** current time
**When** displayed
**Then** shows whether we're in Anthropic peak hours (05:00-11:00 PT) or off-peak

**AC5: Slash command implementation**
**Given** the command
**When** implemented
**Then** exists at `.claude/commands/quota-status.md` as a slash command that invokes the underlying tool

**AC6: Graceful empty state**
**Given** no JSONL data available (fresh install, no sessions)
**When** the command is invoked
**Then** displays a helpful message explaining no usage data found, suggests checking config directory

**AC7: Machine-parseable output option**
**Given** the underlying tool
**When** invoked with a `--json` flag (or similar)
**Then** outputs structured JSON for programmatic consumption

## Tasks

### Task 1: Implement the CLI/script tool
Go command or shell script that combines 76.1+76.2+76.3 outputs into a formatted display.

### Task 2: Create the slash command wrapper
`.claude/commands/quota-status.md` that instructs the agent to invoke the tool and display results.

### Task 3: Implement formatted table output
Human-readable table with alignment, color indicators (via text labels since slash commands are text-based).

### Task 4: Implement JSON output mode
`--json` flag for machine-parseable output.

### Task 5: Write integration tests
Test the full pipeline from JSONL fixtures through to formatted output.

## Dev Notes

- Slash commands are markdown files in `.claude/commands/` — they instruct Claude on what to do
- The actual computation should live in a Go binary or shell script that the slash command invokes
- Consider integration point with future `/rollcall` command for a usage column
- Color coding in terminal output: use text labels like `[GREEN]`, `[YELLOW]`, `[RED]` since slash command output goes through Claude
