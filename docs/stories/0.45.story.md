# Story 0.45: Agent Definition Updates for Scoped Labels

## Story

As the ThreeDoors agent infrastructure maintainer,
I want all agent definition files updated to reference the new scoped label names,
So that agents use the correct label names after the migration (Story 0.44).

**Status:** Done (PR #520)

---

## Background

Story 0.44 migrates all GitHub labels to scoped `.` separator format. Agent definition files currently reference old label names (e.g., `needs-human-input`, `out-of-scope`). These references must be updated to match the new names (e.g., `status.needs-human`, `scope.out-of-scope`).

---

## Acceptance Criteria

### Agent Definition Updates

- **AC1:** `agents/merge-queue.md` updated:
  - `needs-human-input` → `status.needs-human`
  - `out-of-scope` → `scope.out-of-scope`
  - Labels table updated with new scoped names
  - `status.do-not-merge` added to merge gates
- **AC2:** `agents/envoy.md` updated:
  - All label references use scoped names
  - Label authority documented (which scopes envoy can set autonomously)
- **AC3:** `agents/pr-shepherd.md` updated:
  - `priority.*` references use new scoped names
  - `status.blocked` and `status.stale` references added where relevant
- **AC4:** Any other agent files (`agents/*.md`) referencing labels are updated

### Issue Tracker Update

- **AC5:** `docs/issue-tracker.md` updated if it contains label references using old names

### No Behavioral Changes

- **AC6:** Agent authority boundaries remain unchanged — only label name references are updated
- **AC7:** No new agent capabilities or permissions are introduced
