# Story 71.2: Remove darwin/amd64 from Stable Release Workflow

## Status: Done (PR #815)

## Epic

Epic 71: Drop Apple Intel (darwin/amd64) Builds

## References

- Issue: https://github.com/arcaven/ThreeDoors/issues/803
- Release workflow: `.github/workflows/release.yml`
- Depends on: Story 71.1 (GoReleaser config change)

## Story

As a project maintainer,
I want to remove darwin/amd64 from the stable release signing and packaging pipeline,
So that tagged releases no longer produce Intel Mac artifacts and macOS runner time is halved.

## Background

The stable release workflow (`release.yml`) triggers on semver tags and runs GoReleaser followed by a sign-stable job on a macOS runner. The sign-stable job downloads darwin archives for both arm64 and amd64, signs them, notarizes them, builds pkg installers, and re-uploads. After Story 71.1 removes darwin/amd64 from GoReleaser, this workflow must stop trying to download/sign/package the amd64 archive.

## Acceptance Criteria

**AC1: Release workflow stops downloading darwin_amd64 archive**
**Given** `.github/workflows/release.yml` downloads `threedoors_*_darwin_amd64.tar.gz`
**When** the amd64 download pattern is removed
**Then** only `threedoors_*_darwin_arm64.tar.gz` is downloaded

**AC2: Sign, notarize, and pkg loops exclude amd64**
**Given** the release workflow loops `for ARCH in arm64 amd64` in multiple steps
**When** `amd64` is removed from each loop
**Then** only arm64 is processed in sign, notarize, pkg build, pkg notarize, and re-upload steps

**AC3: Checksums and re-upload exclude amd64**
**Given** the checksums update step processes both architectures
**When** amd64 references are removed
**Then** only arm64 archive, arm64 pkg, and checksums.txt are re-uploaded

**AC4: Workflow succeeds on tag push**
**Given** all changes are made
**When** a semver tag is pushed (or workflow is simulated)
**Then** the release pipeline succeeds without referencing amd64 artifacts

## Tasks

### Task 1: Update download step
Remove `--pattern "threedoors_${VERSION}_darwin_amd64.tar.gz"` from `gh release download`.

### Task 2: Update sign-and-re-archive step
Change `for ARCH in arm64 amd64` to process only arm64 (or remove the loop entirely since there's only one arch).

### Task 3: Update notarize step
Same loop change as Task 2.

### Task 4: Update pkg build step
Same loop change as Task 2.

### Task 5: Update pkg notarize step
Same loop change as Task 2.

### Task 6: Update checksums and re-upload step
Remove amd64 archive and pkg from the loop and from `gh release upload`.

### Task 7: Verify
Run `just fmt`, `just lint`, `just test`. Validate YAML syntax of the workflow file.
