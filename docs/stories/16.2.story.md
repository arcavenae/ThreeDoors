# Story 16.2: Task Provider Protocol & Apple Notes Reader

## Status: Draft

**Epic:** 16 тАФ iPhone Mobile App (SwiftUI)
**Depends on:** Story 16.1
**Blocks:** Stories 16.3, 16.4, 16.5

## User Story

As a mobile user,
I want the app to read tasks from the same Apple Notes document used by the desktop TUI,
So that my tasks are consistent across devices.

## Acceptance Criteria

**Given** the need to read tasks from Apple Notes on iOS
**When** the task provider is implemented
**Then:**

- AC1: `TaskProvider` Swift protocol defined mirroring Go's `TaskProvider` interface (`loadTasks`, `saveTask`, `deleteTask`, `markComplete`)
- AC2: `Task` Swift struct defined with `Codable` conformance тАФ fields: `id` (String), `text` (String), `status` (TaskStatus), `notes` (String?), `createdAt` (Date), `updatedAt` (Date)
- AC3: `TaskStatus` enum matches Go version: `todo`, `blocked`, `inProgress`, `inReview`, `complete` тАФ with `Codable` and `CustomStringConvertible` conformance
- AC4: `AppleNotesProvider` implementation reads tasks from Apple Notes using iOS Apple Notes API
- AC5: Checkbox format parsing matches TUI: `- [ ] task` (todo), `- [x] task` (complete)
- AC6: Deterministic UUID generation matches Go implementation (`noteTitle:lineIndex` based hashing)
- AC7: Note title configurable (matches TUI's `config.yaml` note title setting)
- AC8: Error handling for Notes access permission denied тАФ presents user-friendly message
- AC9: Unit tests with mock note content verifying parsing of checkbox formats, status mapping, and edge cases (empty notes, malformed lines)

## Technical Notes

- The `TaskProvider` protocol is the Swift equivalent of Go's `internal/tasks/provider.go` interface тАФ port the interface design, not the implementation
- Apple Notes access on iOS may require EventKit or CloudKit queries тАФ research the current iOS API surface
- Deterministic UUID: use SHA-256 hash of `"\(noteTitle):\(lineIndex)"` truncated to UUID format, matching Go's `uuid.NewSHA1` approach
- `Task` struct should use value semantics (struct, not class) тАФ Swift's equivalent of Go's value receiver pattern
- Keep `AppleNotesProvider` as a concrete type (return concrete, accept protocol) тАФ mirrors Go convention
- Checkbox regex: `^\s*-\s*\[([ xX])\]\s*(.+)$`
- Timestamps always in UTC (use `Date()` which is timezone-independent in Swift)
