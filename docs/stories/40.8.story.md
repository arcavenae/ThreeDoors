## Story

As a user,
I want to navigate between an Overview tab and a Detail tab within the insights view using Tab/Shift-Tab,
So that I can see summary stats at a glance and drill into deeper visualizations by choice.

## Background

Phase 1 establishes the Overview tab. Phase 2 stories (heatmap 40.5, session highlights 40.6, animated counters 40.7) add content that would push the view beyond 24 rows if placed on the Overview. Tab navigation provides a clean separation: essential stats on Overview, deeper analysis on Detail.

## Acceptance Criteria

**Given** the insights view is open on the Overview tab
**When** the user presses Tab
**Then** the view switches to the Detail tab
**And** a tab indicator shows which tab is active: `[Overview]  Detail` vs `Overview  [Detail]`

**Given** the insights view is open on the Detail tab
**When** the user presses Shift-Tab
**Then** the view switches back to the Overview tab

**Given** the Detail tab has content taller than the terminal
**When** the tab renders
**Then** a viewport with scroll support is used (bubbles viewport component)
**And** j/k or Up/Down keys scroll within the tab

**Given** the user presses Esc on either tab
**When** the key is processed
**Then** the view returns to the doors view (existing behavior preserved)

## Technical Notes

- Add `activeTab int` field to InsightsView (0=Overview, 1=Detail)
- Handle `tea.KeyTab` and `tea.KeyShiftTab` in `Update()`
- Tab indicator rendered as part of the footer using Lipgloss styling
- Active tab label: bold/underline; inactive: normal
- Detail tab initially shows a placeholder until Phase 2 stories add content
- Add viewport model from `charmbracelet/bubbles/viewport` for Detail tab scrolling
- Invalidate render cache on tab switch
