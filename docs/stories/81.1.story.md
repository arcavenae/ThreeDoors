# Story 81.1: Merge-Queue Polling Fetch

**Epic:** 81 — Git Coordination Fixes
**Status:** Not Started
**Priority:** P0
**Estimate:** S (< 1 hour)

## User Story

As the merge-queue agent,
I want to fetch the latest origin/main before each polling cycle,
So that my PR checks, branch cleanup, and scope validation use current data instead of a stale local ref.

## Context

Persistent agents (merge-queue, pr-shepherd, envoy, etc.) share the main checkout at the multiclaude repos directory. Unlike worker worktrees, the daemon does NOT auto-refresh this shared checkout. The git safety hook only blocks `git fetch` in worker worktrees (paths containing `/.multiclaude/wts/`), so persistent agents CAN fetch freely.

Currently, merge-queue never fetches. Over time, its local `origin/main` ref drifts behind the actual remote, causing:
- `git branch -r --merged origin/main` to miss branches that are actually merged
- Stale scope checks against an outdated ROADMAP.md
- Post-merge CI checks referencing old SHAs

## Acceptance Criteria

- **AC1:** Merge-queue's polling loop begins with `git fetch origin main` before checking open PRs
- **AC2:** The fetch is added to `agents/merge-queue.md` in the Polling Loop section (step 0, before existing step 1)
- **AC3:** The fetch uses `git fetch origin main` (not `git pull` — no working tree changes needed, just ref update)
- **AC4:** Error handling: if fetch fails (network issue), log a warning but continue the polling cycle with stale data rather than halting

## Technical Notes

- The git safety hook (`scripts/hooks/git-safety.sh`) exempts persistent agents — fetch is NOT blocked outside worker worktrees
- Only the ref needs updating; the working tree checkout state is irrelevant for merge-queue's `gh` CLI operations
- This is a docs-only change to the agent definition file

## Out of Scope

- Refreshing the working tree (checkout) — merge-queue uses `gh` CLI which talks to GitHub API, not local files
- Changing the daemon to refresh the shared checkout (that's Story 81.4)
- Other persistent agents' fetch behavior (covered in Story 81.2)

## Implementation Notes

> [observation] 2026-04-02 — git safety hook detects worker worktrees via `/.multiclaude/wts/` path pattern. Persistent agents at the main checkout path are exempt from fetch blocking.
