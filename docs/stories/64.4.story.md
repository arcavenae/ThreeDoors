## Refined by Spike (ADR-0034)

Conflict resolution strategy determined: extend ADR-0012 property-level LWW with per-device vector clocks. No CRDTs needed at this scale. Git custom merge driver handles file-level conflicts before application-level resolution.

## User Story

As a ThreeDoors user who edits tasks on multiple computers,
I want conflicts to be resolved automatically (with manual override available),
So that I don't lose work when the same task is modified on two machines.

## Acceptance Criteria

- AC1: Extend `FieldVersion` (ADR-0012) with `DeviceID` field — each field change tracks which device made it
- AC2: Per-task vector clock (`map[DeviceID]uint64`) tracks logical timestamps per device for causal ordering
- AC3: Non-overlapping changes (different fields on same task, different devices) merge automatically — both edits preserved
- AC4: Overlapping changes (same field on same task) resolved by: (a) vector clock causal ordering if one happened-before the other, (b) UTC timestamp comparison if concurrent, (c) device ID lexicographic tiebreaker if timestamps equal
- AC5: Custom Git merge driver registered for `tasks.yaml` — performs task-level three-way merge (common ancestor + device A + device B) using application logic, not Git's line-based diff
- AC6: `sessions.jsonl` uses Git's built-in `union` merge strategy (append-only, no conflicts possible)
- AC7: Rejected versions logged to `~/.threedoors/sync/conflicts.jsonl` with: conflict ID, timestamp, task ID, device IDs, conflicting field values, resolution outcome
- AC8: `threedoors sync conflicts` CLI command lists recent conflicts with resolution details
- AC9: `threedoors sync resolve <conflict-id> --keep <device>` allows manual override — replays the rejected version's field values
- AC10: TUI notification when conflicts are auto-resolved (non-blocking, informational — reuse existing notification infrastructure)
- AC11: No data loss — rejected version is always preserved in conflict log for recovery
- AC12: Unit tests for: non-overlapping merge, overlapping LWW, vector clock ordering, three-way merge, concurrent create, concurrent delete, manual override
- AC13: Stress test: 50+ simultaneous conflicting changes from 3 simulated devices resolve correctly

## Technical Notes

- The custom Git merge driver is a ThreeDoors binary mode (`threedoors merge-driver %O %A %B`) registered in `.gitattributes` and `.git/config`
- Three-way merge: parse base (%O), local (%A), remote (%B) as YAML task lists, merge per-task using property-level strategy, write result to %A
- Vector clock increment: each local change increments `vectorClock[localDeviceID]++`
- Conflict log is append-only JSONL — rotates at 1MB (reuse sync log rotation pattern)
- Cross-machine conflicts are peer-to-peer (no authority), unlike provider sync (Epic 47) where the provider is authoritative
