## Technical Notes

- Each theme is a single file in `internal/tui/themes/` ‚Äî one primary type per file
- Render functions are pure: `(content string, width int, selected bool) ‚Üí string` ‚Äî no I/O, no side effects
- Use `fmt.Fprintf` with `strings.Builder` for efficient string construction ‚Äî never `WriteString` + `Sprintf`
- Width calculation must account for Unicode character widths ‚Äî box-drawing characters are single-width in all target terminals
- Content word-wrapping should use Lipgloss `wordwrap` or a simple width-aware split ‚Äî avoid pulling in external dependencies
- Performance target: render must complete within 1ms for standard widths (NFR18) ‚Äî pure string manipulation, this is trivially achieved
- Theme feasibility tiers from analyst review: Modern (Tier 1, lowest risk), Sci-Fi (Tier 1, `‚óà` rivet is only risk), Shoji (Tier 1, zero rendering risk)
- Reference mockups in `../../_bmad-output/planning-artifacts/door-themes-research.md` (PR #116)
