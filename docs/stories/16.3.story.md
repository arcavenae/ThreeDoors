# Story 16.3: Three Doors Card Carousel

## Status: Draft

**Epic:** 16 — iPhone Mobile App (SwiftUI)
**Depends on:** Stories 16.1, 16.2
**Blocks:** Stories 16.4, 16.6

## User Story

As a mobile user,
I want to see three task cards I can swipe through,
So that I get the Three Doors experience on my phone.

## Acceptance Criteria

**Given** tasks loaded from Apple Notes via `TaskProvider`
**When** the carousel view is displayed
**Then:**

- AC1: Three task cards displayed as a horizontal swipeable carousel (`TabView` with `PageTabViewStyle` or custom `ScrollView` with snapping)
- AC2: Each card shows task text, status badge (colored pill), and creation date
- AC3: Cards use distinct visual styling consistent with TUI door aesthetic — rounded corners, shadow, prominent task text
- AC4: Current card indicator (page dots or similar) shows position within the three cards
- AC5: Smooth swipe animation between cards at 60 FPS
- AC6: Empty state handled gracefully: "No tasks found — add tasks in Apple Notes" message with icon
- AC7: Loading state displayed while fetching from Apple Notes (skeleton cards or spinner)
- AC8: Card layout adapts to different iPhone screen sizes (iPhone SE through iPhone 16 Pro Max)

## Quality Gate

## Technical Notes

- The carousel is the core "Three Doors" UX on mobile — three cards map to three doors
- Random selection of three tasks from the full task list mirrors the TUI's door selection algorithm
- Use `@State` / `@StateObject` for carousel state management
- Consider `GeometryReader` for responsive card sizing across screen sizes
- Status badge colors should match TUI conventions (green=complete, yellow=in-progress, red=blocked, etc.)
- The card design should feel like a "door" — consider a header area (door frame) and content area
- Empty state should guide users to Apple Notes, not provide in-app task creation (that's Phase 2)
- Test with VoiceOver early — cards need accessibility labels
