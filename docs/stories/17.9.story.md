# Story 17.9: Simplify Sci-Fi Theme and Improve Modern Theme Contrast

**Status:** Done (PR #183)
**Epic:** 17 вҖ” Door Theme System
**Depends on:** Story 17.7
**Blocks:** None

## User Story

As a user,
I want the Sci-Fi theme to feel futuristic without overwhelming my task text, and the Modern theme to clearly show which door is selected,
So that both themes are usable for daily task management.

## Acceptance Criteria

### Sci-Fi Theme

**Given** the Sci-Fi theme is selected
**When** three doors are rendered
**Then:**

- AC1: Shade rails reduced from 2 characters (`в–‘в–‘`) to 1 (`в–‘`) on each side, or removed entirely вҖ” recovering at least 2-4 characters of content width per door
- AC2: Mid-bar separator (`в• в•җв•җв•Әв•җв•җв•җв•җв•Әв•җв•җв•Ј`) removed вҖ” content and `[ACCESS]` label share a single panel
- AC3: `[ACCESS]` label rendered below task text within the same content area (not in a separate panel)
- AC4: Minimum 15 characters of usable text width at `MinWidth` (reduced from current 18 if border overhead decreases)
- AC5: Double-line outer frame (`в•”в•җв•—в•‘в•ҡв•қ`) preserved вҖ” this is the core Sci-Fi identity
- AC6: Selected state still uses `в–“` (or brighter shade) for visual distinction

### Modern Theme

**Given** the Modern theme is selected
**When** a door is selected vs unselected
**Then:**

- AC7: Unselected frame color changed from `245` to `238` or dimmer вҖ” creating at least a 15-point brightness gap to the selected color
- AC8: Selected state retains heavy line weight (`в”Ғв”ғ`) AND bright color `255` вҖ” the combination of weight + brightness makes selection unmistakable
- AC9: Top and bottom padding each reduced by 1 blank line вҖ” doorknob sits vertically centered rather than bottom-heavy
