# Story 17.9: Simplify Sci-Fi Theme and Improve Modern Theme Contrast

**Status:** To Do
**Epic:** 17 — Door Theme System
**Depends on:** Story 17.7
**Blocks:** None

## User Story

As a user,
I want the Sci-Fi theme to feel futuristic without overwhelming my task text, and the Modern theme to clearly show which door is selected,
So that both themes are usable for daily task management.

## Acceptance Criteria

### Sci-Fi Theme

**Given** the Sci-Fi theme is selected
**When** three doors are rendered
**Then:**

- AC1: Shade rails reduced from 2 characters (`░░`) to 1 (`░`) on each side, or removed entirely — recovering at least 2-4 characters of content width per door
- AC2: Mid-bar separator (`╠══╪════╪══╣`) removed — content and `[ACCESS]` label share a single panel
- AC3: `[ACCESS]` label rendered below task text within the same content area (not in a separate panel)
- AC4: Minimum 15 characters of usable text width at `MinWidth` (reduced from current 18 if border overhead decreases)
- AC5: Double-line outer frame (`╔═╗║╚╝`) preserved — this is the core Sci-Fi identity
- AC6: Selected state still uses `▓` (or brighter shade) for visual distinction

### Modern Theme

**Given** the Modern theme is selected
**When** a door is selected vs unselected
**Then:**

- AC7: Unselected frame color changed from `245` to `238` or dimmer — creating at least a 15-point brightness gap to the selected color
- AC8: Selected state retains heavy line weight (`━┃`) AND bright color `255` — the combination of weight + brightness makes selection unmistakable
- AC9: Top and bottom padding each reduced by 1 blank line — doorknob sits vertically centered rather than bottom-heavy

## Target Visuals

### Sci-Fi Target (28-char wide, unselected)

The spaceship door should feel like a futuristic airlock panel — heavy outer frame
with clean interior. The `[ACCESS]` label stays but as inline content, not a
separate panel. Side rails reduced to 1 char.

```
╔═╤══════════════════════╤═╗
║░│                      │░║
║░│                      │░║
║░│  Deploy to staging   │░║
║░│                      │░║
║░│                      │░║
║░│                      │░║
║░│            [ACCESS]  │░║
║░│                      │░║
╚═╧══════════════════════╧═╝
```

vs current (too heavy, content lost in decoration):
```
╔══╤════════════════╤══╗
║░░│                │░░║
║░░│  Deploy to     │░░║
║░░│  staging       │░░║
╠══╪════════════════╪══╣
║░░│                │░░║
║░░│   [ACCESS]     │░░║
║░░│                │░░║
╚══╧════════════════╧══╝
```

Removing the mid-bar and reducing rails from 2→1 recovers **4 chars** of content width.

### Modern Target (28-char wide)

The Modern door is about negative space. The problem is that unselected (color `245`)
and selected (color `255`) are only 10 values apart on the grayscale — barely visible.

**Unselected** — dim gray (`238`), thin lines:
```
──────────────────────────
│                        │
│                        │
│   Write unit tests     │
│                        │
│                    ●   │
│                        │
──────────────────────────
```

**Selected** — bright white (`255`), heavy lines:
```
━━━━━━━━━━━━━━━━━━━━━━━━━━
┃                        ┃
┃                        ┃
┃   Write unit tests     ┃
┃                        ┃
┃                    ●   ┃
┃                        ┃
━━━━━━━━━━━━━━━━━━━━━━━━━━
```

Key changes:
- Unselected frame dimmed from `245` → `238` (17-point gap instead of 10)
- Bottom padding reduced from 2 blank lines to 1 — doorknob sits closer to content
- Top padding reduced from 2 blank lines to 1 — tighter, more intentional whitespace
