## Story

As a user,
I want numeric stats to count up smoothly from 0 to their final value when I first enter the insights view,
So that opening the dashboard feels like an intentional, satisfying reveal rather than a static page.

## Background

SOUL.md says "every interaction should feel deliberate" and describes the difference between "I clicked a flat screen" and "I pressed a physical button." Animated counters add the "button feel" — the numbers tick up over ~500ms, creating a brief moment of anticipation and reward.

The animation is subtle: only hero numbers (total tasks, streak count) animate. Sparklines, bars, and panel borders render immediately. Animation plays once on initial view entry and is skipped on re-entry within the same session.

## Acceptance Criteria

**Given** the user enters the insights view for the first time in a session
**When** the view renders
**Then** the hero number counts up from 0 to its final value over 300-500ms
**And** the streak count counts up similarly
**And** sparklines, bars, and panels render immediately (no animation)

**Given** the animation is in progress
**When** each tick fires (every 30ms)
**Then** displayed values are `int(finalValue * progress)` where progress goes from 0.0 to 1.0

**Given** the animation has completed (progress >= 1.0)
**When** no more tick messages are sent
**Then** the tick timer stops
**And** CPU usage returns to zero for the insights view

**Given** the user has already viewed insights during this session
**When** they return to the insights view
**Then** numbers display at their final values immediately (no animation)

**Given** the user presses Esc during animation
**When** the view exits
**Then** the animation stops cleanly with no lingering tick messages

## Technical Notes

- Add to InsightsView: `animating bool`, `animationProgress float64`, `hasAnimatedThisSession bool`
- On first view entry: set `animating = true`, `animationProgress = 0.0`, start `tea.Tick(30*time.Millisecond)`
- `Update()` handles `StatsAnimationTickMsg`: increment progress by `30ms/500ms = 0.06` per tick
- When `animationProgress >= 1.0`: set `animating = false`, `hasAnimatedThisSession = true`
- `View()` renders values as `int(float64(value) * progress)` when animating
- ~16 frames total (500ms / 30ms)
- New message type: `StatsAnimationTickMsg struct{}`
- Ensure cache invalidation during animation (every tick must re-render)
