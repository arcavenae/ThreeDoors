# Story 3.5.7: Adapter Test Scaffolding & CI Coverage Floor

**Status:** in-progress
**Epic:** 3.5 — Bridging
**Blocks:** Epic 9 (Testing Strategy)

## User Story

As a developer,
I want test infrastructure scaffolding and CI coverage enforcement,
So that Epic 9 (Testing Strategy) has a foundation and coverage doesn't erode.

## Acceptance Criteria

**Given** the current CI pipeline without coverage enforcement
**When** scaffolding is complete
**Then:**

- AC1: Test fixture directory `testdata/adapters/` created with sample YAML task data for adapter testing
- AC2: Mock/stub helpers created in `internal/testing/` for common test patterns (mock provider, test task factories)
- AC3: CI pipeline updated to measure coverage (`go test -coverprofile`) and fail if below threshold (set to current level ~75%)
- AC4: Coverage report summary posted as PR comment via GitHub Actions
- AC5: `internal/adapters/contract_scaffold_test.go` scaffolding ready for Epic 9 to fill with new adapter contract tests
- AC6: Tests use stdlib `testing` package only; table-driven for >2 cases; `t.Helper()` in helpers; `t.Cleanup()` for resources
- AC7: All existing tests continue to pass

## Quality Gate

AC-Q1: gofumpt formatted
AC-Q2: golangci-lint passes with zero warnings
AC-Q3: All tests pass
AC-Q4: Rebased on latest main
AC-Q5: Scope-checked against story
AC-Q6: Errors handled with context wrapping

## Tasks

1. Create `testdata/adapters/` with sample YAML task fixtures (valid, empty, malformed)
2. Create `internal/testing/` package with exported mock provider, test task factories, and assertion helpers
3. Update CI workflow to enforce coverage floor (~75%) and fail PRs that drop below
4. Add coverage report comment step to CI workflow
5. Create `internal/adapters/contract_scaffold_test.go` with scaffold structure for Epic 9
6. Verify all existing tests pass and quality gates met
