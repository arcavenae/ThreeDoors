## Story

As a ThreeDoors user,
I want a data layer that reads and aggregates my completed tasks,
So that the history TUI view and CLI command can display what I've accomplished.

## Background

ThreeDoors already tracks completions in two places:
- `completed.txt` — append-only log of completed task lines with timestamps
- JSONL session logs — `task_completed` events with full metadata

This story creates the data reader that both the TUI view (Story 70.2) and CLI command (Story 70.3) will consume. It's the foundation layer — no UI.

## Acceptance Criteria

- [ ] New type `CompletionRecord` in `internal/core/` with fields:
  - `Title string`
  - `CompletedAt time.Time`
  - `Source string` (which adapter/source, empty for local tasks)
  - `TaskID string` (optional, empty if unavailable)
- [ ] New `CompletionReader` struct with constructor `NewCompletionReader(configDir string) *CompletionReader`
- [ ] Method `Read(ctx context.Context) ([]CompletionRecord, error)` — reads from `completed.txt`
- [ ] Method `Today(ctx context.Context) ([]CompletionRecord, error)` — filters to today (local timezone)
- [ ] Method `ThisWeek(ctx context.Context) ([]CompletionRecord, error)` — filters to current calendar week
- [ ] Method `ThisMonth(ctx context.Context) ([]CompletionRecord, error)` — filters to current calendar month
- [ ] Method `Since(ctx context.Context, since time.Time) ([]CompletionRecord, error)` — filters from arbitrary date
- [ ] Records sorted newest-first by default
- [ ] Handles empty/missing `completed.txt` gracefully (returns empty slice, nil error)
- [ ] Handles malformed lines gracefully (skip with logged warning, don't fail)
- [ ] Table-driven tests with test fixtures in `testdata/`
- [ ] 80%+ test coverage on `CompletionReader`
- [ ] `make lint` passes with zero warnings

## Notes

- `context.Context` as first parameter per CLAUDE.md rules
- Return concrete type (`*CompletionReader`), not interface — per "accept interfaces, return concrete types"
- Timestamps in UTC internally, but filter methods use local timezone for "today"/"this week" boundaries
- Do NOT parse JSONL session logs in this story — `completed.txt` is sufficient. JSONL can be added later if needed.
