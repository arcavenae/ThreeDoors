## Story

As a developer,
I want views to use lipgloss.JoinVertical and lipgloss.Place for layout composition,
So that manual `\n` concatenation and padding math are eliminated, improving maintainability.

## Background

ThreeDoors currently only uses `lipgloss.JoinHorizontal`. Manual `\n` concatenation is used for vertical composition, and manual padding math is used for centering. `lipgloss.JoinVertical` and `lipgloss.Place` are built-in functions that handle these cases correctly and consistently. This is a pure refactor — no functional behavior change.

## Acceptance Criteria

**Given** views that concatenate content vertically with `\n`
**When** the refactor is applied
**Then** `lipgloss.JoinVertical` is used where it produces cleaner code

**Given** views that center content with manual padding calculations
**When** the refactor is applied
**Then** `lipgloss.Place` is used for centered content placement

**Given** all golden file tests
**When** the refactor is complete
**Then** all golden file tests pass (updated if rendering differs trivially)

**Given** the application runs normally
**When** users interact with any view
**Then** there is no functional behavior change — only code quality improvement

## Technical Notes

- Audit all `View()` methods in `internal/tui/` for manual vertical joins
- Replace `strings.Join(parts, "\n")` and `part1 + "\n" + part2` patterns with `lipgloss.JoinVertical`
- Replace manual centering (spaces padding) with `lipgloss.Place(width, height, lipgloss.Center, lipgloss.Center, content)`
- Both functions already exist in the lipgloss dependency — no new imports needed
- Focus on greeting view (empty state), onboarding views, and any centered modal content
- Do NOT refactor where manual `\n` is intentionally used for fine-grained spacing
