## User Story

As a ThreeDoors developer/analyst, I want dependency events logged to session metrics, so that usage patterns can be analyzed and the feature can be improved over time.

## Context

This is the final story in Epic 29. It adds session metrics logging for dependency-related events: added, removed, unblocked, and cycle rejected. The auto-unblock event (AC 29.4.3) can be fully wired now since the DependencyUnblockedMsg handler exists. The add/remove/cycle-reject recording methods are created and ready for Story 29.3 to call when the TUI dependency management UI is implemented.
