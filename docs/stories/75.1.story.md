## Story

As a multiclaude operator,
I want Perplexity MCP server installed but disabled by default with a per-session toggle mechanism,
So that agents can perform web-grounded research when explicitly enabled without burning API tokens by default.

## Background

Perplexity's official MCP server (`@perplexity-ai/mcp-server`) provides 4 tools:
- `perplexity_search` — Direct web search with ranked results
- `perplexity_ask` — Conversational Q&A with web grounding (sonar-pro)
- `perplexity_research` — Comprehensive research reports (sonar-deep-research, expensive)
- `perplexity_reason` — Complex reasoning and problem-solving (sonar-reasoning-pro)

The operator has an API key. However, Perplexity usage costs real money (Sonar: $1/$1 per M tokens + $5/1K requests; Deep Research significantly more). It must be DISABLED by default to prevent agents from casually burning tokens. Enablement should be an explicit per-session decision.

## Acceptance Criteria

**AC1: MCP server configuration exists but is disabled**
**Given** the project's `.claude/settings.json` exists
**When** the Perplexity MCP configuration is added
**Then** it is present but in a disabled state (e.g., in `.claude/settings.local.json.example` or commented documentation)
**And** agents do NOT have access to Perplexity tools by default

**AC2: Per-session enablement mechanism documented**
**Given** the operator wants to enable Perplexity for a session
**When** the enablement process is documented
**Then** it explains how to:
  - Set `PERPLEXITY_API_KEY` environment variable
  - Enable the MCP server for the current session
  - Verify the tools are available
  - Disable when done

**AC3: Worker dispatch can optionally enable Perplexity**
**Given** some worker tasks benefit from web research
**When** the dispatch mechanism is designed
**Then** it documents how to enable Perplexity for specific workers:
  - Option A: Worker-specific `.claude/settings.local.json` in worktree
  - Option B: Environment variable passed at spawn time
  - Option C: Supervisor enables before spawning research workers

**AC4: Usage guidelines documented**
**Given** Perplexity costs real money
**When** usage guidelines are created
**Then** `docs/operations/perplexity-usage.md` explains:
  - Cost per tool (search: cheap, research: expensive)
  - When to use each tool
  - Budget awareness: check quota before heavy use
  - Which agents should have access (research workers, supervisor) vs not (merge-queue, pr-shepherd)

**AC5: MCP configuration template created**
**Given** the MCP server needs specific configuration
**When** a template is created
**Then** `.claude/settings.local.json.perplexity-example` contains:
  ```json
  {
    "mcpServers": {
      "perplexity": {
        "type": "stdio",
        "command": "npx",
        "args": ["-y", "@perplexity-ai/mcp-server"],
        "env": {
          "PERPLEXITY_API_KEY": "${PERPLEXITY_API_KEY}"
        }
      }
    }
  }
  ```

**AC6: npm/npx availability verified**
**Given** the MCP server requires npx
**When** the installation is tested
**Then** npx is available in the multiclaude environment
**And** `@perplexity-ai/mcp-server` installs and runs correctly

## Dev Notes

- The API key must NEVER be committed to the repo — use environment variable only
- `.claude/settings.local.json` is gitignored and per-machine — ideal for API keys
- npx caches packages after first download, so subsequent starts are fast
- The MCP server communicates via stdio — one process per Claude session
- Consider: should the supervisor memory startup checklist include a "Perplexity enabled: yes/no" status check?
- Marvel portability: MCP server configuration is per-project — the pattern of "disabled by default, enable per-session" applies to any costly external tool integration

## Implementation Notes

> [decision] 2026-03-29 — Disabled Perplexity by default using `.local.json.example` pattern rather than a feature flag.
> Alternative considered: settings.json with `"enabled": false` — rejected because it would still load the MCP server process.

> [risk] 2026-03-29 — Sonar Deep Research costs are significantly higher than standard search — accidental invocation could burn budget quickly.
> Impact: Documented cost tiers prominently in usage guide; recommend restricting deep-research to supervisor-only.

> [observation] 2026-03-29 — npx caches the MCP server after first download, making subsequent session starts near-instant.
