# Story 77.3: Rename docs-pass to skip-pass

## Status: Not Started

## Epic

Epic 77: CI Path Filtering Improvements

## References

- Research: `_bmad-output/planning-artifacts/ci-path-filtering-research.md` (PR #868)
- CI workflow: `.github/workflows/ci.yml`

## Story

As a contributor reading CI results,
I want the no-op CI job to be named `skip-pass` instead of `docs-pass`,
So that the job name accurately reflects that it covers scripts-only, story-only, and other non-code PRs — not just docs.

## Background

The current `docs-pass` job name was created when the only non-code PRs were documentation changes. With the addition of shell scripts, story files, and other non-Go content, the name is misleading. A scripts-only PR skipping CI shows "Docs Only" in the check, which confuses contributors.

## Acceptance Criteria

**AC1: Job ID renamed from `docs-pass` to `skip-pass`**
**Given** the `ci.yml` workflow
**When** the rename is applied
**Then** the job ID is `skip-pass` and the job name is `CI Skipped (no code changes)`

**AC2: Echo message updated**
**Given** the skip-pass job runs
**When** a non-code PR triggers it
**Then** the echo message reads `"No code changes — Go CI skipped."` (or similar neutral wording)

**AC3: No functional change to skip conditions**
**Given** the skip-pass job
**When** it evaluates whether to run
**Then** the `if:` condition remains identical to the current `docs-pass` job (PR event, code != true, docker != true)

## Tasks

### Task 1: Rename job ID and display name in `ci.yml`
Change `docs-pass:` to `skip-pass:` and update the `name:` field.

### Task 2: Update echo message
Change the step's `run: echo` to reflect the broader scope.
