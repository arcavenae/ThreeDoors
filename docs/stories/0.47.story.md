## Story

As a user installing alpha builds via Homebrew,
I want alpha versions to sort in chronological order,
So that `brew upgrade` always installs the newest build and `threedoors --version` reflects actual recency.

## Background

Alpha versions are generated on every push to main (`.github/workflows/ci.yml` line 235). The current format `0.1.0-alpha.YYYYMMDD.SHASHASHA` doesn't sort chronologically because the SHA suffix is random. Multiple same-day builds have unpredictable ordering per SemVer 2.0 §11.4.

The fix inserts a UTC `HHMMSS` timecode between the date and SHA, producing `0.1.0-alpha.YYYYMMDD.HHMMSS.SHASHASHA`. SemVer treats numeric identifiers with numeric comparison, so `143022 > 090015` sorts correctly.

## Acceptance Criteria

**AC-1: Version format includes timecode**
**Given** a push to main triggers the CI release job
**When** the version is generated
**Then** the version matches `0.1.0-alpha.YYYYMMDD.HHMMSS.SHASHASHA` format
**And** the tag matches `alpha-YYYYMMDD-HHMMSS-SHASHASHA` format

**AC-2: Chronological sorting**
**Given** two alpha releases on the same day
**When** compared using SemVer precedence rules
**Then** the later release sorts higher than the earlier release

**AC-3: Backward compatibility**
**Given** existing alpha releases in old format (`0.1.0-alpha.20260310.abc1234`)
**When** compared against new format releases from the same day
**Then** old-format releases sort lower (fewer identifiers = lower precedence per SemVer §11.4.4)

**AC-4: SHA traceability preserved**
**Given** a new-format alpha version string
**When** examined by a developer
**Then** the commit SHA prefix is still present as the last identifier

## Technical Notes

- Change is isolated to `.github/workflows/ci.yml` lines 235-236
- Use `$(date -u +%H%M%S)` for UTC time — same `date` invocation pattern already used for `%Y%m%d`
- Tag format uses hyphens (not dots) to match existing convention: `alpha-YYYYMMDD-HHMMSS-SHASHASHA`
- No changes needed to GoReleaser (only used for stable tagged releases)
- No changes needed to Homebrew formula template (accepts any version string)
- No changes needed to version display code (`internal/cli/version.go`, `internal/dist/version.go`)
