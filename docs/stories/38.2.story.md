# Story 38.2: Alpha Publishing Toggle

## Status: Done (PR #287)

**Epic:** 38 — Dual Homebrew Distribution
**Priority:** P1
**Estimate:** S (1-2 hours)

---

## User Story

As a repository maintainer, I want to control whether alpha formula updates are pushed to the Homebrew tap so that I can activate alpha publishing only when the infrastructure is validated.

## Context

PR #273 adds alpha formula publishing to the CI pipeline but has no mechanism to disable it. The tap push is hardcoded to always run after creating an alpha release. This story adds a `vars.ALPHA_TAP_ENABLED` toggle matching the existing `vars.SIGNING_ENABLED` pattern.

Party mode artifact: [Homebrew Dual Publish Course Correction](../../_bmad-output/planning-artifacts/homebrew-dual-publish-course-correction.md) (Decision CC-3)

## Acceptance Criteria

### AC-1: Toggle Gate on Formula Push
- [ ] The "Update alpha Homebrew formula" step in `ci.yml` is gated by `vars.ALPHA_TAP_ENABLED == 'true'`
- [ ] When `ALPHA_TAP_ENABLED` is not set or set to any value other than `'true'`, the formula push is skipped
- [ ] When `ALPHA_TAP_ENABLED` is `'true'`, the formula push runs as before

### AC-2: Alpha Release Always Created
- [ ] The GitHub alpha release creation is NOT gated by the toggle — releases are always created on push to main
- [ ] Only the tap formula push is controlled by the toggle

### AC-3: Formula Template Refactor (COULD-HAVE)
- [ ] If touching the formula generation step, consider refactoring from shell heredoc to a template file at `scripts/threedoors-a.rb.tmpl` using `envsubst`
- [ ] Template file is version-controlled and reviewable in PRs
- [ ] This is a COULD-HAVE improvement, not a blocker for this story

## Tasks

1. **Add toggle condition** — Add `if: vars.ALPHA_TAP_ENABLED == 'true'` to the formula push step in `ci.yml`
2. **Verify release creation is independent** — Confirm the release creation step has no dependency on the toggle
3. **Optionally refactor formula generation** — Replace heredoc with template file if scope allows
4. **Update story status** — Mark as Done with PR number

## Technical Notes

- The toggle follows the established `vars.SIGNING_ENABLED` pattern — repository variables set in GitHub Settings > Secrets and variables > Actions > Variables
- Default is OFF (not set = skip) — maintainer must explicitly enable
- The alpha release creation (GitHub pre-release) always runs regardless of toggle
- Only the tap formula push (`arcaven/homebrew-tap`) is gated

## Dependencies

- PR #273 (Story 38.1) — must be merged first so there is a formula push step to gate

## Out of Scope

- Changes to the stable formula publishing (GoReleaser controls that)
- Changes to the signing pipeline
