## Story

As a multiclaude operator,
I want clear documentation that the workspace window is the primary human interaction point,
So that I stop working in the supervisor window and avoid prompt injection conflicts.

## Background

The supervisor tmux window receives automated injections every 2 minutes (daemon wake nudges) plus message deliveries from agents. When the human operator types in the supervisor window, these injections corrupt input mid-keystroke. The workspace window (`tmux window 1`) is explicitly exempted from all automated injections — it's the intended human environment but is currently underutilized.

Root cause: `wakeAgents()` (daemon.go:447) and `routeMessages()` (daemon.go:387) both `continue` when `agent.Type == state.AgentTypeWorkspace`, making the workspace the only injection-free window.

## Acceptance Criteria

  - The supervisor window belongs to the supervisor Claude agent, not the human
