# Story 39.8: Command Autocomplete/Completion

## Status: Done (PR #381)

**Epic:** 39 — Keybinding Display System
**Priority:** P1
**Estimate:** M (3-5 hours)

---

## User Story

As a user, I want to see matching commands as I type in command mode, so that I can discover available commands and quickly select the right one without memorizing the full list.

## Context

The command palette (`:` prefix in SearchView) has 16+ commands but no discoverability — users must know or remember command names. Adding a dynamic completion list that filters as the user types would reduce friction and aid discovery.

Research artifact: [Global Command Mode Analysis](../../_bmad-output/planning-artifacts/global-command-mode-analysis.md)

**Charm ecosystem evaluation:** The `bubbles/list` component has built-in fuzzy filtering but is heavyweight for 16 items. A custom lightweight prefix-match implementation is more appropriate and consistent with the existing `filterTasks()` pattern.

## Acceptance Criteria

- [x] When `isCommandMode` is true and input is `:` alone, all commands are listed (serves as reference)
- [x] When input is `:` + characters, only commands matching the prefix are shown
- [x] Filtering is case-insensitive prefix match on command name
- [x] Suggestions update dynamically on each keystroke (including backspace)

### AC-2: Suggestion Display
- [x] Each suggestion shows: command name (styled/emphasized) and brief description
- [x] Format example: `  :add         Create a new task`
- [x] Suggestions render inline between the header and text input (consistent with search results pattern)
- [x] Maximum display: all 16 commands (no pagination needed for this count)
- [x] When no commands match the prefix, show "No matching commands" help text

### AC-3: Suggestion Navigation and Selection
- [x] Arrow keys (↑/↓) highlight suggestions
- [x] Tab completes the highlighted suggestion into the text input (fills command name)
- [x] Enter on a highlighted suggestion executes the command
- [x] Default highlight: first matching suggestion
- [x] Navigation wraps (down from last → first, up from first → last)

### AC-4: Command Registry
- [x] Commands are defined as a slice of structs with `name` and `description` fields
- [x] Registry is defined once (DRY with `executeCommand()` switch cases)
- [x] Adding a new command to the registry automatically makes it appear in suggestions

## Technical Notes

- Command registry: `var commands = []commandDef{{name: "add", desc: "Create a new task"}, ...}`
- Filter function: simple `strings.HasPrefix` loop — instant for 16 items
- Render in SearchView.View() when `isCommandMode` is true
- Navigation state: add `commandSelectedIndex int` to SearchView
- No new dependencies — uses existing `textinput`, `lipgloss`, stdlib only
