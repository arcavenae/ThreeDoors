## Story

As a developer collaborating remotely via SSH,
I want a wrapper script that sends a message and polls for replies,
So that I don't have to manually run `multiclaude message list` in a loop waiting for responses.

## Background

The multiclaude message system is polling-based — agents check for messages every 5-30 minutes. When a remote developer sends a message via `multiclaude message send supervisor "..."`, they must manually poll `multiclaude message list` to check for replies. This is tedious and error-prone.

A small shell script can automate this: send a message, poll at configurable intervals, display replies, and optionally auto-acknowledge. This reduces the friction of SSH-based collaboration significantly.

## Acceptance Criteria

**Given** the wrapper script `scripts/remote-collab.sh`
**When** a developer runs `./scripts/remote-collab.sh send supervisor "Review my branch feature-x"`
**Then** the script sends the message via `multiclaude message send` and prints the message ID

**Given** a sent message
**When** the developer runs `./scripts/remote-collab.sh wait-reply --timeout 300`
**Then** the script polls `multiclaude message list` every 10 seconds (configurable via `--interval`) and displays any new messages addressed to the sender

**Given** the `--timeout` flag
**When** the timeout elapses without a reply
**Then** the script exits with a non-zero status and prints a timeout message

**Given** the wrapper script
**When** a developer runs `./scripts/remote-collab.sh status`
**Then** the script displays the current multiclaude system status (agents, workers, pending messages)

**Given** the wrapper script invoked over SSH
**When** a developer runs `ssh host './scripts/remote-collab.sh send supervisor "task request"'`
**Then** it works identically to local invocation (no interactive prompts, no TTY requirements)

**Given** the script
**When** run with `--help`
**Then** it displays usage instructions for all subcommands (send, wait-reply, status, list)

## Technical Notes

- Shell script (bash), not Go. Lives at `scripts/remote-collab.sh`.
- Subcommands: `send`, `wait-reply`, `status`, `list` (list pending messages).
- `wait-reply` filters messages by timestamp (only messages newer than the send time).
- Default poll interval: 10 seconds. Configurable via `--interval`.
- Default timeout: 300 seconds (5 minutes). Configurable via `--timeout`.
- Must work non-interactively for SSH pipe usage.
- No dependencies beyond bash, multiclaude CLI, and standard Unix tools (jq optional for JSON parsing).
- Exit codes: 0 = success/reply received, 1 = error, 2 = timeout.
