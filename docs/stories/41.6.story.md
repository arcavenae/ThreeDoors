## Story

As a user in a constrained terminal (SSH, old Terminal.app, screen),
I want the TUI to gracefully degrade its colors based on my terminal's capabilities,
So that I don't see garbled output or missing colors.

## Background

ThreeDoors hardcodes 256-color (ANSI256) values throughout its styles. Lipgloss and termenv support automatic terminal color profile detection (TrueColor, ANSI256, ANSI 16-color, ASCII/no-color). By using `lipgloss.AdaptiveColor` and profile-aware color definitions, ThreeDoors can render correctly on any terminal without manual configuration. Most users won't notice any change (modern terminals support 256+ colors), but SSH sessions, CI output, and older terminals will render correctly.

## Acceptance Criteria

**Given** the application starts on a TrueColor terminal
**When** styles are applied
**Then** the full color palette is used (no visible change from current behavior)

**Given** the application starts on an ANSI256 terminal
**When** styles are applied
**Then** the full color palette is used (no visible change from current behavior)

**Given** the application starts on an ANSI 16-color terminal
**When** styles are applied
**Then** styles degrade gracefully to the nearest 16-color equivalents
**And** all content remains readable and visually distinct

**Given** the theme system (DoorTheme)
**When** adaptive colors are applied
**Then** theme colors adapt to the terminal's capability
**And** theme contrast ratios remain sufficient for readability

**Given** golden file tests (which already use ASCII color profile)
**When** adaptive colors are implemented
**Then** existing golden file tests continue to pass (they already test the lowest profile)

## Technical Notes

- Use `lipgloss.AdaptiveColor{Light: "...", Dark: "..."}` for light/dark terminal detection
- Use `lipgloss.CompleteAdaptiveColor{TrueColor: "...", ANSI256: "...", ANSI: "..."}` for full profile coverage
- Detect color profile at startup via `lipgloss.ColorProfile()` (returns termenv profile)
- Update styles in `internal/tui/styles.go` to use adaptive color definitions
- Update theme colors in `internal/tui/themes/` to support adaptive fallbacks
- Golden file tests already use `lipgloss.ColorProfile() == termenv.Ascii` — no change needed
- Focus on high-visibility elements: door borders, selection highlights, status colors
- Low-visibility elements (subtle separators, dim text) can use simple fallbacks
