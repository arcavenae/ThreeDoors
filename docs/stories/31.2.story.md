## User Story

As a ThreeDoors user, I want to press E on a task and create multiple subtasks in sequence without leaving expand mode, so that I can quickly decompose a task into actionable pieces.

## Context

The Expand feature currently exists but creates unlinked standalone tasks. This story enhances it to:
1. Set `ParentID` on created subtasks (using the field from Story 31.1)
2. Stay in `DetailModeExpandInput` after each subtask creation (sequential mode)
3. Show a running count of subtasks added
4. Only exit on Esc, not on Enter

**Architecture reference:** `_bmad-output/planning-artifacts/architecture-expand-fork.md` (Section 2)
**Party mode artifact:** `_bmad-output/planning-artifacts/party-mode-expand-fork-2026-03-08.md`
**Requirements:** FR120, FR121

### Current State

- `DetailModeExpandInput` mode exists at `internal/tui/detail_view.go:21`
- `handleExpandInput()` at `detail_view.go:333-359` captures text, emits `ExpandTaskMsg`
- `ExpandTaskMsg` already carries `ParentTask *core.Task` (defined in `messages.go:40-44`)
- Main model handler at `main_model.go:943-953` creates task via `NewTask()` — does NOT set ParentID
- Current behavior: Enter creates task AND exits expand mode

### Key Changes

- `handleExpandInput()`: On Enter, create subtask, clear input, increment counter, stay in expand mode. Only Esc exits.
- Main model `ExpandTaskMsg` handler: Set `newTask.ParentID = &msg.ParentTask.ID` before adding to pool
- Display prompt: "Subtask N added. Next subtask (Esc to finish):"
