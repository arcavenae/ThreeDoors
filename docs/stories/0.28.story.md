# Story 0.28: Issue Tracker File Structure, Authority Configuration & Initial Content

## Story

As the ThreeDoors envoy agent,
I want a local issue tracker file (`docs/issue-tracker.md`) with proper structure, authority tier configuration, and initial content,
So that I can track issue lifecycle, detect duplicates, and route issues based on reporter authority.

**Status:** Done (PR #255)

---

## Background

The envoy agent definition (`agents/envoy.md`) and comprehensive rules of behavior were established in PRs #227 and #232, including a 5-round party mode deliberation. The envoy needs a local state file as its "awareness backbone" ظ¤ a hybrid approach using a local markdown tracker validated against GitHub on each patrol cycle.

**Party Mode Artifact:** `_bmad-output/planning-artifacts/envoy-rules-of-behavior-party-mode.md`
**Agent Definition:** `agents/envoy.md`

**Key Decisions from Party Mode:**
- Tracker file at `docs/issue-tracker.md` (not JSON, not SQLite, not GitHub Projects)
- 8 metadata fields per issue (number+title, status, linked story, linked PRs, reporter, date reported, last envoy update, one-line state)
- Authority tiers stored in tracker file header as HTML comment
- Patrol watermark via `<!-- last-patrol: ... -->` HTML comment
- Recently Resolved section: last 50 closed issues, pruned at 90 days

## Acceptance Criteria

### Tracker File Structure

- **AC1:** `docs/issue-tracker.md` exists with the following sections:
  - HTML comment header with authority tiers and patrol watermark
  - `## Open Issues` table with 8 metadata columns
  - `## Recently Resolved` table for closed issues (last 50, 90-day prune window)
- **AC2:** Authority tiers are configured in the tracker header:
  ```
  <!-- authority-tiers:
    owner: [arcaven]
    contributors: []
  -->
  <!-- last-patrol: 2026-03-08T00:00:00Z -->
  ```
- **AC3:** Issue status values follow the defined lifecycle: `open` ظْ `triaged` ظْ `story-created` ظْ `pr-open` ظْ `resolved`
- **AC4:** Table columns match the party mode consensus: Issue # + Title | Status | Linked Story | Linked PR(s) | Reporter | Date Reported | Last Envoy Update | Current State

### Initial Content

- **AC5:** Any currently open GitHub issues are populated in the tracker with their current status
- **AC6:** Recently closed issues (last 50, within 90 days) are populated in the Recently Resolved section
- **AC7:** Issue-to-story and issue-to-PR linkages are populated for any issues that already have associated stories or PRs

### SOUL.md Alignment Reference

- **AC8:** A `## SOUL.md Alignment Reference` section is included in the tracker (or a linked file) documenting:
  - The three-category classification system (clearly aligned, clearly misaligned, gray area)
  - Common misalignment patterns table (from party mode consensus)
  - Polite decline template structure
- **AC9:** Each misalignment pattern includes the specific SOUL.md principle it conflicts with
