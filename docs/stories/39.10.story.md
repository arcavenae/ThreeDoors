# Story 39.10: Door View Inline Hints

## User Story

As a new user, I want to see key labels like `[a]`, `[w]`, `[d]` rendered directly on the three door frames, so that I immediately know which key opens which door without reading a separate help line.

## Context

This is the flagship use case for inline hints. The three doors are the primary interaction surface, and the key labels should feel like doorknob labels — part of the door, not a separate UI element. When a door is selected, its hint brightens to confirm the selection. The existing help text line at the bottom of DoorsView can be simplified or removed when hints are active.

Design reference: [Default Tooltips Party Mode](../../_bmad-output/planning-artifacts/default-tooltips-mode-party-mode.md)

## Acceptance Criteria

### AC-1: Door Frame Hints (Doorknob Metaphor)
- [ ] Left door shows `[a]` in bottom-right of door frame
- [ ] Middle door shows `[w]` in bottom-right of door frame
- [ ] Right door shows `[d]` in bottom-right of door frame
- [ ] Hints rendered via theme `Render()` hint parameter (Story 39.9)
- [ ] Hints only shown when `show_inline_hints` config is true

### AC-2: Selection State Awareness
- [ ] When a door is selected: selected door's hint brightens (ANSI 255 or bold)
- [ ] When a door is selected: unselected doors' hints dim along with the door
- [ ] When no door is selected: all hints at normal dim brightness

### AC-3: Additional Action Hints
- [ ] `[s]` re-roll hint shown near the doors area (footer region or between doors and help text)
- [ ] `[n]` add task hint shown near the doors area
- [ ] `[enter]` or `[space]` confirm hint shown when a door is selected
- [ ] These hints also conditioned on `show_inline_hints`

### AC-4: Help Text Simplification
- [ ] When inline hints are active: the long help text line is shortened or hidden (inline hints make it redundant)
- [ ] When inline hints are inactive: full help text line remains as fallback
- [ ] Footer includes `hints: on [type :hints to hide]` indicator when active

- [ ] All registered themes render door hints correctly

- [ ] Other themes: hint positioned appropriately per theme aesthetic
- [ ] Width-fallback to Classic still works with hints
