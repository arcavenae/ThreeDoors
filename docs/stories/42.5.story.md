## Story

As a maintainer,
I want third-party GitHub Actions pinned to SHA hashes and govulncheck running in CI,
So that tag mutation attacks cannot compromise our build pipeline and known CVEs in dependencies are caught automatically.

## Background

All GitHub Actions are currently pinned to major version tags (e.g., `@v6`, `@v9`). While standard practice, this allows tag mutation — a compromised maintainer could point `@v9` to malicious code. First-party GitHub actions (`actions/*`) are lower risk but third-party actions (`golangci/golangci-lint-action`, `dorny/paths-filter`, `goreleaser/goreleaser-action`) should be pinned to SHA hashes.

Additionally, `govulncheck` is not currently in the CI pipeline. Running it catches known CVEs in Go dependencies before they reach production.

## Acceptance Criteria

**AC-1: Third-party actions pinned to SHA**
**Given** the CI workflow files (`.github/workflows/ci.yml`, `release.yml`, `release-verify.yml`)
**When** referencing third-party GitHub Actions
**Then** they are pinned to full SHA hashes with a comment indicating the version
**Example:** `golangci/golangci-lint-action@abcdef123456 # v9`

**AC-2: First-party actions remain tag-pinned**
**Given** the CI workflow files
**When** referencing first-party GitHub Actions (`actions/checkout`, `actions/setup-go`, etc.)
**Then** they remain pinned to major version tags (acceptable risk for first-party)

**AC-3: govulncheck in CI**
**Given** a PR is submitted
**When** CI runs
**Then** `govulncheck ./...` runs as a quality gate step
**And** the build fails if govulncheck reports any known vulnerabilities

**AC-4: Renovate/Dependabot awareness**
**Given** Dependabot is configured for GitHub Actions updates (Story 0.24)
**When** a SHA-pinned action has a new version available
**Then** Dependabot can still detect and propose updates (Dependabot supports SHA pinning)

## Technical Notes

### Third-party actions to pin (with current SHAs):
- `golangci/golangci-lint-action@v9` → look up current SHA for v9
- `dorny/paths-filter@v3` → look up current SHA for v3
- `goreleaser/goreleaser-action@v7` → look up current SHA for v7

### First-party actions (keep tag-pinned):
- `actions/checkout@v6`
- `actions/setup-go@v6`
- `actions/upload-artifact@v7`
- `actions/download-artifact@v8`
- `actions/cache@v5`
- `actions/github-script@v8`
- `docker/setup-buildx-action@v4` (Docker official, low risk)
- `docker/build-push-action@v7` (Docker official, low risk)

### govulncheck setup:
- Install via `go install golang.org/x/vuln/cmd/govulncheck@latest`
- Run `govulncheck ./...` as a CI step after `go build`
- Consider adding as a separate job for parallelism
