# Story 78.2: Fix merge-queue Protected Paths and Detection Pattern

**Epic:** 78 — CODEOWNERS Documentation Drift Fix
**Status:** Not Started
**Priority:** P1 (root cause of incorrect status.needs-human labeling)
**Estimate:** XS (single file edit)

## Context

merge-queue.md (lines 17-21) contains a hardcoded list of protected paths AND a grep pattern used to detect CODEOWNERS-protected files in PRs. Both include four files that are NOT actually in CODEOWNERS:
- `ROADMAP.md`
- `docs/prd/epic-list.md`
- `docs/prd/epics-and-stories.md`
- `docs/decisions/BOARD.md`

**This is the ROOT CAUSE** of governance sync PRs being incorrectly labeled `status.needs-human`. When project-watchdog creates a PR touching ROADMAP.md, merge-queue's grep pattern matches it and blocks automated merging.

## Acceptance Criteria

- [ ] Protected paths list (line 17) updated to match actual CODEOWNERS: `SOUL.md`, `CLAUDE.md`, `.claude/`, `.env`, `.gitignore`, `.github/`, `agents/`, `_bmad/`
- [ ] Detection grep pattern (line 21) updated to match — remove `ROADMAP\.md`, `docs/prd/epic-list\.md`, `docs/prd/epics-and-stories\.md`, `docs/decisions/BOARD\.md`
- [ ] Add `\.env|\.gitignore|_bmad/` to the grep pattern
- [ ] Verify the grep pattern is syntactically correct (test with a sample PR if possible)
- [ ] No other references to the stale list exist in merge-queue.md

## Technical Notes

merge-queue.md is in `agents/` which IS CODEOWNERS-protected. This PR will require @skippy approval.

The corrected grep pattern should be:
```bash
gh pr diff <number> --name-only | grep -qE '^(SOUL\.md|CLAUDE\.md|\.claude/|\.env|\.gitignore|\.github/|agents/|_bmad/)'
```
