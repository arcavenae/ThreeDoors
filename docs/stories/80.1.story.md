**Epic:** 80 — Stream Deck Plugin (Elgato SDK)

## Context

The Stream Deck plugin lives in a separate repository from ThreeDoors. This story bootstraps the repo using the official Elgato CLI scaffolding tool, configures the TypeScript build pipeline, and implements the daemon client that communicates with `threedoors serve` (Epic 79) over Unix socket.

**Research:** [Stream Deck Research](../../_bmad-output/planning-artifacts/streamdeck-research.md) — Section 1 (SDK Architecture), Section 3 (Repository Structure), Section 6 (Tech Stack).

## Acceptance Criteria

- [ ] New repository `ArcavenAE/threedoors-streamdeck` created (private initially)
- [ ] Scaffolded via `streamdeck create` with TypeScript template
- [ ] `manifest.json` configured: UUID `com.arcavenae.threedoors`, SDK v3, macOS 13+, Node.js 20
- [ ] TypeScript build pipeline: `npm run build` compiles to `bin/plugin.js`
- [ ] `npm run watch` enables hot reload during development

- [ ] Client emits typed events for server notifications (`doors.changed`, `task.completed`)

- [ ] GitHub Actions CI: lint, build, validate
- [ ] README with development setup instructions

### Daemon Client Architecture

```typescript
class ThreeDoorsClient extends EventEmitter {
  private socket: net.Socket;
  private requestId: number = 0;
  private pending: Map<number, {resolve, reject, timer}>;

  connect(socketPath?: string): Promise<void>;
  request<T>(method: string, params?: object): Promise<T>;
  disconnect(): void;

  // Events: 'connected', 'disconnected', 'doors.changed', 'task.completed'
}
```
