# Story 64.3: Sync Transport Layer

## Status: Not Started

## Refined by Spike (ADR-0034)

Transport mechanism determined: Git-based sync via shared bare repository. The `SyncTransport` interface allows future alternatives, but the initial implementation uses Git.

## User Story

As a ThreeDoors user with multiple computers,
I want my task data to transfer reliably between machines,
So that I can work on any computer and see my latest tasks.

## Acceptance Criteria

- AC1: `SyncTransport` interface defined with `Push(ctx context.Context, changeset Changeset) error` and `Pull(ctx context.Context, since time.Time) (Changeset, error)` methods
- AC2: `GitSyncTransport` implements `SyncTransport` — initializes Git repo in `~/.threedoors/sync/`, manages the stage → commit → pull --rebase → push cycle
- AC3: Git remote configured via `threedoors sync init <remote-url>` — validates remote is reachable, clones/initializes
- AC4: Delta sync via Git — only changed files are transferred (Git's native behavior)
- AC5: Encryption in transit delegated to Git transport (SSH or HTTPS) — ThreeDoors validates the remote URL scheme
- AC6: Sync trigger: debounced 30s after last task change (file watcher on `~/.threedoors/tasks.yaml`) + manual via `threedoors sync` command
- AC7: `threedoors sync` CLI command shows: sync status (idle/syncing/error), last sync time, unpushed commits count, remote URL
- AC8: Sync progress surfaced in TUI status bar — reuse existing sync status infrastructure from Epic 44
- AC9: `GitSyncTransport` registered as a `GitSyncConnection` in Connection Manager — reuses state machine, health checking, and circuit breaker (ADR-0034)
- AC10: Network failures queue as unpushed Git commits — push retries on next sync cycle with exponential backoff via circuit breaker
- AC11: `.gitattributes` in sync repo configures merge strategies: `tasks.yaml merge=threedoors-task-merge`, `sessions.jsonl merge=union`
- AC12: Integration tests with two `t.TempDir()` directories simulating two devices syncing via a bare Git repo

## Technical Notes

- Git operations via `os/exec` calling `git` binary (avoid CGo dependency on libgit2)
- Sync repo is separate from ThreeDoors data dir: `~/.threedoors/sync/` contains the Git repo, files are copied to/from `~/.threedoors/` on sync
- Debounce uses `time.AfterFunc` reset on each file change — only triggers sync after 30s of quiet
- Connection Manager integration: `GitSyncConnection` state transitions: Disconnected → Connecting (clone/fetch) → Connected → Syncing (commit+push) → Connected
- Circuit breaker trips after 5 consecutive push/pull failures; probe interval 30s–30m
