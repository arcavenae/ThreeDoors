## Story

As a multiclaude operator,
I want handover events archived with timing metrics and anomaly tracking,
So that I can debug handover issues, tune thresholds, and monitor system health.

## Background

Each handover produces valuable operational data: how long the supervisor ran before degradation, what metrics triggered the handover, how long the handover took, whether the delta was received, and any anomalies. Archiving this data enables threshold calibration and process improvement.

**Party mode consensus (Session 4):** State files archived to `~/.multiclaude/handover/<repo>/history/` with timestamps. Write-once per shift — after handover, becomes an audit trail.

## Acceptance Criteria

**AC-1: State file archival**
**Given** a handover completes (normal or emergency)
**When** the incoming supervisor confirms ready
**Then** the current shift-state.yaml is copied to `~/.multiclaude/handover/<repo>/history/shift-state-<ISO-timestamp>.yaml`

**AC-2: Handover event log**
**Given** a handover completes
**When** recording the event
**Then** an entry is appended to `~/.multiclaude/handover/<repo>/handover-log.jsonl` containing:
- timestamp
- outgoing supervisor name
- incoming supervisor name
- handover type (normal/emergency)
- trigger metrics (JSONL size, compression count, message count)
- handover duration (seconds)
- delta received (boolean)
- anomalies (list of strings, empty if none)

**AC-3: History directory management**
**Given** archived state files accumulate
**When** the history directory exceeds 50 files
**Then** the oldest files beyond 50 are deleted (rolling window)

**AC-4: Handover summary command**
**Given** the handover log exists
**When** the user runs `multiclaude supervisor history` (or similar)
**Then** it displays the last N handover events with key metrics

**AC-5: Anomaly alerting**
**Given** handover frequency exceeds 3 per hour
**When** detected by the daemon
**Then** it logs a warning: "HIGH_HANDOVER_FREQUENCY: N handovers in the last hour. Consider increasing shift clock thresholds."

## Technical Notes

- Archive directory: `~/.multiclaude/handover/<repo>/history/`
- JSONL log: `~/.multiclaude/handover/<repo>/handover-log.jsonl`
- The summary command could be a new `multiclaude supervisor` subcommand or a simple shell script
- Anomaly detection runs as part of the daemon's refresh loop after each handover
- History retention: 50 state files is ~50 shifts, covering days to weeks of operation depending on session length
