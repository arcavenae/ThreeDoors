## User Story

As a developer,
I want a thin HTTP client for the ClickUp REST API v2,
So that the ClickUpProvider can read and update tasks without a third-party SDK dependency.

## Acceptance Criteria

- AC1: `Client` struct in `internal/adapters/clickup/clickup_client.go` with `NewClient(config AuthConfig) *Client`
- AC2: API token authentication via `Authorization: <api_token>` header
- AC3: `GetTasks(ctx, listID, page) ([]ClickUpTask, error)` method using `GET /api/v2/list/{list_id}/task`
- AC4: `UpdateTaskStatus(ctx, taskID, status) error` method using `PUT /api/v2/task/{task_id}`
- AC5: `GetSpaces(ctx, teamID) ([]ClickUpSpace, error)` method for hierarchy navigation
- AC6: `GetLists(ctx, folderID) ([]ClickUpList, error)` method for list discovery
- AC7: `GetAuthorizedUser(ctx) (*ClickUpUser, error)` method for health check (GET /api/v2/user)
- AC8: HTTP 429 handling: parse `Retry-After` header, return `*RateLimitError` (reuse pattern from Jira adapter)
- AC9: `ClickUpConfig` struct with `APIToken`, `TeamID`, `SpaceIDs`, `ListIDs`, `Assignee`, `PollInterval` fields
- AC10: Env var `CLICKUP_API_TOKEN` takes precedence over config file `api_token` when both are set
- AC11: Config validation: at least one of `SpaceIDs` or `ListIDs` must be specified
- AC12: Unit tests using `httptest.NewServer` with canned ClickUp API responses
- AC13: No third-party dependencies beyond stdlib

## Technical Notes

- Use `net/http` + `encoding/json` — no third-party ClickUp SDK
- Auth: `Authorization: <api_token>` on all requests (ClickUp uses plain token, not Bearer)
- ClickUp API v2 base URL: `https://api.clickup.com/api/v2`
- Rate limits: 100 requests per minute per token — respect `Retry-After` header on 429
- ClickUp hierarchy: Workspace → Space → Folder → List → Task
- Pagination: page-based (`page` query param, 0-indexed)
- Accept interfaces, return concrete types: `NewClient(config AuthConfig) *Client`
- Context as first parameter on all methods
- Reuse `RateLimitError` pattern from `internal/adapters/jira/`
