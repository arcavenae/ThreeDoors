# Story 79.5: Real-Time Event Notifications

**Epic:** 79 — ThreeDoors Daemon/Server Mode
**Status:** Not Started
**Priority:** P2
**Estimate:** S (event bus, subscription management)

## Context

Stream Deck buttons need to update their display when doors change — whether from the daemon itself, another client, or external file changes detected by `TaskProvider.Watch()`. Without push notifications, clients would need to poll `doors.get` on a timer, which is wasteful and laggy.

This story adds a server-initiated notification system using JSON-RPC 2.0 notifications (requests without an `id` field, per spec).

## Acceptance Criteria

- [ ] `events.subscribe` method registers the connection for event notifications
- [ ] `events.unsubscribe` removes the connection from notifications
- [ ] `doors.changed` notification sent to all subscribers when doors change (refresh, select, complete, skip, snooze)
- [ ] `doors.changed` payload includes the new door set (same format as `doors.get` response)
- [ ] `task.completed` notification sent when any task is marked complete (includes task ID and text)
- [ ] `session.stats` notification sent periodically (configurable interval, default 60s) with session metrics
- [ ] `provider.changed` notification sent when `TaskProvider.Watch()` emits a `ChangeEvent` (external file modification)
- [ ] Subscriber cleanup on disconnect (no leaked subscriptions)
- [ ] Slow/blocked subscribers don't block other subscribers or the daemon (buffered channels with drop policy)
- [ ] Tests: subscribe/unsubscribe, event delivery, slow subscriber handling, disconnect cleanup

## Technical Notes

### Event Bus Architecture

```go
type EventBus struct {
    mu          sync.RWMutex
    subscribers map[*conn]chan Event
}
```

Each subscriber gets a buffered channel (capacity 16). If the buffer fills (slow client), the oldest event is dropped — the next `doors.changed` will contain the full current state anyway.

### JSON-RPC Notifications

Per JSON-RPC 2.0, notifications are requests without an `id`. The server sends them unilaterally:

```json
{"jsonrpc": "2.0", "method": "doors.changed", "params": {"doors": [...]}}
```

### Watch Integration

Wire `TaskProvider.Watch()` into the event bus. When external changes are detected (e.g., user edits YAML file directly), the daemon refreshes its door state and notifies subscribers.
