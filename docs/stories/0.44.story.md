# Story 0.44: Scoped Label Migration — Rename and Create GitHub Labels

## Status: Done (PR #513)

## Story

As the ThreeDoors project maintainer,
I want all GitHub labels migrated to scoped `.` separator format with the finalized 27-label taxonomy,
So that agents and humans have consistent, queryable, namespaced labels for issue and PR management.


---

## Background

Party mode deliberation (P-005, 3 rounds) produced a finalized 27-label taxonomy using `.` as the scope separator. The consumer challenge in Round 2 trimmed the initial 35 labels to 27 by cutting labels that duplicated GitHub-native state or lacked concrete agent consumers. Decisions D-106 through D-111 document the rationale.

**Party Mode Artifact:** `_bmad-output/planning-artifacts/scoped-labels-party-mode.md`
**Research Artifact:** `_bmad-output/planning-artifacts/scoped-labels-research.md`

**Key Decisions:**
- D-106: `.` as label scope separator
- D-107: 9 scopes, 27 total labels
- D-108: `status.do-not-merge` for merge-queue hard stop
- D-109: Agent labels limited to `agent.envoy` + `agent.worker`
- D-110: Migration via rename-first strategy (preserves label-issue associations)
- D-111: Implementation as separate story from research PR

## Acceptance Criteria

### Phase 1: Rename Existing Colon-Separated Labels (11 renames)

- **AC1:** All `triage:*` labels renamed to `triage.*`:
  - `triage:new` → `triage.new`
  - `triage:in-progress` → `triage.in-progress`
  - `triage:needs-info` → `triage.needs-info`
  - `triage:complete` → `triage.complete`
- **AC2:** All `priority:*` labels renamed to `priority.*` (lowercase):
  - `priority:P0` → `priority.p0`
  - `priority:P1` → `priority.p1`
  - `priority:P2` → `priority.p2`
  - `priority:P3` → `priority.p3`
- **AC3:** All `scope:*` labels renamed to `scope.*`:
  - `scope:in-scope` → `scope.in-scope`
  - `scope:out-of-scope` → `scope.out-of-scope`
  - `scope:needs-decision` → `scope.needs-decision`

### Phase 2: Rename Unscoped Labels (8 renames)

- **AC4:** Unscoped labels renamed to scoped equivalents:
  - `bug` → `type.bug`
  - `enhancement` → `type.feature`
  - `documentation` → `type.docs`
  - `question` → `type.question`
  - `infrastructure` → `type.infra`
  - `duplicate` → `resolution.duplicate`
  - `needs-human-input` → `status.needs-human`
  - `fast-track` → `process.fast-track`

### Phase 3: Create New Labels (6 new)

- **AC5:** New labels created with correct colors and descriptions:
  - `status.blocked` (#D93F0B) — "Blocked on dependency or decision"
  - `status.stale` (#BFD4F2) — "No activity past staleness threshold"
  - `status.do-not-merge` (#B60205) — "Must not merge even if CI passes"
  - `agent.envoy` (#0E8A16) — "Envoy is triaging or responsible"
  - `agent.worker` (#0E8A16) — "Worker agent actively implementing"
  - `contrib.good-first-issue` (#7057ff) — "Good for newcomers to the project"
  - `contrib.help-wanted` (#7057ff) — "Community contributions welcome"

### Phase 4: Clean Up Obsolete Labels

- **AC6:** `multiclaude` label deleted (replaced by `agent.envoy` and `agent.worker`)
- **AC7:** `ux` label deleted (absorbed into `type.feature` per SL-003)

### Phase 5: Color Standardization

- **AC8:** All renamed labels updated to match the finalized color scheme:
  - `type.*` → `#0075ca`
  - `triage.*` → gradient (`#C2E0F4`, `#6FBAED`, `#BFD4F2`, `#1D76B5`)
  - `priority.*` → heat map (`#B60205`, `#D93F0B`, `#FBCA04`, `#D4C5F9`)
  - `scope.*` → traffic light (`#0E8A16`, `#E11D48`, `#FEF2C0`)
  - `status.*` → per-label (see party mode artifact)
  - `resolution.*` → `#cfd3d7`
  - `process.*` → `#C5DEF5`

### Verification

- **AC9:** `gh label list` shows exactly 27 labels (+ `dependencies` label from Renovate = 28 total)
- **AC10:** No issues or PRs lost label associations during migration (spot-check 5 issues that had labels before migration)
