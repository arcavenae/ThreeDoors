## Story

As a TUI user,
I want to type `:extract` to extract tasks from text, files, or clipboard and review them before importing,
So that I can quickly convert unstructured text into actionable tasks without leaving the TUI.

## Background

The extraction TUI flow is the primary user-facing integration for LLM services. It follows ThreeDoors' philosophy: user-initiated, transparent, reversible, optional.

**Flow:** `:extract` → source picker (file/clipboard/paste) → spinner → review screen with toggle/edit → import selected → flash message → return to doors.

**Party mode decision (S4-D2):** TUI commands `:extract`, `:enrich`, `:breakdown` via command palette.
**Party mode decision (S4-D8):** Review-then-import for all LLM outputs.
**Party mode decision (S4-D6):** Esc cancels in-progress LLM operations.

## Acceptance Criteria

**AC-1: `:extract` command registered**
**Given** the TUI command palette
**When** user types `:extract`
**Then** the extraction flow begins

**AC-2: Source selection prompt**
**Given** the extraction flow starts
**When** the source picker appears
**Then** user can choose: [f]ile, [c]lipboard, [p]aste
**And** file prompts for a path, clipboard reads via pbpaste, paste opens a text input area

**AC-3: Loading spinner with cancel**
**Given** text is submitted for extraction
**When** the LLM is processing
**Then** a spinner is displayed with "Extracting tasks..."
**And** pressing Esc cancels the operation and returns to previous view

**AC-4: Review screen displays extracted tasks**
**Given** extraction completes successfully
**When** the review screen appears
**Then** extracted tasks are listed with: index, toggle checkbox, text, effort estimate
**And** the source and backend name are shown at the bottom

**AC-5: Task selection and editing**
**Given** the review screen
**When** user interacts
**Then** Space toggles selection on current task
**And** A selects all, N selects none
**And** E opens inline editor for the current task (text, effort, tags)
**And** j/k or arrows navigate between tasks

**AC-6: Import selected tasks**
**Given** tasks are selected
**When** user presses Enter
**Then** selected tasks are added to the task pool
**And** a flash message shows "Imported N tasks from [source]"
**And** user returns to the doors view

**AC-7: Empty/error states**
**Given** extraction returns no tasks
**When** the result is shown
**Then** a message says "No actionable tasks found in this text"
**Given** LLM is unavailable
**When** `:extract` is triggered
**Then** a message says "LLM service unavailable" with setup instructions

**AC-8: Latency target**
**Given** a short text input (<1KB)
**When** extraction runs
**Then** results appear within 5 seconds (S4-D7)
