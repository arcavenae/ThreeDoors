# Story 35.3: Modern Theme — Door-Like Proportions

## Status: Done (PR #234)

**Epic:** 35 — Door Visual Appearance — Door-Like Proportions
**Depends On:** 35.1 (PR #226)

## User Story

As a user,
I want the Modern theme to render doors with minimalist door-like proportions,
So that Modern doors clearly read as sleek, minimal doors.

## Acceptance Criteria

**Given** the Modern theme render function
**When** rendering a door with height >= MinHeight (12)
**Then:**

- AC1: The door uses portrait orientation with heavy top/bottom bars (`━━━`)
- AC2: A minimalist panel line (`─`) divides upper and lower panels at ~45% height
- AC3: A minimalist handle (`○`) is rendered on the right side at ~60% height
- AC4: The door number is centered in the top bar
- AC5: Selected state uses heavy frame characters (`┃`) as currently implemented

**Given** the Modern theme render function
**When** rendering a door with height < MinHeight (12) or height == 0
**Then:**

- AC6: Falls back to compact rendering (current card-like behavior)

**Given** golden file tests for the Modern theme
**When** tests run at minimum, standard, and wide heights
**Then:**

- AC7: Golden files match expected door-like output
- AC8: Both selected and unselected states are tested

## Technical Notes

- Follow the same DoorAnatomy pattern used in Classic (35.2)
- Modern uses `─`/`│` unselected and `━`/`┃` selected (existing pattern)
- Handle character is `○` (minimalist circle) instead of Classic's `●` (filled circle)
- Panel divider uses thin `─` even in selected state for minimalist look
- Compact fallback preserves existing modern card behavior
