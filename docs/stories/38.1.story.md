# Story 38.1: Alpha Homebrew Formula (`threedoors-a`)

**Epic:** 38 — Dual Homebrew Distribution
**Status:** Done (PR #273)
**Priority:** P1
**Estimate:** M (3-5 hours)

---

## User Story

As a ThreeDoors user, I want to install the alpha channel via `brew install arcaven/tap/threedoors-a` so that I can get the latest features on every push to main, independent of stable releases.

## Context

PR #262 established GoReleaser for stable releases (`threedoors` formula, updated on `v*` tags). The existing CI already builds alpha binaries on every push to main, but they are only available as GitHub release downloads. This story adds a parallel Homebrew formula (`threedoors-a`) that auto-updates on every main push.

## Acceptance Criteria

### AC-1: Alpha Binary Naming
- [x] CI builds produce binaries named `threedoors-a-darwin-arm64`, `threedoors-a-darwin-amd64`, `threedoors-a-linux-amd64`
- [x] Binaries are built from the same `./cmd/threedoors` entrypoint as stable builds
- [x] Alpha version string injected via ldflags: `0.1.0-alpha.YYYYMMDD.SHA7`

### AC-2: Alpha Formula in Tap
- [x] `Formula/threedoors-a.rb` exists in `arcaven/homebrew-tap`
- [x] Formula class is `ThreedoorsA` (valid Ruby class name)
- [x] Formula installs binary as `threedoors-a` to `bin/`
- [x] Formula includes `test do` block verifying `--version` output
- [x] Formula includes `on_arm`, `on_intel`, and `on_linux` blocks
- [x] Formula passes `ruby -c` syntax validation

### AC-3: CI Auto-Update
- [x] On every push to main, CI generates/updates `threedoors-a.rb` with correct version, URLs, and SHA256 checksums
- [x] CI pushes updated formula to `arcaven/homebrew-tap` using `HOMEBREW_TAP_TOKEN`
- [x] Formula syntax is validated (`ruby -c`) before pushing to tap
- [x] If push fails, the old formula remains valid (atomic update)

### AC-4: Apple Code Signing
- [x] `threedoors-a-darwin-*` binaries are signed with Developer ID certificate
- [x] Code signing uses distinct identifier: `com.arcaven.threedoors-a`
- [x] Signed binaries are notarized (when `SIGNING_ENABLED=true`)

### AC-5: No Conflicts with Stable
- [x] `brew install arcaven/tap/threedoors` and `brew install arcaven/tap/threedoors-a` can both be installed simultaneously
- [x] No `conflicts_with` or `keg_only` directives needed
- [x] `brew upgrade` updates each formula independently
- [x] Stable installs `/usr/local/bin/threedoors`, alpha installs `/usr/local/bin/threedoors-a`

### AC-6: Channel Identifier in Version Output
- [x] `threedoors-a --version` outputs `ThreeDoors (alpha) v0.1.0-alpha.YYYYMMDD.SHA7`
- [x] `threedoors --version` (stable) output unchanged (no channel label)
- [x] Channel injected via `-X main.channel=alpha` ldflag

### AC-7: Alpha GitHub Release Includes New Binaries
- [x] Alpha GitHub releases include `threedoors-a-darwin-arm64`, `threedoors-a-darwin-amd64`, `threedoors-a-linux-amd64`
- [x] Existing `threedoors-*` binaries may optionally be retained for backward compat

## Tasks

1. **Modify `ci.yml` `build-binaries` job** — Add `threedoors-a-*` binary builds
2. **Modify `ci.yml` `sign-and-notarize` job** — Sign `threedoors-a-*` with distinct identifier
3. **Add `main.channel` ldflag** — In `cmd/threedoors/main.go`, display channel in `--version`
4. **Add tap update step to `ci.yml` `release` job** — Generate and push `threedoors-a.rb`
5. **Create formula template** — Script or Go template for generating `threedoors-a.rb`
6. **Verify end-to-end** — `brew install arcaven/tap/threedoors-a` works after CI push
7. **Update story status** — Mark as Done with PR number

## Technical Notes

- The alpha binary uses the same Go source as stable -- only the output name and ldflags differ
- Apple codesign identifier is independent of filename; no signing constraints on naming
- The tap repo already has write access configured via `HOMEBREW_TAP_TOKEN` secret
- Alpha version `0.1.0-alpha.YYYYMMDD.SHA7` is already generated by CI
- GoReleaser is NOT used for alpha builds -- only for stable `v*` tag releases
