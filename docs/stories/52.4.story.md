## Story

As the supervisor,
I want clearly documented criteria for when the envoy should recommend BMAD party mode escalation,
So that complex issues get multi-agent deliberation while routine issues are handled efficiently.

## Background

Layer 3 is the final firewall stage — reserved for issues requiring architectural review, multi-perspective analysis, or direction-level decisions. Currently, the envoy escalates to the supervisor but doesn't distinguish between "supervisor can decide alone" and "this needs party mode." This story formalizes the criteria for recommending BMAD deliberation.

The key insight: the envoy doesn't invoke party mode (it can't — that's supervisor authority). Instead, it recommends party mode in its escalation message with specific reasons and suggested agent participants.

## Acceptance Criteria

**Given** the Layer 3 criteria documentation
**When** the envoy escalates an issue to the supervisor
**Then** the escalation message classifies the issue into one of two categories:
- **Supervisor-only:** Issue can be resolved by supervisor decision alone
- **BMAD recommended:** Issue warrants multi-agent party mode deliberation

**Given** the BMAD recommendation criteria
**When** an issue matches any of the following
**Then** the envoy recommends party mode:
- Feature request estimated at >3 stories (new epic scope)
- Request that would change project architecture or introduce new patterns
- Gray-area direction request from a contributor or owner
- Issue that reveals a systemic problem (not just a point fix)
- Bug report suggesting a fundamental design flaw
- Any issue where 3+ agent perspectives would add value

**Given** the BMAD recommendation
**When** the envoy sends the escalation message
**Then** the message includes:
- Which criteria were met (specific, not vague)
- Suggested participating agents with rationale (e.g., "Architect + PM + Dev for architecture change")
- What question(s) the party mode should address
- The envoy's own preliminary assessment

**Given** the supervisor-only criteria
**When** an issue matches the following
**Then** the envoy escalates WITHOUT a party mode recommendation:
- Scope decision on a well-defined feature (in-scope vs out-of-scope)
- Priority override (reporter says P0, envoy assesses P2)
- Routine story creation from a triaged bug or small enhancement
- Issue closure confirmation (envoy recommends, supervisor approves)

**Given** the agent participation guide
**When** party mode is recommended
**Then** the documentation maps issue types to relevant agents:
- Architecture/design: Architect + PM + Dev
- User-facing feature: UX + PM + QA
- Security/reliability: Architect + QA + Dev
- Direction/strategy: PM + Analyst + Innovation Strategist
- Testing/quality: QA + Test Architect + Dev
