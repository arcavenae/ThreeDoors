## Description

The product scope document (`docs/prd/product-scope.md`) has 16 "phases" with chaotic decimal numbering (1, 2, 3, 3.5, 3.5+ ×3, 4 ×2, 4+, 4.5, 5 ×2, 5.5, 5.5+, 6, 6+, 7, 8). Two Phase 4 entries and two Phase 5 entries contain overlapping content. Party mode consensus: consolidate to 7 integer-only phases with `###` sub-sections for thematic grouping within phases.

## Acceptance Criteria

### AC1: Phase numbering uses integers only
- [ ] All `##` phase headings use integer numbers (Phase 1 through Phase 7)
- [ ] No decimal phases (3.5, 4+, 5.5, etc.) exist in the document
- [ ] Each phase heading includes its epic list: `## Phase N: Title (Epics X, Y, Z)`

### AC2: Duplicate content merged
- [ ] The two Phase 4 entries (lines ~139 and ~157) merged into a single Phase 4
- [ ] The two Phase 5 entries (lines ~173 and ~209) merged into a single Phase 5
- [ ] No content lost during merge — all scope items from both entries preserved
- [ ] No content duplicated — overlapping items appear exactly once

### AC3: Thematic grouping correct
- [ ] Each epic maps to exactly one phase per the table above
- [ ] Epics within each phase are thematically coherent (integration epics in Phase 4, visual polish in Phase 5, etc.)
- [ ] `###` sub-sections used within phases that cover multiple themes (e.g., Phase 5 might have "Feature Richness" and "Visual Polish" sub-sections)

### AC4: In Scope / Out of Scope structure preserved
- [ ] Each phase retains its "In Scope" items list
- [ ] Each phase retains its "Out of Scope" items where applicable
- [ ] Phase separators (`---`) between each phase

### AC5: Document is self-consistent
- [ ] No orphaned epic references (every epic mentioned exists in epic-list.md)
- [ ] No missing epics (every epic in epic-list.md that has scope content is represented)
- [ ] Phase 7 (Future) correctly identifies deferred/icebox items
