## User Story

As a ThreeDoors user, I want to press F on a task to create a variant that preserves the original's text, context, effort, and tags while starting fresh, so that I can explore alternative approaches to the same work.

## Context

The Fork feature currently creates a new task with only the original's text copied (`core.NewTask(dv.task.Text)`). This story replaces that with a proper `ForkTask` factory that preserves relevant fields, resets others, adds provenance tracking via a note, and creates an enrichment DB cross-reference.

**Architecture reference:** `_bmad-output/planning-artifacts/architecture-expand-fork.md` (Section 3)
**Party mode artifact:** `_bmad-output/planning-artifacts/party-mode-expand-fork-2026-03-08.md`
**Requirements:** FR125, FR126

### Current State

- Fork at `internal/tui/detail_view.go:194-196`: `core.NewTask(dv.task.Text)` ‚Üí emits `TaskAddedMsg`
- Only copies text ‚Äî loses context, effort, tags
- No provenance tracking
- No enrichment cross-reference

### Target State

- `core.ForkTask(original *Task) *Task` factory function
- **Preserves:** Text, Context, Effort, Tags (defensive copy)
- **Resets:** Status ‚Üí todo, Blocker ‚Üí "", Notes ‚Üí empty, Timestamps ‚Üí `time.Now().UTC()`
- **Adds:** Note "Forked from: [truncated original text]" (truncate at 60 chars)
- **Does NOT copy:** ParentID (fork is independent)
- New `TaskForkedMsg` replaces `TaskAddedMsg` for fork actions
- Main model handler creates enrichment DB cross-reference (`forked-from` relationship)
