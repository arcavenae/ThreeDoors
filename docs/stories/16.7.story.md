# Story 16.7: Polish & TestFlight Distribution

**Status:** Draft
**Epic:** 16 — iPhone Mobile App (SwiftUI)
**Depends on:** Stories 16.1, 16.2, 16.3, 16.4, 16.5, 16.6

## User Story

As a developer,
I want the app polished and distributed via TestFlight,
So that it can be tested on real devices before wider release.

## Acceptance Criteria

**Given** all core functionality is implemented (Stories 16.1-16.6)
**When** the app is polished and prepared for distribution
**Then:**

- AC1: App icon designed and configured — Three Doors motif, all required sizes in asset catalog
- AC2: Launch screen configured with ThreeDoors branding (logo + name, minimal design)
- AC3: Dark mode support — all views adapt to system appearance setting (light/dark)
- AC4: Accessibility: VoiceOver labels on all interactive elements (cards, buttons, gestures)
- AC5: Accessibility: Dynamic Type support — text scales with user's system text size preference
- AC6: App configured in App Store Connect for TestFlight distribution
- AC7: Privacy labels configured: "Data Not Collected" (tasks stay in Apple Notes, no tracking)
- AC8: TestFlight build uploaded and available for testing
- AC9: Minimum iOS version validated — app runs on iOS 17+
- AC10: No crashes verified across iPhone SE (smallest screen) through iPhone 16 Pro Max (largest screen)

## Quality Gate

- QG1: `xcodebuild build` succeeds with zero warnings
- QG2: `xcodebuild test` passes — all tests green
- QG3: Accessibility audit passes (Xcode Accessibility Inspector)
- QG4: Dark mode renders correctly for all views
- QG5: App runs without crashes on minimum (iOS 17) and latest iOS versions
- QG6: Scope-checked against this story — no extras

## Technical Notes

- App icon: use SF Symbols or simple geometric design for the three doors motif — 1024x1024 master, Xcode generates all sizes
- Launch screen: use SwiftUI `LaunchScreen` storyboard or Info.plist configuration — keep it minimal
- Dark mode: use SwiftUI semantic colors (`Color.primary`, `Color.secondary`, `.background`) and `@Environment(\.colorScheme)`
- VoiceOver: add `.accessibilityLabel()` and `.accessibilityHint()` to all interactive elements; test with VoiceOver enabled in Simulator
- Dynamic Type: use `.font(.body)`, `.font(.headline)` etc. (SwiftUI text styles) rather than fixed font sizes
- TestFlight requires Apple Developer Program membership ($99/year) — coordinate with project owner
- Privacy labels: straightforward since ThreeDoors doesn't collect, track, or share user data
- Archive and upload via Xcode Organizer or `xcodebuild archive` + `altool` in CI
- Test matrix: iPhone SE 3rd gen, iPhone 14, iPhone 15 Pro, iPhone 16 Pro Max (covers small, medium, large, extra-large)

## Dependencies

- All previous stories (16.1-16.6) must be complete
- Apple Developer Program membership for TestFlight upload
- App Store Connect account configuration
