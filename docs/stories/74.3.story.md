## Story

As a project maintainer,
I want every AI-generated story, commit, and PR to carry an autonomy level tag (L0-L4),
So that the provenance of all work is traceable and the dark factory has a foundation for tracking AI vs human contributions.

## Background

The dark factory model (R-003) requires knowing who did what and at what autonomy level. Provenance tagging introduces a 5-level spectrum:

| Level | Name | Description | Example |
|-------|------|-------------|---------|
| L0 | Human-only | No AI involvement | Human writes code in editor |
| L1 | AI-assisted | Human directs, AI helps | Human asks Claude for help, reviews every change |
| L2 | AI-paired | Collaborative | /implement-story with human oversight |
| L3 | AI-autonomous | AI works independently, human reviews PR | Worker implements story, human merges |
| L4 | AI-full | AI end-to-end, no human review | Dark factory variant (future) |

Per Q-C-007: Mandatory for AI-generated work, opt-in for human. This means all multiclaude worker output MUST carry provenance tags. Human commits may optionally include them.

## Acceptance Criteria

**AC1: Story file provenance field defined**
**Given** story files follow a standard format
**When** a provenance field is added
**Then** story files include:
  ```markdown
  ## Provenance
  - **Autonomy Level:** L3 (AI-autonomous)
  - **Implementation Agent:** worker/<name>
  - **Review:** Human PR review required
  ```

**AC2: PR label for autonomy level**
**Given** GitHub labels exist for the project
**When** provenance labels are created
**Then** labels exist for:
  - `provenance.L0` (human-only)
  - `provenance.L1` (AI-assisted)
  - `provenance.L2` (AI-paired)
  - `provenance.L3` (AI-autonomous)
  - `provenance.L4` (AI-full)
**And** merge-queue applies the appropriate label based on PR source

**AC3: Commit message provenance trailer**
**Given** the commit message format includes story references
**When** provenance is added to commits
**Then** AI-generated commits include a `Provenance: L3` trailer (or appropriate level)
**And** the trailer is added by the worker agent, not post-hoc

**AC4: Worker agent definition updated**
**Given** worker agents create PRs and commits
**When** the worker agent definition is updated
**Then** it instructs workers to:
  - Add provenance level to story file after implementation
  - Include `Provenance: L3` trailer in commit messages
  - Apply the appropriate `provenance.*` label to PRs

**AC5: merge-queue provenance validation**
**Given** merge-queue validates PR metadata
**When** a PR from a worker branch (`work/*`) is processed
**Then** merge-queue checks for a `provenance.*` label
**And** warns (not blocks) if missing

**AC6: Provenance specification documented**
**Given** the provenance system is defined
**When** documentation is created
**Then** `docs/operations/provenance.md` explains:
  - The L0-L4 spectrum with examples
  - Where provenance appears (story files, commits, PR labels)
  - Mandatory vs opt-in requirements
  - How dark factory will use provenance data
