# Story 23.10: Shell Completions and Interactive Doors Mode

## Status: Done (PR #187)

**Epic:** 23 — CLI Interface
**Depends on:** Story 23.4

## User Story

As a CLI power user,
I want shell completions for all commands and an interactive doors selection mode,
So that I can efficiently use the CLI with tab completion and choose doors interactively.

## Acceptance Criteria

**Given** all CLI commands from previous stories are in place
**When** shell completions and interactive mode are implemented
**Then:**

- AC1: `threedoors completion bash` outputs bash completion script
- AC2: `threedoors completion zsh` outputs zsh completion script
- AC3: `threedoors completion fish` outputs fish completion script
- AC4: Completions cover all subcommands, flags, and status/type/effort enum values
- AC5: `threedoors doors --interactive` prompts the user to pick a door (1, 2, or 3) with arrow key selection
- AC6: Interactive mode auto-disabled when stdout is not a TTY (piped) — falls back to non-interactive
- AC7: Installation instructions printed with `threedoors completion --help`
- AC8: Cobra's built-in completion generation used — no custom completion logic

## Quality Gate

- All standard checks pass (see CLAUDE.md)
- QG5: Scope-checked against this story — no extras

## Technical Notes

- Cobra has built-in `GenBashCompletion`, `GenZshCompletion`, `GenFishCompletion` methods
- Custom completions for enum values (status, type, effort) use `RegisterFlagCompletionFunc`
- Interactive mode can use a simple `bufio.Scanner` prompt — no need for full Bubbletea
- TTY detection: if `!term.IsTerminal(int(os.Stdout.Fd()))`, force non-interactive
- Reference: `../../_bmad-output/planning-artifacts/cli-interface-design-research.md` sections 4, 8
