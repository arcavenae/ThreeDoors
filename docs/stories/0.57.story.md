# Story 0.57: t.Helper() Audit — Add t.Helper() to All Test Helper Functions

## Status: Done (PR #TBD)

**Epic:** 0 — Infrastructure & Process
**Priority:** P1
**Depends On:** None

## User Story

As a developer,
I want all test helper functions to call `t.Helper()`,
So that test failure messages report the caller's line number instead of the helper's, making failures faster to diagnose.

## Context

The state-of-testing audit (TEA report, 2026-03-09) found only 23% of test files use `t.Helper()`. Many test helper functions (especially in adapters and TUI packages) would benefit from adding `t.Helper()` to improve failure reporting. This is a low-effort, high-payoff improvement across the entire test suite.

## Acceptance Criteria

- AC1: Every function in `*_test.go` files that accepts `*testing.T` or `*testing.B` as a parameter and is NOT itself a `Test*`/`Benchmark*`/`Fuzz*` function calls `t.Helper()` as its first statement
- AC2: Audit covers all packages: `internal/cli/`, `internal/core/`, `internal/tui/`, `internal/adapters/`, `internal/mcp/`, `internal/dispatch/`, `internal/enrichment/`, `internal/intelligence/`, `internal/testkit/`, and all others
- AC3: No functional test behavior changes — this is purely a reporting improvement
- AC4: All tests still pass with `go test ./... -race`
- AC5: `make lint` passes with zero warnings

## Technical Notes

- Pattern to find: functions like `func setupTest(t *testing.T)`, `func assertTaskEqual(t *testing.T, ...)`, `func createMockProvider(t *testing.T)` that don't have `t.Helper()`
- `t.Helper()` must be the FIRST statement in the function body (before any other logic)
- Do NOT add `t.Helper()` to `Test*`, `Benchmark*`, or `Fuzz*` functions — only to helper functions
- Do NOT add `t.Helper()` to functions that don't accept `*testing.T`/`*testing.B`
- `testkit` package helpers are especially important since they're shared across many packages
- This is a mechanical change — can be done with search-and-verify pattern

## Tasks

1. Grep all `*_test.go` files for functions accepting `*testing.T` or `*testing.B`
2. Filter out `Test*`/`Benchmark*`/`Fuzz*` functions
3. Check each remaining function for `t.Helper()` call
4. Add `t.Helper()` as first statement where missing
5. Run `make test` to verify no regressions
6. Run `make lint` to verify clean
7. Run `go test -race ./...` for final verification

## Audit Results

Exhaustive audit of all 358 `*_test.go` files across the entire codebase found **zero violations**. All 130+ test helper functions that accept `*testing.T` or `*testing.B` already include `t.Helper()` or `b.Helper()` as their first statement.

**Verification:**
- `make test` — all tests pass
- `make lint` — zero warnings
- `go test -race ./...` — all tests pass with race detector
- Packages audited: `internal/cli/`, `internal/core/`, `internal/tui/`, `internal/adapters/`, `internal/mcp/`, `internal/mcpbridge/`, `internal/dispatch/`, `internal/enrichment/`, `internal/docaudit/`, `internal/testkit/`, and all sub-packages

**Conclusion:** The codebase achieved 100% t.Helper() compliance prior to this story. No code changes required — all ACs are already satisfied.
