# Story 27.4: Energy Level Matching & Time-of-Day Inference

## Story Info

- **Epic:** 27 — Daily Planning Mode
- **Priority:** P1
- **Status:** In Review (PR #354)
- **Estimated Effort:** Small-Medium
- **Dependencies:** 27.1 (planning data model — merged PR #323)

## User Story

As a ThreeDoors user, I want focus task suggestions sorted and filtered by my current energy level so that I see the most suitable tasks first and my door selection reflects my energy-appropriate capacity.

## Acceptance Criteria

### AC 27.4.1: Energy Match Scoring (Gradient)
- [x] `EnergyMatchScore(task *Task, energy EnergyLevel) float64` returns a gradient score (0.0–1.0)
- [x] Perfect match (e.g., high energy + deep-work) returns 1.0
- [x] Adjacent match (e.g., high energy + medium effort) returns 0.5
- [x] Mismatch (e.g., high energy + quick-win) returns 0.25
- [x] Tasks with no effort tag return 0.75 (neutral, slightly below perfect)

### AC 27.4.2: Energy-Based Sorting and Filtering
- [x] `SortByEnergyMatch(tasks []*Task, energy EnergyLevel)` sorts tasks in-place by match score (best first)
- [x] Stable sort — tasks with equal scores maintain original order
- [x] `FilterByEnergy(tasks []*Task, energy EnergyLevel) []*Task` returns only tasks where `MatchesEnergy()` is true
- [x] Both functions handle nil/empty slices gracefully

### AC 27.4.3: Energy-Aware Door Selection
- [x] `SelectDoorsWithEnergy(pool *TaskPool, count int, energy EnergyLevel, planningTimestamp *time.Time) []*Task`
- [x] Adds energy match score to candidate set scoring alongside diversity and focus boost
- [x] Energy score weight is additive (same scale as diversity score)
- [x] Falls back to regular selection when energy is empty string

### AC 27.4.4: Time-of-Day Display Labels
- [x] `EnergyLabel(energy EnergyLevel) string` returns human-readable label: "High", "Medium", "Low"
- [x] `TimeOfDayLabel(t time.Time) string` returns "morning", "afternoon", "evening"
- [x] `EnergyDisplayString(energy EnergyLevel, t time.Time, overridden bool) string` returns formatted display:
  - Auto-inferred: "Energy: High (morning)"
  - Overridden: "Energy: Medium (override)"

### AC 27.4.5: Energy Level Cycling
- [x] `NextEnergyLevel(current EnergyLevel) EnergyLevel` cycles High → Medium → Low → High
- [x] Used by TUI for E-key cycling in planning select view
