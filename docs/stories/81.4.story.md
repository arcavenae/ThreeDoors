# Story 81.4: Daemon Refresh for Workspace and Persistent Agent Worktrees

**Epic:** 81 — Git Coordination Fixes
**Status:** Not Started
**Priority:** P2
**Estimate:** M (half day)

## User Story

As a multiclaude operator,
I want the daemon to automatically refresh the workspace worktree and the shared persistent-agent checkout,
So that all agents and the human operator always work with reasonably current code without manual intervention.

## Context

The multiclaude daemon's refresh loop (`multiclaude refresh`) currently only targets worker worktrees. This leaves two environments perpetually stale:

1. **Workspace worktree** — where the human operator works. Drifts behind main during active sessions.
2. **Shared persistent-agent checkout** — where merge-queue, pr-shepherd, envoy, project-watchdog, arch-watchdog, and retrospector run. Their local refs and working tree drift behind main.

Stories 81.1-81.3 provide immediate mitigations (agent self-fetch, operator /refresh). This story addresses the root cause by extending daemon refresh to cover all worktree types.

## Acceptance Criteria

- **AC1:** The daemon refresh loop includes workspace-type worktrees in its refresh cycle
- **AC2:** The daemon refresh loop includes a `git fetch origin main` for the shared persistent-agent checkout
- **AC3:** Refresh interval for workspace/persistent checkout is configurable (default: 5 minutes, matching worker refresh)
- **AC4:** Refresh does NOT interrupt active git operations (check for `.git/rebase-merge` or `.git/MERGE_HEAD` before refreshing)
- **AC5:** Refresh failures are logged but do not crash the daemon

## Technical Notes

- This requires changes to the multiclaude daemon source code (Go binary at `/Users/skippy/work/multiclaude/`)
- The daemon already has a refresh loop for worker worktrees — extend it, don't create a separate loop
- For the shared checkout: `git fetch origin main` suffices (no rebase needed — persistent agents work from refs, not working tree)
- For workspace worktree: full `git fetch + rebase` is needed since the human works with the actual files
- Must handle the case where workspace has uncommitted changes (stash or skip)

## Dependencies

- Requires modifying upstream multiclaude daemon code
- May need coordination with multiclaude maintainer if submitting upstream

## Out of Scope

- Per-agent isolated worktrees (architectural change for persistent agents — would be a separate epic)
- Worker worktree refresh changes (already working)
