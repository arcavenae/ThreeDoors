# Story 65.3: Remaining Command Coverage — Task, Sources, Connect, Extract

## Status: Done (PR #736)

**Epic:** 65 — CLI Test Coverage Hardening
**Priority:** P1
**Depends On:** None

## User Story

As a developer,
I want comprehensive test coverage for all remaining CLI commands,
So that the overall `internal/cli` package reaches ≥70% coverage and no user-facing command is untested.

## Context

After Stories 65.1 (core paths) and 65.2 (subcommands), the remaining CLI commands need coverage to hit the ≥70% target. This includes task management commands, source management, connect wizard, extract, and interactive mode. The TEA report set 70% as the minimum acceptable threshold for this critical package.

## Acceptance Criteria

- AC1: Task commands (`task.go`, `task_status_cmds.go`) tested — add, list, complete, block, unblock with mock providers
- AC2: Source management commands tested — list, status, test, connect, disconnect flows
- AC3: Connect command tested — wizard flow with mock provider factory, error handling
- AC4: Extract command tested — text extraction with mock LLM backend
- AC5: Interactive mode tested — REPL loop basics with mock input
- AC6: Doc audit command tested — basic invocation and output
- AC7: Completion command tested — shell completion generation for bash/zsh/fish
- AC8: Flag completion helpers tested — provider names, mood values, config keys
- AC9: Overall `internal/cli` package coverage ≥ 70% (measured by `go test -cover`)
- AC10: All tests pass with `go test -race ./internal/cli/...`

## Technical Notes

- Task commands are thin wrappers around `core` functions — test the CLI layer (flag parsing, output formatting), not the core logic
- Source commands interact with `connection.Service` — mock the service interface
- Extract command needs mock `LLMBackend` — return canned extraction results
- Interactive mode can be tested with piped stdin — use `strings.NewReader` for input
- Use `cobra.Command.SetArgs()` and `cobra.Command.SetOut()` for isolated command testing
- After all three stories complete, run `go test -cover ./internal/cli/...` to verify ≥70%
