# Story 38.6: Fix Alpha Homebrew Formula Template DSL

## User Story

As a ThreeDoors maintainer, I want the alpha Homebrew formula to pass `brew audit --strict` so that CI on homebrew-tap is green and real regressions are not masked by persistent style violations.

## Context

PR #273 (Story 38.1) introduced the `threedoors-a.rb` alpha formula using `on_arm`/`on_intel`/`on_linux` blocks with `url`/`sha256`. Homebrew's DSL does not permit `url` or `sha256` inside these blocks. The stable formula (`threedoors.rb`) was already fixed to use `if Hardware::CPU.arm?` conditionals, but the alpha formula template in ThreeDoors CI was not updated to match.

This causes 7 `brew audit --strict` offenses on every push to `arcaven/homebrew-tap`, making CI permanently red. The fix is in ThreeDoors `ci.yml` (not homebrew-tap), since the formula template gets overwritten on every push to main.

Party mode artifact: [Issue #296 Triage](../../_bmad-output/planning-artifacts/issue-296-alpha-formula-ci-triage.md)

## Acceptance Criteria

### AC-1: Correct Homebrew DSL in Alpha Formula Template
- [x] Alpha formula template in `ci.yml` uses `if OS.mac? && Hardware::CPU.arm?` / `elsif OS.mac?` / `elsif OS.linux?` pattern for URL/SHA256 selection
- [x] No `on_arm`, `on_intel`, or `on_linux` blocks with `url`/`sha256` remain in the template

### AC-2: Formula Passes Audit
- [ ] Generated `threedoors-a.rb` passes `brew audit --strict --formula threedoors-a`
- [ ] CI on `arcaven/homebrew-tap` is green after the fixed formula is pushed

### AC-3: Functional Correctness

- [ ] `brew install arcaven/tap/threedoors-a` still works on macOS Intel
- [ ] Linux binary URL is still included in the formula for Linux users

### AC-4: Template Validation
- [x] Formula template passes `ruby -c` syntax validation before pushing (existing CI step)
- [x] Description does not start with formula name (existing style requirement)

## Technical Notes

- The fix is in ThreeDoors repo's `.github/workflows/ci.yml`, not in `arcaven/homebrew-tap`
- The formula is regenerated on every push to main, so fixing the template is the only durable fix
- The `install` block in the formula already correctly uses `if OS.mac? && Hardware::CPU.arm?` ‚Äî only the `url`/`sha256` section needs updating
- The stable formula is managed by GoReleaser and is not affected by this change
