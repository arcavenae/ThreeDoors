## Story

As a project owner,
I want CODEOWNERS protection on governance-critical files with `require_code_owner_review` enabled,
So that AI agents cannot modify project philosophy, agent definitions, CI pipelines, or planning docs without human approval.

## Background

ThreeDoors has 800+ merged PRs, all processed by merge-queue without human review of governance files. While this has worked (agents follow prompt-level instructions), it relies on prompt compliance — not mechanical enforcement.

CODEOWNERS creates a two-tier merge system within a single ruleset:
- Files WITH CODEOWNERS entries: require human review to merge
- Files WITHOUT CODEOWNERS entries: merge with CI-only gates (existing behavior)

Per Q-C-001: Apply CODEOWNERS immediately. Per Q-C-002: agents/*.md requires human review. Per Q-007: story files remain ungated (workers must update story status to Done).

## Acceptance Criteria

**AC1: CODEOWNERS file created**
**Given** no CODEOWNERS file exists
**When** `.github/CODEOWNERS` is created
**Then** it protects:
  - `SOUL.md` — @skippy (project philosophy)
  - `CLAUDE.md` — @skippy (agent instructions)
  - `.claude/` — @skippy (agent rules, settings)
  - `ROADMAP.md` — @skippy (scope control)
  - `docs/prd/epic-list.md` — @skippy (epic definitions)
  - `docs/prd/epics-and-stories.md` — @skippy (story breakdown)
  - `docs/decisions/BOARD.md` — @skippy (architectural decisions)
  - `.github/` — @skippy (CI/CD, CODEOWNERS itself)
  - `agents/` — @skippy (agent behavior definitions)
**And** all other paths are ungated (no CODEOWNERS entry)

**AC2: Branch ruleset updated to require code owner review**
**Given** the existing branch protection ruleset on main
**When** `require_code_owner_review` is enabled
**Then** PRs touching CODEOWNERS-covered files require @skippy approval
**And** PRs touching only uncovered files can merge with CI-only gates (0 required approvals)

**AC3: merge-queue handles CODEOWNERS-blocked PRs**
**Given** merge-queue processes PRs for merging
**When** a PR touches CODEOWNERS-protected files
**Then** merge-queue detects the review requirement and skips the PR (does not attempt to merge)
**And** merge-queue labels the PR with `status.needs-human` or equivalent
**And** merge-queue sends a message to supervisor noting the PR needs human review

**AC4: Agent definitions updated for CODEOWNERS awareness**
**Given** merge-queue and pr-shepherd process PRs
**When** their agent definitions are updated
**Then** they include logic to check CODEOWNERS coverage before attempting merge
**And** they understand that some PRs require human approval

**AC5: Documentation updated**
**Given** CODEOWNERS changes project workflow
**When** CLAUDE.md is updated
**Then** it documents:
  - Which files are CODEOWNERS-protected
  - That workers should NOT modify protected files without story justification
  - That merge-queue will not merge PRs touching protected files without human approval

## Tasks

```bash
gh api repos/arcavenae/ThreeDoors/rulesets/<id> --method PUT \
  --field 'rules[][type]=pull_request' \
  --field 'rules[][parameters][require_code_owner_review]=true'
```

### Task 3: Update merge-queue agent definition
Add CODEOWNERS awareness: detect protected files in PR diff, skip merge, label PR, notify supervisor.

### Task 4: Update pr-shepherd agent definition

### Task 5: Update CLAUDE.md
Document CODEOWNERS protection, which files are gated, and workflow implications.

### Task 6: Test with a sample PR
Create a test PR that touches a CODEOWNERS-protected file and verify the merge-queue correctly skips it.

## Dev Notes

- CODEOWNERS syntax: `path @owner` — GitHub matches paths from repo root
- Wildcards: `*.md` matches all markdown in root, `docs/**` matches all docs recursively
- Order matters: later rules override earlier ones in CODEOWNERS
- GitHub requires the CODEOWNERS file be in `.github/`, `docs/`, or repo root
- The existing branch ruleset ID can be found via `gh api repos/arcavenae/ThreeDoors/rulesets`
- Critical: Story files (`docs/stories/`) must NOT be in CODEOWNERS — workers need to update status
- Marvel portability: CODEOWNERS is the first layer of the DFCP pattern — design for extraction into .dfcp.yaml
