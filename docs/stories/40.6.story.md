## Story

As a user,
I want to see session metrics that are currently tracked but never displayed (session duration, fastest door pick, detail views, notes added, streak history, peak hour),
So that I discover personal patterns I didn't know existed and feel the depth of what ThreeDoors knows about my work style.

## Background

ThreeDoors captures 14+ behavioral dimensions in `sessions.jsonl` but only surfaces ~6 to users. The following data is tracked but never shown in the TUI:
- Time to first door (how quickly the user starts engaging)
- Detail view count (curiosity/engagement metric)
- Notes added count (engagement metric)
- Session duration (how long sessions last)
- Streak history (longest and average streaks)
- Peak productivity hour

This story adds a "Session Highlights" section to the insights view and implements `GetSessionHighlights()` in PatternAnalyzer.

## Acceptance Criteria

**Given** sufficient session data exists
**When** the insights view renders
**Then** a "Session Highlights" panel displays with:
  - Total doors opened (all-time)
  - Total tasks completed (all-time)
  - Average session duration
  - Fastest first-door pick time
  - Total detail views opened
  - Total notes added
  - Longest streak (days)
  - Peak productivity hour (e.g., "10am")

**Given** a metric has insufficient data (e.g., no detail views ever opened)
**When** the panel renders
**Then** that metric is omitted rather than showing "0" or "N/A"

**Given** the overview tab is active
**When** the session highlights panel would push the view beyond 24 rows
**Then** the panel is placed on the Detail tab instead (Phase 2 compatibility)

## Technical Notes

- New method: `PatternAnalyzer.GetSessionHighlights() SessionHighlights` in `internal/core/pattern_analyzer.go`
- New type: `SessionHighlights` struct with all computed metrics
- Iterate all loaded sessions to compute aggregates
- Peak hour: bucket `start_time` hours, find mode
- Fastest first door: minimum non-zero `time_to_first_door_seconds`
- Streak history: extend `CompletionCounter` or compute in PatternAnalyzer
- Render as a bordered panel with aligned key-value pairs
- Place on overview if space allows, otherwise on detail tab
