## Story

As a user viewing a vague task,
I want to press E (or type `:enrich`) to have an LLM add context, tags, and effort estimates,
So that sparse tasks become actionable without manual research.

## Background

Task Enrichment is a P1 service. A task like "taxes" becomes "Gather 2025 tax documents (W-2, 1099s, receipts) and schedule appointment with accountant. Due: April 15. Tags: finance, personal. Effort: 3."

The UX shows a before/after diff so the user sees exactly what the LLM added (S4-D3). The user accepts, edits, or discards the enrichment.

**Party mode decision (S1-D3):** Task Enrichment is P1. User-initiated, not automatic.
**Party mode decision (S4-D3):** Always show before/after for enrichment. Transparency builds trust.

## Acceptance Criteria

**AC-1: TaskEnricher service created**
**Given** the services package
**When** `TaskEnricher` is initialized with an `LLMBackend`
**Then** it exposes `Enrich(ctx, task) (*EnrichedTask, error)`

**AC-2: EnrichedTask contains enrichment data**
**Given** an enrichment response
**When** parsed
**Then** `EnrichedTask` has: OriginalText, EnrichedText, Tags ([]string), Effort (int), Context (string)

**AC-3: `:enrich` command in detail view**
**Given** the user is viewing a task in detail view
**When** they press E or type `:enrich`
**Then** the enrichment flow begins

**AC-4: Before/after display**
**Given** enrichment completes
**When** results are shown
**Then** the original task and enriched version are displayed side-by-side
**And** changes are visually highlighted

**AC-5: Accept/edit/discard**
**Given** the enrichment result
**When** user presses A
**Then** the task is updated with the enriched version
**When** user presses E
**Then** an editor opens with the enriched version for manual adjustments
**When** user presses D or Esc
**Then** the enrichment is discarded, original task unchanged

**AC-6: Latency target**
**Given** a single task
**When** enrichment runs
**Then** results appear within 3 seconds (S4-D7)
