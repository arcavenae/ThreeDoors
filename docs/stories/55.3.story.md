## Story

As a developer,
I want a `make test-fast` target that runs tests in short mode,
So that I can get rapid local feedback (~10s) without running the full test suite (~33s).

## Background

The full `make test` runs all tests including slow E2E workflows, time-dependent tests, and large-scale NFR13 variants. For rapid iteration during development, a `-short` mode that skips these expensive tests provides faster feedback. This is a local-only optimization — CI always runs the full suite.

Additionally, `actions/setup-go@v6` supports `cache-dependency-path` for optimized build caching. Verifying this is set correctly can save 10-20s per CI job on recompilation.

## Technical Notes

**File: `Makefile`**

Add after the existing `test` target:
```makefile
test-fast:
	go test -short ./...
```

Update `.PHONY` to include `test-fast`.

**File: `.github/workflows/ci.yml`**

Verify/add `cache-dependency-path: go.sum` to each `setup-go` step:
```yaml
- uses: actions/setup-go@v6
  with:
    go-version-file: 'go.mod'
    cache-dependency-path: go.sum
```

There are three `setup-go` steps: quality-gate (line 56), benchmarks (line 143), build-binaries (line 228). All three should have `cache-dependency-path`.

**Note on `-short` tests:** Currently no tests in the codebase use `testing.Short()` guards, so `make test-fast` will run all tests but without the `-v` flag (faster output). As Phase 2 and 3 add `-short` guards to slow tests, `make test-fast` will become progressively faster. This is intentional — we set up the infrastructure now.
