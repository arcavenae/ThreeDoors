# Story 25.4: Contract Tests & Integration Testing

**Status:** In Review (PR #353)
**Epic:** 25 — Todoist Integration
**Depends On:** Story 25.2

## User Story

As a developer,
I want the Todoist adapter to pass the contract test suite and have comprehensive integration tests,
So that I can be confident the adapter meets the TaskProvider contract and handles edge cases correctly.

## Acceptance Criteria

**Given** the implemented Todoist adapter
**When** the contract test suite runs
**Then:**

- AC1: `adapters.RunContractTests` passes for `TodoistProvider` with mock HTTP server backing
- AC2: Contract tests validate all `TaskProvider` methods: `Name`, `LoadTasks`, `SaveTask`, `SaveTasks`, `DeleteTask`, `MarkComplete`, `Watch`, `HealthCheck`
- AC3: Read-only methods correctly return `core.ErrReadOnly` for `SaveTask`, `SaveTasks`, `DeleteTask`
- AC4: Table-driven field mapping tests cover all 5 Todoist priority values (0, 1, 2, 3, 4)
- AC5: Table-driven tests for `is_deleted` filtering — verify deleted tasks excluded from results
- AC6: Config validation tests: verify error when both `project_ids` and `filter` are set
- AC7: Rate limit test: mock 429 response with `Retry-After` header, verify `RateLimitError` returned
- AC8: Empty response test: verify `LoadTasks` returns empty slice (not nil) when no tasks match
- AC9: Health check test: verify `HealthCheck` returns success when API is reachable, structured error when not
- AC10: Special characters test: verify task content with Unicode, emoji, and special characters maps correctly

## Technical Notes

- Contract test pattern: see `internal/adapters/contract.go` for the `RunContractTests` function
- Mock HTTP server: use `httptest.NewServer` with handler that returns canned Todoist API JSON responses
- Follow the same test structure as `internal/adapters/jira/jira_provider_test.go`
- Integration tests use `//go:build integration` tag — not run in CI, require real Todoist account
- Test fixtures in `internal/adapters/todoist/testdata/` for canned API responses
- Use `t.Parallel()` for independent tests
- Use `t.Helper()` in test helper functions
