# Story 73.2: Remove CronCreate Heartbeats from Startup Checklist

## Status: Reverted — CronCreate restored after observation test failed. Daemon wake loop alone insufficient for workflow triggers. Blocked on 73.6 implementation.

## Epic

Epic 73: Operational Foundation — Agent Reliability & Operator UX

## References

- Research: `_bmad-output/planning-artifacts/multiclaude-operator-ux-research.md` (R-007 S-2)
- Decision: Q-C-011 (drop CronCreate heartbeats — YES, daemon wake loop already nudges every 2 min)

## Story

As a multiclaude supervisor,
I want CronCreate heartbeat crons removed from the startup checklist,
So that the supervisor session is not double-injected by both daemon wake nudges and CronCreate prompts.

## Background

The supervisor startup checklist currently creates 7 CronCreate jobs (one per persistent agent plus a SYNC_OPERATIONAL_DATA cron). These fire prompts into the supervisor's Claude REPL every 7-23 minutes. However, the multiclaude daemon already has a wake loop (`wakeAgents()`, daemon.go:432) that nudges every agent every 2 minutes with role-specific prompts. The CronCreate heartbeats are redundant and cause double-injection — the supervisor receives both daemon wake nudges AND CronCreate prompts, competing for attention and consuming context window.

Per Q-C-011 decision: Drop CronCreate heartbeats immediately. Daemon-native configurable heartbeats will replace them in Story 73.6.
