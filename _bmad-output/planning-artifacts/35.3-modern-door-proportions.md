# Story 35.3: Modern Theme — Door-Like Proportions — Planning Artifact

## Adopted Approach

**Follow Classic (35.2) pattern with Modern-specific styling**

- Reuse `DoorAnatomy` helper for row position calculations
- Height-aware rendering: door mode when `height >= 12`, compact card when `height < 12`
- Modern-specific elements:
  - Heavy top/bottom bars (`━━━` unselected, same selected)
  - No corner characters — straight bars across full width for minimalist look
  - Thin panel divider (`─`) for minimalist feel, even in selected state
  - Minimalist handle: `○` (open circle) instead of Classic's `●` (filled)
  - Content with 3-char left padding (matching existing modern spacing)
  - Threshold line using `▔` below door (same as Classic)

**Rationale:** The Classic 35.2 implementation proved the DoorAnatomy pattern works well. Reusing it for Modern keeps the codebase consistent. The styling choices (thin divider, open circle handle, heavy bars) differentiate Modern from Classic while maintaining the door metaphor.

## Rejected Options

### Option A: Custom row calculations per theme
- **Why rejected:** DoorAnatomy already provides correct proportions (~45% panel divider, ~60% handle). No need for theme-specific math.

### Option B: Use `┄` (dashed line) for panel divider
- **Why rejected:** The AC specifies `─` (thin solid line). Dashed lines would add visual noise to the minimalist theme.

### Option C: Use `◦` (small open circle) for handle
- **Why rejected:** `○` (white circle U+25CB) matches the minimalist aesthetic better and is in the allowed Unicode geometric shapes range. `◦` may not render consistently across terminals.

### Option D: Different threshold character for Modern
- **Why rejected:** Consistency across themes. The `▔` threshold is a shared door metaphor element. Changing it per theme adds unnecessary complexity.
