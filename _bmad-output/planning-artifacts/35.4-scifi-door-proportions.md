# Story 35.4 Planning Artifact — Sci-Fi Theme Door-Like Proportions

## Adopted Approach: Row-by-row rendering using DoorAnatomy

**Rationale:** Use the `DoorAnatomy` model from Story 35.1 to calculate structural row positions, then render the Sci-Fi theme as a proper bulkhead/airlock door with:
- Double-line outer frame (`╔═╗║╚═╝`) — existing Sci-Fi style, preserved
- Shade rails (`░`/`▓`) on sides — existing Sci-Fi feature, preserved
- Bulkhead divider at `PanelDivider` row using `╞═════════╡` (T-junction with horizontal bars)
- Access panel handle `◈──┤` at `HandleRow` on right side
- `[ACCESS]` label right-aligned in lower panel area
- Floor grating line (`▓▓▓`) as final row below bottom frame
- Compact fallback (height=0 or height < MinHeight): use existing fixed-layout rendering

**Implementation:** Split `scifiRender` into compact mode (existing code) and door-like mode (new row-by-row builder). Each row is one of: top border, content, bulkhead divider, handle row, ACCESS label, blank, bottom border, floor grating.

**Testing:** Add `scifi_door_test.go` with height-aware tests for structural elements (divider, handle, grating). Add golden tests at h14, h16, h24 for both selected/unselected states. Update existing compact tests to use height=0 explicitly.

## Rejected Options

### Option A: Template-based rendering
Define door layout as a multi-line string template with row placeholders.
**Rejected because:** Templates are rigid and don't adapt well to variable heights and content wrapping. The row-by-row builder approach is the established pattern in this codebase (used by Classic 35.2, Modern 35.3).

### Option B: Modify existing render in-place without compact fallback
Replace the current fixed-layout render entirely with the new height-aware version.
**Rejected because:** Breaks backward compatibility when height=0 (which is what the existing golden tests pass). Following the Classic theme pattern of compact fallback preserves existing behavior.

### Option C: Use Lipgloss nested styles for door proportions
Use Lipgloss borders and padding to approximate door structure.
**Rejected because:** Lipgloss borders don't support mid-door dividers, handle placement at specific rows, or floor grating. Row-by-row rendering gives precise control over every element.
