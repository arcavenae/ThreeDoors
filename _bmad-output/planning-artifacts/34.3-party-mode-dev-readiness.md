**Story:** 34.3 — Story Template Update and Integration Notes

## Adopted Recommendations

1. **Refine AC1 to verify-first approach** — The current template from 34.2 already has DRY guards (steps 4-5 say "Do NOT include Pre-PR checklist" and "Do NOT include Dev Notes that duplicate CLAUDE.md rules"). AC1 should verify compliance rather than assume changes are needed.

2. **Add structure guidance for integration notes** — Specify concrete sections: document purpose, layered architecture, DRY story approach, slash command reference, story author guide.

3. **Layered information model as architectural concept** — The integration notes should explain the four-layer model:
   - SOUL.md = philosophy/values (why)
   - CLAUDE.md = standards/rules (how)
   - Slash commands = workflow automation (do)
   - Story files = task-specific context (what)

4. **Add explicit template audit task** — First task should audit the existing template before making changes.

## Rejected Options

1. **Adding cross-references from CLAUDE.md to integration notes** — Rejected as scope creep. CLAUDE.md modifications are explicitly out of scope for 34.3.

2. **Creating a separate "story author checklist"** — Rejected as over-engineering. The integration notes doc covers this in its "For Story Authors" section without needing a separate document.

3. **Modifying SOUL.md to reference the integration notes** — Rejected. SOUL.md changes are out of scope, and SOUL.md should remain focused on philosophy, not tooling logistics.
